## 監査範囲と制約

- 対象リポジトリとHEAD
- 監査日時
- 利用可能なSkillカタログと探索root
- repository-owned / externalの判定基準、同名entryの統合・variant件数

- 取得できなかったSkill、履歴、review、外部情報
- 作業ツリーの既存差分
- 依頼モード: 監査のみ / 監査と適用 / 候補指定の適用
- 許可された変更と、許可されていない高影響操作

## 変更適用の状態

- 監査のみ: 「既存Skillは変更していない」と明記する。
