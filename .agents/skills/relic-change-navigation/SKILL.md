# Relic Navigation Change

## 対象と状態遷移を確定する

1. 調査、説明、レビューだけの依頼では編集しない。追加、変更、修正が明示された場合だけ実装する。
2. `git status --short` を確認し、無関係な差分とユーザーの変更を保護する。
3. `docs/features/commands.md`、`docs/features/navigation.md`、`docs/design/DESIGN.md` の対象節を読む。
4. タブを閉じる、ワークスペースを切り替える、ウィンドウを閉じる変更ではeditorの保存契約も確認する。
5. コマンド、候補検索、ナビゲーション状態、タブstore、表示モーションのどこを変えるか分け、受入条件を状態遷移として定める。

## コマンドと候補を一貫させる

1. コマンドID、表示条件、ラベル、ショートカット、actionを一つの登録境界で追跡できるようにする。
2. macOSのMetaとWindows・LinuxのCtrlを同じ主修飾キー契約で扱い、入力欄やIME変換中の意図しないショートカット実行を防ぐ。
3. コマンドパレットとクイックスイッチャーを同時に表示せず、同じ起動操作では開閉を切り替える。
4. 入力変更時に選択候補を先頭へ戻し、上下、Enter、hover、Escape、外側クリックの選択・終了規則を一致させる。
5. 退場表示中の追加終了を二重処理せず、終了後に古い候補やフォーカスを残さない。
6. クイックスイッチャーは現在のワークスペース内Markdownだけを対象にし、ファイル名、相対パス、`aliases` と現行件数上限を保つ。
7. 既存ファイルを選んだ場合は重複タブを作らず、未表示ならフォーカス中ペインのアクティブタブ右側へ開く。
8. 全文・タグ検索の契約変更は `$relic-change-search-replace` に委ねる。

## タブとペインの不変条件を守る

1. Markdown、画像、PDF、パネル、チャートのkindを保ち、パネルとチャートには安定したIDを使って同じ画面を重複作成しない。
2. 新規タブの挿入位置、表示履歴に基づくclose後の選択、タブ0件時の空状態を現行規則に合わせる。
3. ピン留めタブを先頭側へ保ち、複数close操作では対象外にする。単一closeや明示的解除との違いをテストする。
4. drag中のタブ、移動元・移動先ペイン、drop位置、フォーカス、active tabを一つのtransactionとして整合させる。
5. 分割開始、共有タブ、右ペインだけのタブ、分割解除時の統合とactive tab選択を別々に扱う。
6. close、ワークスペース切替、ウィンドウ終了の前に未保存内容のflushと失敗・外部競合の停止条件を迂回しない。分割解除はタブを統合し、不要な保存要求で妨げない。
7. 退場中のタブやペインをクリック、再close、drag開始の対象にせず、モーション終了後だけstoreから除く。
8. 同じタブを両ペインで共有する場合、片側から閉じてももう片側に残る本体を削除しない。

## 入口とパネルを整合させる

1. 左レールの入口を機能トグル、開いている状態、アクティブ状態、対象タブのkindから導出する。
2. パネルタブとチャートタブを区別し、グラフ・年表を開くときだけ現行仕様どおりサイドバー状態を変える。
3. 右パネルの利用可能view、最後に選んだview、開閉状態、単一view時のショートカットを一貫させる。
4. ワークスペース未選択、読込中、空タブ、分割中、機能トグル無効を通常状態と同じ経路で扱う。
5. 表示位置、サイズ、テーマなどの永続化は既存の保存先だけを使い、タブ本文やMarkdownへ不可視状態を埋め込まない。
6. 見た目、spacing、hover、focus、モーション表現も変える場合は `$relic-change-ui` の状態別確認を追加する。
7. 退場待ちを含むJavaScriptとCSSのモーションは、動きを減らすOS設定でも状態確定の時刻が食い違わないよう既存方針を保つ。

## 検証する

1. コマンドの表示条件、OS別表記、実行action、相互排他、候補0件、キーボード・mouse操作を確認する。
2. クイックスイッチャーのaliases、同名、件数上限、既存タブ、左右ペイン、ワークスペース切替を確認する。
3. Markdown・画像・PDF・パネル・チャートについて、追加位置、重複防止、履歴選択、ピン留め、一括close、drag、共有、分割統合、保存失敗を対象テストで確認する。
4. レール、サイドバー、右パネル、機能トグル、各空・読込・退場状態のrendererテストを更新する。
5. `app/` で対象rendererテストと `pnpm typecheck` を実行し、責務境界へ触れた場合は `pnpm architecture:check`、影響が広い場合は `pnpm verify` を実行する。
6. 入力感やdrag・focus・動きを減らすOS設定を自動テストで判断できない場合だけ、その作業で起動した一時データの開発版で確認する。
7. commands、navigation、design、editorの該当正本と差分を照合し、コミット時は `$relic-commit` に従う。

## 完了する

変更した状態遷移、維持したタブ・ペイン契約、保存保護、OS差、更新した正本、テスト、実画面確認を報告する。
