## 基本Markdown

Relicでは次を使用できる。

- H1〜H6見出し
- 太字、斜体、打ち消し線、インラインコード
- 引用、箇条書き、番号付きリスト、チェックボックス
- 水平線、表、コードブロック、脚注
- Markdownリンク、相対パス画像
- インライン数式 `$...$`、ブロック数式 `$$...$$`
- ハイライト `==...==`
- 下線 `<u>...</u>`

ライブプレビューではチェックボックスをクリックでき、Markdownテーブルをセル単位で編集できる。表はヘッダー行の次に区切り行を置く標準形式で書く。

```markdown
| 項目 | 状態 | 期限 |
|---|---|---|
| 調査 | 進行中 | 2026-08-01 |
```

HEXカラーだけのセルは色見本も表示される。色コードは補助情報として扱い、色名や用途を別列へ書く。

## Relic拡張記法

### 内部リンク

```markdown
[[ファイル名]]
[[フォルダ/ファイル名]]
[[ファイル名|表示テキスト]]
[[ファイル名#見出し]]
[[ファイル名^block-id]]
![[ファイル名]]
```

ブロック参照先は段落末尾へ `^block-id` を手入力する。ファイル埋め込みはMarkdownだけを一段階展開し、埋め込み先では直接編集しない。20,000文字を超えるファイルの全文埋め込みへ依存しない。

### フレーバーテキスト

`flavortext` はカードビューの短い説明であり、通常本文を置き換えない。最初の完結したブロックだけが使われ、内部はMarkdownとして再解釈されない。

````markdown
```flavortext
調査対象の概要を、改行を含めて短く記す。
```
````

### 図表

MermaidとD2のコードブロックは図として表示され、Markdownにはソースが残る。

````markdown
```mermaid
flowchart LR
  A[入力] --> B[確認]
  B --> C[出力]
```
````

````markdown
```d2
input -> review -> output
```
````

図表内の外部画像やHTMLへ依存しない。ソースに機微情報を含めない。

## フロントマター

フロントマターはファイル先頭のYAMLだけを認識する。すべて任意であり、新規資料へ無条件に付けない。

```yaml
---
aliases:
  - 別名
category: 調査
card: ./images/overview.webp
tags:
  - 資料/一次情報
  - 状態/確認済み
chronicle:
  calendar: 基準暦
  start: 2025
  end: 2026
status: published
reviewed: true
reviewed_at: 2026-07-20
---
```

### 固定プロパティ

| キー | 型と意味 | 注意 |
|---|---|---|
| `aliases` | 文字列の配列。別名検索と内部リンク候補 | 他ファイルの名前を別名として複製しない |
| `tags` | 文字列の配列。タグ検索と横断分類 | 本文の `#タグ` は通常のタグ検索に使われない |
| `category` | 単一値。主分類 | 配列にしない |
| `card` | 空値または代表画像の相対パス | キーがあればカード対象になる |
| `chronicle` | `calendar`、`start`、`end` の対応 | 年は0以外の整数、`start <= end` |

`chronicle` の単年も開始年と終了年を同じ値で明記する。

```yaml
chronicle:
  calendar: 基準暦
  start: 1185
  end: 1185
```

負の年は使えるが0年は使えない。設定されていない暦名は年表に表示されないため、既存ワークスペースの暦設定を確認する。

### 任意プロパティ

任意のトップレベルプロパティはテーブルビューの列にでき、全文検索の対象にもなる。同じ資料群では次を揃える。

- キー名: `status` と `状態` のような重複を作らない。
- 型: 同じキーへ文字列、配列、真偽値を混在させない。
- 列挙値: `draft`、`Draft`、`下書き` のような表記揺れを避ける。
- 日付: `YYYY-MM-DD` にする。
- 配列: YAMLブロック配列にする。

任意プロパティはテーブル表示には使えるが、フロントマター検索UIで選べるのはRelicが認識する固定プロパティに限られる。任意プロパティは全文検索またはテーブルの絞り込みで使う。

## リンクと埋め込み

- 拡張子なしの内部リンクを使用できる。
- 同じフォルダの同名ファイルが優先され、ワークスペース全体で一意なら別フォルダの同名ファイルにも解決される。
- パス付きリンクはリンク元ファイルのフォルダを基準に書く。たとえば `人物/名前.md` から `場所/都市.md` を指す場合は `[[../場所/都市]]` とする。ワークスペースルート基準のように `[[場所/都市]]` と書かない。
- 同名ファイルが複数ある場合は、各リンク元からの相対パスを含むリンクを使う。
- 未作成の内部リンクは、クリック時にリンク元と同じフォルダへ新規ファイルを作る。
- Markdown標準リンクのリンク先へ `relative/path.md` を指定した場合も、リンク元基準で開ける。
- 外部URL、プロトコル付きURL、ページ内アンカーだけのリンクはワークスペースファイルとして扱われない。
- Wikiリンクと相対Markdownリンクはグラフ・スフィアのファイル間接続になる。
- リネームや移動時は内部リンクが追跡更新されるが、コードブロック内は更新されない。

内部リンクは概念の関係、Markdownリンクは出典URLや移植性を重視する相対参照に使う。

## 検索と各ビューへの影響

| 機能 | 資料側で用意するもの |
|---|---|
| アウトライン | 意味のあるH1〜H6階層 |
| 全文検索 | 本文とフロントマターに一貫した用語を使う |
| ファイル名検索 | 明確なファイル名と必要な `aliases` |
| タグ検索 | フロントマター `tags` |
| フロントマター検索 | 固定プロパティと値 |
| カードビュー | `card` と任意の最初の `flavortext` |
| テーブルビュー | 比較軸として揃えたトップレベルプロパティ |
| グラフ・スフィア | 内部リンク、相対Markdownリンク、`tags`、必要な画像参照 |
| クロニクル | 有効な `chronicle` と分類用 `category` |
| 右パネル | 見出し、内部リンク、相対Markdownリンク |

本文中の `#語` はグラフ・スフィアでは可視化用タグノードになるが、通常のタグ一覧やタグ検索には使われない。意図しないグラフノードを避けるため、通常の文章でハッシュ記法を乱用しない。

カード、テーブル、グラフ、スフィア、クロニクルはMarkdownから得た派生表示であり、本文を置き換えない。各ビューを使わなくても資料の意味が伝わるようにする。

## 画像・PDF・出力

- 実画像として表示されるのは、ワークスペース内の `.avif`、`.bmp`、`.gif`、`.jpeg`、`.jpg`、`.png`、`.svg`、`.webp` への相対パスだけである。
- 外部URL画像、絶対パス、ワークスペース外へ出る `..` 参照は実画像として読み込まれない。
- Markdown画像は `![代替テキスト](relative/path.png)` とし、代替テキストへ内容または目的を書く。
- PDFは添付として閲覧できるが、本文検索、タグ、注釈、Markdown変換の対象ではない。PDFの要点と出典は別のMarkdownへ記す。
- Markdown本文はPDF出力できる。アプリUI、右パネル、バックリンク一覧はPDFへ含まれない。
- Mermaid・D2は可能な限り図としてPDFへ出力される。重要な意味は本文でも説明する。
- 出力文書では外部スクリプトや外部リソースを前提にしない。

## 避けること

- Relic専用の不可視メタデータを想定する。
- すべてのファイルへ同じフロントマターを機械的に付ける。
- 本文に必要な説明をカード、テーブル、グラフだけへ委ねる。
- 分類目的のタグと、個別概念への内部リンクを混同する。
- グラフ密度を上げるためだけにリンクや本文 `#タグ` を増やす。
- 未確認の年を `chronicle` へ入れる。
- 外部URL画像がRelic内で表示されると仮定する。
- 多段埋め込み、巨大ファイル埋め込み、埋め込み先での直接編集を前提にする。
- 生HTMLによるレイアウトや装飾へ依存する。Relicが明示的に保証するHTML由来の装飾は下線だけである。
