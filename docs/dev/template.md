# dev/template.md

今後の開発プロジェクトへ流用するための文書・リポジトリ構造テンプレート。

この文書はRelic固有の仕様ではなく、AIと人間が同じ前提で開発を継続するための汎用構造を定義する。

---

## ルート構成

ルート直下は、プロジェクトの入口と実体ディレクトリだけに絞る。

| パス | 役割 |
|------|------|
| `README.md` | 人間向けの入口。概要、起動、検証、配布手順を書く |
| `AGENTS.md` | AIエージェント向けの共通ルール |
| `SECURITY.md` | セキュリティ方針 |
| `app/` | アプリ本体 |
| `docs/` | 文書群 |
| `scripts/` | 人間が実行する補助スクリプト |

---

## docs構成

| パス | 役割 |
|------|------|
| `docs/INDEX.md` | 文書索引 |
| `docs/_rules.md` | 文書作成・管理ルール |
| `docs/product/` | プロダクト意図、計画、用語など、そのプロジェクト固有の前提 |
| `docs/dev/` | 開発運用、フェーズ、規約、テスト、未決定事項 |
| `docs/dev/phases/` | 現在フェーズの正本文書 |
| `docs/spec/` | 機能仕様 |
| `docs/architecture/` | アーキテクチャ、データモデル、設計判断 |
| `docs/tech/` | 技術選定、技術調査 |
| `docs/ui/` | UI構成、画面遷移、デザイン原則 |

---

## AIの読み込み導線

AIは以下の順で読む。

1. `AGENTS.md`
2. `docs/product/project.md`
3. `docs/dev/phases.md`
4. `docs/dev/phases.md` が指定する現在フェーズの正本文書
5. 必要になった詳細文書だけ

この導線を壊す変更は禁止する。

---

## フェーズ管理

- フェーズ状態の正本は `docs/dev/phases.md`
- 現在フェーズ正本は `docs/dev/phases/` 配下に置く
- フェーズIDとファイル名の先頭は一致させる
- 完了済みフェーズの詳細は、必要な履歴だけ現在フェーズ正本へ統合する
- 次フェーズへはユーザーが明示した場合だけ移行する

---

## 未決定事項と作業記録

- 未決定事項は `docs/dev/open-questions.md` に集約する
- 解決済みの履歴は現在フェーズ正本の統合済み履歴と反映先文書を参照する
- 作業記録は現在フェーズ正本に日時付きで書く
- 作業記録を書いた場合は、その時点の全差分をまとめてコミットし、pushする

## 検証導線

- 実装の検証コマンドは `docs/dev/testing.md` に集約する
- アプリ本体をサブディレクトリに置く場合は、型チェック、テスト、ビルドの実行場所を必ず明記する
- リポジトリ全体の文書整理では、実装テストと別に Markdown 差分の確認方法を明記する
- 文書整理だけの作業では、コードテストを省略する理由と、代わりに確認した文書整合を現在フェーズ正本へ残す

---

## 整理時の原則

- 実装、仕様、技術選定、UI方針の変更を文書整理に混ぜない
- ファイル移動時は同時に参照を更新する
- ルート直下へ新しい補助ファイルを増やさない
- プロジェクト固有の判断を、汎用ルール文書へ混ぜない
