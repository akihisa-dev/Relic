# dev/conventions.md

このファイルはコーディング規約を定義するドキュメントです。
ファイル・フォルダの命名規則・Electron / React / TypeScript のアーキテクチャパターン・コメントの書き方など、
AIがコードを書く際に従うべきルールをここに記述します。

---

## 基本方針

- TypeScriptで実装する
- 仕様は `docs/spec/`、設計判断は `docs/architecture/`、技術スタックは `docs/tech/stack.md` を正とする
- ファイルシステム・Git・ネットワークに触る処理はメインプロセスに置く
- UIとエディタ操作はレンダラープロセスに置く
- レンダラーからOS機能へ直接アクセスしない。必ずIPCを通す
- ユーザーのMarkdownファイルを勝手に独自形式へ変換しない
- 追加機能より、ローカルMarkdownファイルを安全に読み書きできることを優先する

---

## ディレクトリ構成

初期実装では、以下の責務分離を基本にする。

| 場所 | 内容 |
|------|------|
| `src/main/` | Electronメインプロセス。ファイル操作・Git操作・設定保存・IPCハンドラ |
| `src/preload/` | レンダラーへ公開する安全なAPI |
| `src/renderer/` | React UI・CodeMirror・画面状態 |
| `src/shared/` | メイン / レンダラーで共有する型・定数・純粋関数 |
| `src/test/` | テスト共通ユーティリティ |

必要になったら、各層の中で機能単位のサブディレクトリを作る。

## 命名規則

| 対象 | 形式 | 例 |
|------|------|----|
| Reactコンポーネント | PascalCase | `FileTree.tsx` |
| TypeScriptファイル | camelCase | `workspaceStore.ts` |
| 型・interface | PascalCase | `Workspace`, `MarkdownFile` |
| 関数 | camelCase | `openWorkspace` |
| 定数 | camelCase または UPPER_SNAKE_CASE | `defaultEditorSettings`, `MAX_FRONTMATTER_FIELDS` |
| テストファイル | 対象ファイル名 + `.test.ts` | `linkParser.test.ts` |

UIに表示する日本語文言は、後から集約できるようにコンポーネント内へ散らしすぎない。

## TypeScript

- `any` は避ける。外部ライブラリ境界などで必要な場合は範囲を狭くする
- IPCの入力・出力は共有型で定義する
- ファイルパスは、絶対パスとワークスペース相対パスを混ぜない
- ワークスペース相対パスを保存・表示の基本単位にする
- 日時は内部的にはISO文字列または `Date` として扱い、表示時に整形する
- 例外は握りつぶさず、UIで原因別メッセージに変換できる形で返す

---

## Electron / IPC

- メインプロセスは、ファイルシステム・Git・検索インデックス・アプリ設定の責務を持つ
- レンダラーは、画面表示とユーザー操作の責務を持つ
- preloadで公開するAPIは最小限にする
- IPCチャンネル名は機能ごとに分かる名前にする
- レンダラーから任意のファイルパスを直接操作できる汎用APIを作らない
- ワークスペース外のファイルへ書き込む操作は、ユーザーが明示的に選んだパスだけを対象にする

---

## React

- 画面は仕様上のコンポーネント単位に分ける
- UI状態と永続化データを混ぜない
- コンポーネントは表示責務を中心にし、ファイル操作やGit操作はAPI層へ逃がす
- タブ・サイドバー・右パネルなど、画面全体に関わる状態は専用の状態管理単位を持つ
- 深いpropsリレーが増えたら、状態の置き場を見直す

---

## CodeMirror

- Markdown本文の編集状態はCodeMirrorを正とする
- ライブプレビュー・ソースモード・内部リンク・タグなどの挙動は、可能な限りCodeMirror拡張として実装する
- Markdownファイル自体には、Relic専用の不可視メタデータを埋め込まない
- 自動保存はエディタ変更から直接ファイルへ即時書き込みせず、入力停止後1秒の仕様を守る

---

## ファイル操作

- 対象はワークスペース内の `.md` ファイルを基本とする
- サイドバー表示では `.md` 拡張子を隠す
- 作成時は `.md` 拡張子を自動付与する
- 削除は確認ダイアログを出す
- リネーム・移動時は、本文とフロントマター内の内部リンクを確認なしで自動更新する
- コードブロック内の内部リンク文字列は自動更新しない
- 同名・禁止文字・ワークスペース外参照はエラーとして扱う

---

## コメント

- コメントは、処理の意図や仕様上の理由がコードだけでは伝わりにくい箇所にだけ書く
- コードをそのまま言い換えるコメントは書かない
- 仕様に由来する分岐は、関連仕様ファイル名が分かる短いコメントを許容する

---

## ドキュメント更新

- 仕様変更を伴う実装をした場合は、該当する `docs/spec/` または `docs/architecture/` も更新する
- 未決定の問いが生まれたら `docs/dev/open-questions.md` に追加する
- セッション終了時は `docs/journal/YYYY-MM-DD.md` を更新する
