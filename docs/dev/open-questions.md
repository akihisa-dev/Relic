# dev/open-questions.md

未決定事項の正本。
仕様書・設計書・UI文書には未決定事項を書かず、ここに集約する。

---

## 現在の未決定事項

なし。

現在の正本判断で未決定事項はない。解決済みの経緯や過去の判断理由は、この文書へ再掲せず、日誌、過去フェーズ文書、反映先の正本文書で確認する。

---

## 運用

- 新しい未決定事項が出たら、`- [ ] 問い` の形式で追加する
- 各問いには `関連:` で背景となる文書を示す
- 決定したら `[x]` にし、決定内容を該当する仕様書・設計書・ADRへ反映する
- 解決済みの大量履歴をここに溜めない。経緯は `../journal/` と反映先文書を参照する
- 過去の未決定事項を現在基準で説明し直す必要がある場合は、履歴を書き換えず、反映先の正本文書か現在フェーズ正本に整理する
