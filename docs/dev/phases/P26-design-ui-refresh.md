# P26-design-ui-refresh.md

Relicのデザイン・UI刷新フェーズの正本。

このフェーズでは、Relic全体の見た目、情報密度、操作感、画面ごとの視認性と一貫性を、既存機能と既存データを壊さず段階的に刷新する。
ただし、具体的な変更対象はユーザーが指定したもの、または調査後にユーザーへ提示して合意したものだけを対象にする。

---

## 読み込みルール

Relicのデザイン・UI刷新フェーズ。

AIはこのフェーズを前提にユーザーへ接する。
対象はアプリ全体のUIデザイン、画面構成、視覚階層、余白、色、タイポグラフィ、モーション、操作入口、コンポーネント整理であり、保存形式変更、IPC/preload API変更、store状態構造変更、既存機能の仕様変更へ広げる場合は、事前にユーザーへ確認する。

最初に読む:

- この文書
- `../ui/DESIGN.md`
- `../ui/screens-macos.md`
- `../ui/navigation.md`

必要になったときだけ読む:

- 実装規約: `../conventions.md`
- テスト方針: `../testing.md`
- 対象画面に関係する仕様: `../../spec/`
- 対象画面に関係する実装: `../../../app/src/renderer/components/`, `../../../app/src/renderer/hooks/`, `../../../app/src/renderer/styles/`, `../../../app/src/renderer/store/`
- アーキテクチャ前提に触れる場合: `../../architecture/overview.md`, `../../architecture/decisions.md`

注意:

- ユーザーが指定した画面・場所・操作対象、または調査後にユーザーへ提示して合意した対象だけを扱う
- 実装前に、対象、問題、変更範囲、変えてはいけない挙動、検証方法を明示する
- UI変更では、指定された画面・場所・操作対象だけを変更する
- 保存形式変更、IPC/preload API変更、store状態構造変更、仕様変更が必要になった場合は、P26内の通常作業から分けて確認する
- 見た目の刷新でも、既存のファイル操作、編集、検索、リンク、チャート、グラフ、設定の挙動を壊さない
- 変更後は対象に応じたテスト、`pnpm typecheck`、`pnpm test`、`git diff --check` の必要範囲を確認する
- 実アプリ確認が必要な見た目や操作感は、確認した範囲と未確認理由を記録する

---

## フェーズの目的

- Relicの画面全体を、書く道具として落ち着きがあり、視認性が高く、使い続けやすいUIへ刷新する
- 画面ごとの情報階層、余白、濃淡、操作入口、状態表示を整理する
- 既存機能、既存データ、既存ファイル操作を壊さず、変更単位ごとに検証する
- 次回作業者が判断できる粒度で、実施したことだけを簡潔に記録する

---

## 作業方針

- 1回の作業は、画面単位または近い原因を共有するUI問題単位に分ける
- 実装前に、対象、今の問題、変更範囲、変えない挙動、検証方法を明示する
- ユーザーの指定範囲を越える修正候補は、実装せず提案として分ける
- UI刷新と機能仕様変更を混ぜない
- 変更後は、対象テスト、関連回帰テスト、型検査、全体テスト、diff whitespaceの必要範囲を確認する
- このフェーズ文書には日誌を書かない。日付順の作業履歴は `docs/journal/` に分ける

---

## 開始時チェック

UI刷新対象ごとに、実装前に以下を確認する。

- [ ] 対象ファイル・対象画面・対象操作
- [ ] 今の問題、または修正したい違和感
- [ ] 変更してよい範囲
- [ ] 変えてはいけない既存挙動
- [ ] 仕様・保存形式・IPC/preload API・store状態構造への影響有無
- [ ] 最初に作る最小修正
- [ ] 自動テストで確認する範囲
- [ ] 実アプリ確認が必要な範囲

---

## 修正管理

P26では、事前に固定した長い実施リストは置かない。
ユーザー指定、または調査で合意したUI刷新単位ごとに、この文書の「進捗」へ実施したことだけを簡潔に記録する。

完了条件:

- 合意したUI刷新単位が完了している
- 変更範囲に応じた自動テスト、`pnpm typecheck`、`pnpm test`、`git diff --check` の必要範囲を確認している
- 実アプリ確認が必要な項目は、確認結果または未確認理由を記録している
- 次フェーズへ移る場合は、ユーザーが明示している

## 進捗

このフェーズでは、修正単位ごとに次回作業時の判断に必要な状態だけを記録する。
日付順の作業ログ、感想、細かな経緯はここに書かない。

必要な確認結果や未確認事項がある場合も、実施内容の一部として短くまとめる。

### フェーズ開始

- ユーザー指示によりP25を終了し、P26デザイン・UI刷新フェーズへ移行した。
- ユーザー指示により、P26は現行デザインを起点にやり直す。直前の新デザイン試行は採用しない。
- ファイルビューの現状デザイン確認用として、静的モック `docs/ui/mockups/file-view-current.html` を追加した。アプリ本体のUI挙動は変更していない。
- Markdownツールバーを描画から外し、同じMarkdown装飾・挿入コマンドをエディタ右クリックメニューへ移して作業領域を広げた。保存形式、IPC/preload API、store状態構造、編集コマンドの挙動は変更していない。
- チャートタブ上部の `chronicle` / `date` 切替ボタンで、ボタン内テキストが切れないように表示幅と行揃えを固定した。チャート切替などの挙動は変更していない。
- Markdownファイル表示時の右パネル初期状態を閉じた状態に変更した。上部のアウトライン・リンクボタンによる開閉挙動は維持している。
- Markdownファイル表示時のファイル名を本文上部のUI表示へ移し、上部のファイル名・移動行を削除した。ファイル名はMarkdown本文には入れず、ソース・分割・右パネル操作はタブ列右側へ移した。
- Markdownファイル表示時の本文上部ファイル名を、Obsidian風に本文幅へ揃えた大きなタイトル表示へ調整した。Markdown本文への書き込みは行っていない。
- Markdownファイル表示時の本文上部ファイル名を直接編集できるようにした。編集確定時は既存のファイルリネーム処理を使い、Markdown本文にはタイトルを書き込まない。
- Markdownファイル表示時の本文上部ファイル名は、編集中も入力枠を表示しない見た目へ調整した。編集挙動と保存形式は変更していない。
- タブ列右側へ移したソース・分割・アウトライン・リンク操作ボタンは、選択中も枠・背景・上線を表示しない見た目へ調整した。各操作の挙動は変更していない。
- タブ列右側の操作ボタンは幅を固定し、選択中も背景・枠・下線を出さず文字色だけで示す見た目へ調整した。選択切り替え時にボタン位置が揺れないようにした。
- ソース・分割・アウトライン・リンク操作ボタンを分割ペイン内のタブ列から外し、メイン領域右上の共通操作として表示するようにした。アクティブペインに応じた操作対象と状態表示は維持している。
- ファイルサイドバーの新規ファイル・新規フォルダ操作を、横並びのアイコンボタンへ変更した。作成挙動とアクセシブル名は維持している。
- ファイルサイドバーの新規ファイル・新規フォルダ操作は、枠なしのレール風アイコンボタンへ調整した。アイコンは線画のファイル編集・フォルダ追加に変更した。
- メイン領域右上のソース・分割・アウトライン・リンク操作をアイコンボタンへ変更し、名称はホバー時のツールチップで表示するようにした。各操作の挙動とアクセシブル名は維持している。
- 右パネル上部のアウトライン・リンク見出しを削除し、右上アイコンのホバー表示に名称表示を集約した。右パネルの内容表示とアクセシブル名は維持している。
- 右パネル本文の開始位置を下げ、メイン領域右上の操作アイコン列と重ならないようにした。右パネルの内容表示と操作挙動は変更していない。
- メイン領域右上の操作アイコン列の左余白とボタン間隔を詰め、操作列の先頭に不要な空白が出ないようにした。各操作の挙動は変更していない。
- メイン領域右上の操作アイコン列から背景面を外し、右パネル上に余白の塊が残って見えないようにした。各操作の挙動は変更していない。
- 右パネル左端をドラッグして幅を変更できるようにした。幅は220pxから520pxに制限し、右パネルの開閉と内容表示の挙動は変更していない。
- メイン領域右上の操作アイコン列で、選択中のボタンをアクセント色で表示するようにした。背景や枠は表示せず、各操作の挙動は変更していない。
- ファイルサイドバー上部の新規フォルダ操作の右に、フォルダ全開・全閉のアイコンボタンを追加した。既存のフォルダ展開/折りたたみ挙動は変更していない。
- メイン領域右上の操作アイコン列で、選択中のボタンに左レールと同じアクセント背景を付けた。各操作の挙動は変更していない。
- 左レールの設定アイコンを、ユーザー提供のスライダー調整系SVGに差し替えた。設定タブを開く挙動、ツールチップ、アクセシブル名は変更していない。
- 左レールのファイルビュー・ダッシュボード・チャート・設定、ファイルサイドバーの新規ファイル・新規フォルダ、メイン領域右上のソース・分割・アウトライン・リンク、チャートタブのアイコンをユーザー提供SVGに差し替えた。各操作の挙動、ツールチップ、アクセシブル名は変更していない。
- ファイルサイドバーのフォルダ全開・全閉、左レールのグラフビュー、Markdownツールバーとエディタ右クリックメニュー内の太字・斜体・下線・箇条書き・番号付きリスト・表・コードブロック・ハイライト・引用・コピー・カット・ペーストの表示をユーザー提供SVGへ差し替えた。各操作の挙動、ツールチップ、アクセシブル名、編集コマンドは変更していない。
- 右クリックメニューのMarkdown操作をアイコンのみの表示に変更し、ホバー時に名称を表示するようにした。Markdownツールバーの追加アイコン、左レールのツール・チャート、チャートタブ、フォルダ全開・全閉もユーザー提供SVGへ差し替え、ブロックID挿入のUI入口を削除した。内部コマンド、編集挙動、IPC/store/保存形式は変更していない。
- 右クリックメニューのアイコンボタンでホバー名称が確実に表示されるようにし、コピー・カット・ペーストを横並びにした。アイコン押下時はエディタの選択を維持してからMarkdown操作を適用するようにし、IPC/store/保存形式は変更していない。
- 右クリックメニュー表示直後に全アイコンの名称が一瞬出る問題を修正し、すべて選択は文字項目に戻した。Markdownリンクのアイコンをユーザー提供の外部リンクSVGに差し替え、内部リンクとは別表示にした。編集コマンド、IPC/store/保存形式は変更していない。
- 右クリック時にエディタの選択範囲とカーソル位置を維持し、右クリックメニューからMarkdown操作や貼り付けを既存選択へ適用できるようにした。右クリックメニュー内のMarkdownリンク・内部リンク・表を横並びにし、ファイルサイドバーのフォルダアイコンを開閉状態で切り替えるようにした。編集コマンド、IPC/store/保存形式は変更していない。
- 右クリックメニュー内のMarkdown操作を、通常ツールバーの対象探索ではなく右クリック時に保持したEditorViewへ直接適用するようにし、右クリック後の太字などが選択範囲に反映されない問題を修正した。編集コマンド、IPC/store/保存形式は変更していない。
- 右クリックメニュー内のSVGアイコン部分を押した時もメニュー内クリックとして扱うようにし、アイコンのpathを押しても太字などのMarkdown操作が選択範囲へ反映されるようにした。編集コマンド、IPC/store/保存形式は変更していない。
- 右クリックメニュー表示中に選択範囲の見た目が消えないよう、保存済み選択範囲へ表示用ハイライトを重ねるようにした。右クリックメニューはクリック位置から少し離して表示し、本文を隠しにくくした。編集コマンド、IPC/store/保存形式は変更していない。
- 左レールのファイル・ダッシュボード・フロントマター、メイン領域右上の分割、ファイルサイドバーのフォルダ開閉、ファイルツリー右クリックメニューのゴミ箱、設定のライト/ダーク表示をユーザー提供SVGへ差し替えた。各操作の挙動、ツールチップ、アクセシブル名、IPC/store/保存形式は変更していない。
- ファイルサイドバー内の個別ファイル表示で、点アイコンの見た目をユーザー提供SVGへ差し替えた。ファイル選択、右クリック、アクセシブル名、IPC/store/保存形式は変更していない。
- メイン領域上部のタブ列と右上操作ボタン列を同一行の別カラムとして構造上分離し、タブとスクロールバーが右上操作列へ侵食しないようにした。タブ操作、右上操作、IPC/store/保存形式は変更していない。
