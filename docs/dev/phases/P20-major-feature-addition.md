# P20-major-feature-addition.md

Relicの大型機能追加フェーズの正本。

このフェーズでは、P19で整理した現行アプリを土台に、大きめの機能追加を進める。
ただし、機能内容はユーザーが指定したものだけを対象にし、AIが先に大型機能案やチェックリストを作らない。

---

## フェーズの目的

- Relicの中心であるMarkdown執筆・整理・検索体験を拡張する
- 既存のローカルMarkdownワークスペースとしての安定性を保ったまま、大型機能を追加する
- Mac / Windowsの両方で破綻しない設計・実装・検証単位に分ける
- 仕様、UI、保存形式、IPC、テストの整合を取りながら進める

---

## 作業方針

- 最初に、追加する大型機能の目的・使う画面・保存するデータ・既存機能への影響を確認する
- 仕様が未確定のまま実装に入らない
- 1回の実装は、動作確認できる最小の縦切り単位に分ける
- 既存ファイル、既存ワークスペース、既存設定を壊さないことを優先する
- Git / GitHub機能は、ユーザーが明示しない限り復活させない
- 変更後は `pnpm typecheck` と、対象に応じたテストを確認する
- 配布ビルド確認が必要な場合だけ `build:mac:safe` / `build:win:safe` を実行する
- このフェーズ文書には日誌を書かない。日付順の作業履歴は `docs/journal/` に分ける

---

## 開始時チェック

大型機能ごとに、実装前に以下を確認する。

- [ ] 追加する機能名
- [ ] その機能で解決したいこと
- [ ] 主に使う画面・導線
- [ ] 保存するデータの有無
- [ ] 既存Markdownファイルへの影響
- [ ] Mac / Windowsで注意する点
- [ ] 最初に作る最小単位
- [ ] 自動テストで確認する範囲
- [ ] 実アプリ確認が必要な範囲

---

## 機能ごとの進捗

このフェーズでは、ユーザーが指定した大型機能ごとに、次回作業時の判断に必要な状態だけを記録する。
日付順の作業ログ、感想、細かな経緯はここに書かない。

必要な場合だけ、以下を同じ項目内にまとめる。

- 方向性
- 実施
- 確認
- 残り

### ダッシュボード

- 方向性: メインエリアにタブとして開くビューにし、既存Markdownファイルを読み取ってファイル統計を視覚的に表示する。保存データは追加しない。
- 実施: 左レールにダッシュボード入口を追加。総ファイル数、文字数、リンク数、タグ数、文字数上位の縦棒グラフ、フォルダ分布、タグ分布ツリーマップ、任意の選択肢・複数候補プロパティ分布を表示するDashboardPanelを追加。タグ分布ツリーマップは横長カードに合わせた二分割レイアウトへ修正し、色はDESIGN.mdに沿って無彩色主体、最大タグのみブルーの細いシグナルにした。
- 確認: `pnpm typecheck` と `pnpm test -- DashboardPanel.test.tsx` を確認済み。
- 残り: 実アプリでの見た目確認、表示内容の追加要望確認、大量ファイル時の読み込み方法の必要性判断。
