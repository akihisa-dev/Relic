# P15-rejected-ai-ui-ux-polish.md

この文書は、AIが先に方針を作りすぎたため取り下げた非正当フェーズの記録。

これは現在フェーズの正本ではない。
作業成果は戻さないが、この方針・チェックリスト・完了条件を以後のUI/UX洗練の根拠にしてはならない。

---

## 取り下げ理由

- ユーザーへのヒアリングより先に、AIがUI/UX方針とチェックリストを作った
- 「やりたい」という段階の要望に対し、AIが勝手に作業範囲を決めた
- デザイン原則に関わる判断を、ユーザー確認より先に進めた

P14でGitHub連携の安全導入はいったん凍結した。P15ではGitHub機能の追加実装へ進まず、現状のアプリが「なんかつまらないものになっている」問題を解消する。

## 基本方針

- GitHub機能の追加実装は凍結する。
- 既存のGitHub関連UIは、必要なら見た目や導線の整理だけ行う。
- 新機能を増やす前に、現在ある画面の密度、階層、余白、言葉、操作の流れを整える。
- ランディングページ風の派手さではなく、書く・探す・整理する日常操作が気持ちよくなる方向へ寄せる。
- 「説明が多いUI」ではなく、見れば分かる構造にする。
- 既存の仕様を大きく変える場合は、実装前にユーザーへ確認する。
