# P19-refactoring-organization.md

Relicのリファクタリングおよび整理フェーズの正本。

このフェーズでは、既存挙動を保ったまま、コード・テスト・文書・構成を読みやすく、直しやすく、壊れにくい状態へ整える。

---

## フェーズの目的

- P18までに増えたUI/UX実装を、保守しやすい単位へ整理する
- 重複、肥大化、責務の混在、テストしにくい構造を減らす
- 仕様変更や新機能追加ではなく、既存機能の維持と安全な整理を優先する
- 次の開発フェーズに入る前に、変更しやすい足場を作る

---

## 作業方針

- 1回の変更範囲を小さく区切る
- 変更前に、対象・目的・影響範囲・確認方法を短く明示する
- UI文言、操作導線、保存形式、IPC API、既存データ形式を不用意に変えない
- 挙動変更が必要になった場合は、リファクタリングとして進めず、理由を説明してユーザー確認を取る
- 変更後は `pnpm exec tsc --noEmit` と、対象に応じたテストを確認する

---

## 実施記録

このフェーズでは、実施した整理だけを追記する。

### 2026-05-14 · Git機能削除と仕様書整合

- Git / GitHub関連のメインプロセス、IPC、preload API、レンダラーUI、設定、依存関係を削除した
- サイドバー、設定、コマンドパレットからGit関連導線を削除した
- Windowsパスをワークスペース相対パスとして誤って受け付けないように、パス検証を強化した
- macOS専用のウィンドウ装飾設定をmacOS分岐に閉じ込め、Mac / Windows両対応の安定性を上げた
- Finder前提の表示文言を、OSを限定しない「ファイルの場所」表現へ変更した
- `pnpm exec tsc --noEmit` と `pnpm test` で全件検証した。package系コマンドは配布検証扱いのため未実行
- 仕様書・設計書・技術文書・UI文書を、現行アプリを正としてGitなし、Mac / Windows対象の内容に更新した
- `package.json` の起動・テスト・ビルドスクリプトをMac / Windows両方の名前で揃え、成果物チェックを共通スクリプト化した
- `README.md` の開発、起動、テスト、Mac / Windowsビルド手順を現行アプリに合わせた
