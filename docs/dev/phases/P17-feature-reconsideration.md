# P17-feature-reconsideration.md

Relicの機能再考フェーズの正本。

このフェーズでは、既存機能・未実装機能・今後入れたい機能を、ユーザーへのヒアリングを起点に見直す。
AIが先に機能一覧、削除案、追加案、再設計案を作らない。

---

## フェーズの目的

Relicを「書く道具」として使い続けやすくするために、機能の必要性、重さ、導線、優先順位を再考する。

特に、以下を重視する。

- ユーザー本人が不要・重い・分かりにくいと感じている機能を起点にする
- 機能追加よりも、機能の整理、削減、統合、導線の見直しを同等以上に重視する
- 仕様変更・実装変更は、対象機能と方向性が合意されてから行う
- 機能判断は `../principles.md` と `../PLAN.md` に反しない範囲で行う

---

## 作業ルール

- まず、ユーザーが再考したい機能・違和感・削りたい機能・重いと感じている導線を確認する。
- AIが先に包括的な機能棚卸しや新チェックリストを作らない。
- 機能追加・削除・仕様変更・UI変更は、対象と方向性が合意されてから行う。
- 既存機能を削る場合は、影響範囲、代替手段、戻しやすさを確認する。
- 実装した場合は型チェック・自動テストで確認する。

## 完了条件

- ユーザーが再考したい主要機能について、残す・削る・後回し・再設計の判断ができている
- 判断結果が正本文書または該当仕様と矛盾していない
- 実装変更を行った場合は型チェック・自動テストが通っている
