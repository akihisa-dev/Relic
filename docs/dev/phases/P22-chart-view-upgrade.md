# P22-chart-view-upgrade.md

Relicのチャートビューアップグレードフェーズの正本。

このフェーズでは、既存のチャートビューを対象に、表示、操作、読み取りやすさ、既存画面との整合を改善する。
ただし、具体的な改善内容はユーザーが指定したものだけを対象にし、AIが先にチャート機能案やチェックリストを作らない。

---

## フェーズの目的

- チャートビューを、既存Markdownワークスペースの内容を読み取りやすくする画面として磨く
- タブ式メインエリア、左レール、ファイルサイドバー、既存チャートタブとの関係を崩さない
- 表示データの意味、操作感、視覚表現、状態管理を必要な範囲で整理する
- Mac / Windowsの両方で破綻しない実装・検証単位に分ける

---

## 作業方針

- 最初に、改善したいチャートビューの範囲と違和感を確認する
- 仕様、UI、保存形式、表示データの意味が未確定のまま実装に入らない
- 1回の変更は、見た目または操作を確認できる小さい単位に分ける
- 既存Markdownファイル、既存ワークスペース、既存設定を壊さない
- 変更後は `pnpm typecheck` と、対象に応じたテストを確認する
- 実アプリ確認が必要な見た目・操作感は、確認した範囲と未確認理由を記録する
- このフェーズ文書には日誌を書かない。日付順の作業履歴は `docs/journal/` に分ける

---

## 進捗

このフェーズでは、ユーザーが指定したチャートビュー改善ごとに、次回作業時の判断に必要な状態だけを記録する。
日付順の作業ログ、感想、細かな経緯はここに書かない。

必要な場合だけ、以下を同じ項目内にまとめる。

- 方向性
- 実施
- 確認
- 残り

### フェーズ開始

- 方向性: P22は「チャートモード」ではなく、既存の「チャートビュー」を対象にしたアップグレードフェーズとして進める
- 実施: `docs/dev/phases.md` の現在フェーズをP22へ変更し、P22正本を追加した
- 確認: コード実装には入らず、フェーズ開始文書だけを更新する
- 残り: 改善したいチャートビューの具体的な違和感、対象画面、最初の変更単位をユーザー確認してから実装する

### チャートバー編集

- 方向性: チャートビューのバーはファイルを開く入口ではなく、日付・年表範囲を直接編集する操作対象として扱う
- 実施: バークリックでファイルを開く挙動を外した。バー本体のドラッグで移動、左右端のドラッグで開始位置・終了位置を変更できるようにした
- 実施: チャートバー編集専用IPCを追加し、対応Markdownファイルのフロントマターを書き換えたうえでチャートデータを再読み込みする形にした。開発中にmain/preload側の反映が遅れても戻らないよう、既存のファイル読み書きIPCで保存するフォールバックも追加した
- 実施: チャート編集後、同じMarkdownファイルを開いているファイルタブがある場合は内容を再読み込みして同期する形にした
- 実施: `chronicle` バーの移動・長さ変更では `chronicle` を更新し、有効な `date` が同じファイルにある場合は同じ年差で `date` も更新する。`date` バーの移動・長さ変更では `date` を更新し、更新後の日付範囲の年から `chronicle` も更新する
- 確認: `pnpm typecheck` と `pnpm test -- App.test.tsx` を確認済み。テストコマンドはプロジェクト設定により全テストを実行し、310件通過した
- 残り: 実アプリでのドラッグ操作感、スナップ幅、長いバー・短いバーでの掴みやすさは未確認

### チャート操作メニュー

- 方向性: チャートビューの操作UIは、バー操作や閲覧に重なるホバーメニューではなく、チャート上端の控えめな固定ツールバーとして扱う
- 実施: ホバーで自動表示される浮遊メニューと、メニュー位置をドラッグ移動するためのハンドルを削除した
- 実施: チャート種別、検索、スケール操作をチャート上端の固定ツールバーへ移し、カード状の浮き・影・勝手に張り付く挙動をなくした
- 実施: 固定ツールバーがチャート領域中央に流れ込むレイアウト崩れを修正し、`ツールバー / チャート本体` の2行構成にした
- 確認: `pnpm typecheck` と `pnpm test -- App.test.tsx` を確認済み。テストコマンドはプロジェクト設定により全テストを実行し、310件通過した
- 残り: 実アプリでの見た目、ツールバーの高さ、狭い幅での収まりは未確認

### バー移動時の横軸安定化

- 方向性: チャートの横軸範囲は、表示中に最小・最大を持つバーを動かしても即座に縮めず、バー操作中の視界を安定させる
- 実施: 表示中のチャートでは一度確保した横軸範囲を保持し、データ更新後も既存範囲を縮めない形にした
- 実施: データ更新後に横軸が左側へ広がる場合は、スクロール位置を補正して見えている場所が跳ねにくいようにした
- 確認: `pnpm typecheck` と `pnpm test -- App.test.tsx` を確認済み。テストコマンドはプロジェクト設定により全テストを実行し、310件通過した
- 残り: 実アプリで、最小値・最大値を持つバーを移動したときの体感確認は未確認

### dateチャートの計画・実行表示

- 方向性: `date` チャートはタスク計画管理に使えるよう、1ファイルの計画日と実行日を別行で比較できる表示にする
- 実施: 正式プロパティ名を `plannedDate` と `actualDate` にした。旧 `date` は `plannedDate` がない場合の計画日として読み込む互換扱いにした
- 実施: `date` チャートでは `plannedDate` をブルー、`actualDate` をオレンジで表示し、同じファイルに両方がある場合は計画行・実行行の2行で表示するようにした
- 実施: 計画バー編集は `plannedDate`、実行バー編集は `actualDate` を更新するようにし、旧 `date` がある場合は計画バー編集時に旧 `date` も同期するようにした
- 実施: フロントマター固定プロパティの候補と予約名に `plannedDate` / `actualDate` を追加し、旧 `date` は既存ファイル編集の互換だけ残す形にした
- 実施: Markdown保存後にチャートデータを再読み込みする経路を追加し、フロントマターで `plannedDate` / `actualDate` を編集した後もチャートへ反映されるようにした
- 実施: 既存テストデータに混ざっていた `Tue May 12 2026 09:00:00 GMT+0900 (日本標準時)` 形式の日付文字列も、チャート読み込み時に `YYYY-MM-DD` として扱えるようにした
- 実施: main側のdateチャート返却に実行行が含まれない場合でも、チャートタブ表示時にrenderer側でMarkdownから `plannedDate` / `actualDate` を読み直して2行表示を補完するようにした
- 実施: `計画` / `実行` 表記はファイル名に混ぜず、dateチャート左列内の別セルとして表示するようにした
- 確認: `pnpm typecheck`、`pnpm test -- chronicle.test.ts SettingsSidebar.test.tsx App.test.tsx`、`pnpm test -- useAutoSave.test.ts App.test.tsx chronicle.test.ts`、`pnpm test -- chronicle.test.ts`、`pnpm test -- App.test.tsx chronicle.test.ts`、`git diff --check` を確認済み。最新の対象テストでは316件通過した
- 残り: 実アプリでの計画/実行2行表示、ブルー/オレンジの見え方、バー編集後の体感確認は未確認
