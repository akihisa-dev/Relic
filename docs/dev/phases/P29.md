# P29

Relicの再考フェーズの正本。

このフェーズでは、ユーザーが気になっている機能、画面、操作、情報設計、開発方針を起点に、Relicの扱いを再考する。

---

## 読み込みルール

AIはこのフェーズを前提にユーザーへ接する。

最初に読む:

- この文書

必要になったときだけ読む:

- プロダクト意図を確認する: `../../product/principles.md`, `../../product/PLAN.md`
- 既存仕様を確認する: 対応する `../../spec/*.md`
- UIへの影響を確認する: `../../ui/screens-macos.md`, `../../ui/navigation.md`, `../../ui/DESIGN.md`
- アーキテクチャ前提に触れる場合: `../../architecture/overview.md`, `../../architecture/decisions.md`
- 実装規約とテスト方針を確認する: `../conventions.md`, `../testing.md`

注意:

- AIが先に機能一覧、削除案、追加案、再設計案、チェックリストを作らない
- まずユーザーが再考したい対象、違和感、変えたい範囲を確認する
- 機能追加、削除、仕様変更、UI変更は、対象と方向性がユーザーと合意されてから行う
- 「便利そう」だけを理由に機能を増やさない
- 既存機能を削る場合は、影響範囲、代替手段、戻しやすさを確認してから進める
- 実装に入る前に、対象、問題、変更範囲、変えてはいけない挙動、検証方法を明示する

---

## フェーズの目的

- Relicに残す機能、弱める機能、見直す機能をユーザーの違和感から整理する
- 書くことに集中できるMarkdownワークスペースとして、画面と機能の役割を再確認する
- 仕様、UI、実装の変更が必要な場合は、合意した小さな単位で進める

---

## 修正管理

P29では、事前に固定した長い実施リストは置かない。
ユーザー指定、またはヒアリング後に合意した再考単位ごとに、この文書の「進捗」へ実施したことだけを簡潔に記録する。

完了条件:

- 合意した再考単位が完了している
- 仕様、UI、実装の変更があった場合は、対象に応じた文書、テスト、`pnpm typecheck`、`pnpm test`、`git diff --check` の必要範囲を確認している
- 実アプリ確認が必要な項目は、確認結果または未確認理由を記録している
- 次フェーズへ移る場合は、ユーザーが明示している

---

## 進捗

このフェーズでは、再考単位ごとに次回作業時の判断に必要な状態だけを記録する。

### フェーズ開始

- ユーザー指示により、P28グラフビュー安定化フェーズを終了し、P29再考フェーズを開始した。
- ユーザー指示により、ダッシュボードとグラフビュー削除前の状態へ戻した。依存を入れ直し、`pnpm typecheck`、`pnpm test`、`git diff --check` が通過した。

### グラフビュー削除

- ユーザー指示により、左レールのグラフ入口、グラフパネルタブ、rendererのグラフ描画・操作・store・worker・関連テスト、main/preload/sharedのグラフIPC/API、グラフ用locale、グラフ専用依存の `d3-force`、`pixi.js`、`@types/d3-force` を削除した。
- Chronicle用CSSだけを `chronicle.css` として残し、`pnpm typecheck`、`pnpm test`、`git diff --check` が通過した。全体テストは67ファイル、485件が通過した。

### ダッシュボード削除

- ユーザー指示により、左レールのダッシュボード入口、ダッシュボードパネルタブ、rendererのダッシュボード集計・表示・関連テスト、ダッシュボード用localeとCSSを削除した。
- ツールとフロントマターで共用していた見出しスタイルだけを `settings.css` へ移し、`pnpm typecheck`、`pnpm test`、`git diff --check` が通過した。全体テストは65ファイル、478件が通過した。

### アプリ目的の再定義

- ユーザー指示により、Relicの目的を「創作設定の作成、保管、閲覧のためのアプリ」として再定義し、`docs/product/project.md`、`docs/product/principles.md`、`README.md` に反映した。

### AIルール文書統合

- ユーザー指示により、AI向けルールを `AGENTS.md` に統合し、`AI.md` と `CLAUDE.md` を削除した。現行の入口・索引・運用文書は `AGENTS.md` 参照へ更新し、完了済みフェーズ本文、日誌、アーカイブの過去記録は書き換えなかった。
