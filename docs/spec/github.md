# spec/github.md

このファイルは、現行アプリにおける Git / GitHub 連携の扱いを定義する。

P19時点のアプリを正とし、RelicはGit機能を持たないローカルMarkdownワークスペースとして動作する。

---

## 現行仕様

- GitHub OAuth認証は提供しない
- Personal Access Token入力は提供しない
- GitHubリポジトリのクローンは提供しない
- 既存フォルダをGitリポジトリとして検出・初期化・接続しない
- コミット、プッシュ、プル、ブランチ切り替え、タグ管理は提供しない
- 自動同期、自動プル、自動プッシュは提供しない
- 差分確認、コミット履歴、コンフリクト解決UIは提供しない
- GitHub認証情報やGit接続情報をアプリ設定・ワークスペース設定に保存しない

---

## UI

- サイドバーにGitビューを表示しない
- 設定画面にGitHub連携、自動同期、Git機能トグルを表示しない
- コマンドパレットにGit関連コマンドを表示しない
- 起動時・ワークスペース作成時にGit状態の検査を行わない

---

## ファイル管理との関係

Relicは、ユーザーが選択したローカルフォルダ内のMarkdownファイルを通常のファイルとして読み書きする。iCloud Drive / OneDrive / Dropbox などの同期フォルダも、各サービスのデスクトップ同期機能によってOS上に見えているローカルフォルダとして扱う。

同期、履歴管理、リモート共有が必要な場合は、Relic外部のツールまたはOS・クラウドストレージ側の機能で扱う。
