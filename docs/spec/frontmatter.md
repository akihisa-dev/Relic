# spec/frontmatter.md

フロントマター機能の詳細仕様書。ファイルに付与する構造化メタデータの定義・フィールド仕様・入力UIの振る舞いを定義する。

---

## フロントマターとは

ファイル先頭の `---` で囲まれた YAML ブロック。本文に書くほどではないが、構造化情報として持っておきたいデータを記述する場所。

```yaml
---
tags: [小説, キャラクター]
date: 2026-05-03
status: draft
---
```

---

## 編集方式

- **ライブプレビューモード**：フォームUIとして表示する。YAML構文を手打ちせずに入力できる。各フィールドの型に合った入力部品を提供する
- **ソースモード**：生のYAMLとして直接編集できる。フォームとYAMLは常に同期する

ライブプレビューモードのフロントマター表示は、本文より控えめなメタ情報領域として扱う。本文の見出しや本文テキストより強く見えないようにし、入力UIはコンパクトにまとめる。

フィールド名・候補値・ピル表示が長い場合、フロントマター領域内では折り返さず省略表示を基本とする。全文確認や編集は、ホバー時のツールチップ、候補一覧、またはソースモードで行えるようにする。

---

## 候補定義ファイル

フロントマターのフォーム入力では、表記揺れを防ぐために候補から選ぶ入力方式を利用できる。

候補はワークスペース直下の `frontmatter.md` で管理する。`frontmatter.md` は初期状態では自動作成しない。ファイルが存在しない場合、フロントマターの各フィールドは自由入力として扱う。

設定画面には `frontmatter.md` を作成するボタンを用意する。ユーザーがボタンを押すと、ワークスペース直下に雛形ファイルを作成する。作成後は通常のMarkdownファイルとして開き、編集できる。

候補定義の例：

```md
# フロントマター候補

## status
- draft
- review
- published

## author
- 自分
```

`frontmatter.md` に定義された見出し名をフィールド名、配下の箇条書きを候補値として扱う。たとえば `## status` の下にある項目は、`status` フィールドの選択候補になる。

候補定義ファイルがある場合も、必要に応じて候補外の値を入力できる。候補外の値を入力した場合、ユーザーはその値を候補として `frontmatter.md` に追加できる。

## フィールドの種類

フロントマターのフィールドは3種類に分類する。

| 種類 | 内容 | 上限 |
|------|------|------|
| 組み込みフィールド | アプリの内部機能と連動する特別なフィールド | 2つ（固定） |
| 固定システムフィールド | アプリが型と専用UIを定義する標準フィールド | 5つ（固定） |
| ユーザー定義フィールド | ユーザーが名前・型・入力方式を設定できるフィールド | 最大13個 |

合計上限：**20フィールド**

---

## 組み込みフィールド（2つ）

アプリの機能と深く連動しており、削除・型変更はできない。

### `tags`

- **型**：文字列の配列
- **連動機能**：本文中の `#tag` 記法と同じタグインデックスに統合される。タグ検索・タグ一覧に反映される
- **入力UI**：タグ名を入力してEnterで追加。追加済みタグはピル表示。既存タグのオートコンプリートあり

### `aliases`

- **型**：文字列の配列
- **連動機能**：`[[]]` 内部リンクのリンク解決に使われる。ファイル名の代わりに別名でリンクできるようになる
- **入力UI**：テキスト入力でEnter追加。ピル表示

---

## 固定システムフィールド（5つ）

アプリが型を知っており専用UIを提供する。削除はできないが、使わなければYAMLに現れない。

| フィールド | 型 | 入力UI | 用途例 |
|---|---|---|---|
| `date` | 日付 | 日付ピッカー | 作成日・公開日など（用途はユーザーが決める） |
| `status` | 選択肢 | ドロップダウン | draft / in-review / published など（選択肢は `frontmatter.md` で定義） |
| `publish` | 真偽値 | トグル | 公開・非公開フラグ |
| `url` | URL | テキスト入力 | 参照元URL。クリックでブラウザが開く |
| `author` | 文字列の配列 | 候補つきピル入力 | 著者名。単独・複数どちらにも対応 |

---

## ユーザー定義フィールド（最大13個）

ユーザーが設定画面でフィールドを追加・編集・削除できる。

### 設定できる内容

| 設定項目 | 内容 |
|----------|------|
| フィールド名 | YAMLキー名（半角英数字・アンダースコア） |
| 型 | text / number / date / boolean / select / multi-select / url から選択 |
| 選択肢（select系のみ） | 選択肢の値をカンマ区切りで定義 |

### 入力UI

型に応じた入力部品をフォームUIで提供する。カスタムフィールドもYAML構文を手打ちせずに入力できる。

`select` / `multi-select` 型は、`frontmatter.md` の候補定義と連動できる。候補定義がない場合は自由入力として扱う。

---

## 自由フィールド

上記20フィールド以外に、ユーザーがソースモードで直接YAMLに書いたフィールド。アプリは型を認識しないため、フォームUIではテキストとして表示する。検索はフロントマター全文検索の範囲内で対応する。

---

## 検索・フィルターとの連動

- 組み込みフィールド・固定システムフィールド・ユーザー定義フィールドはすべて構造的な絞り込みの対象になる
- 自由フィールドは全文検索でのみヒットする
