# spec/file-management.md

カード・カードフォルダ管理機能の詳細仕様書。作成・削除・リネーム・移動・複製など、ファイルシステム操作に関わる機能の振る舞いを定義する。

---

## カード操作

### 作成
- カードサイドバーの新規カード操作、カードフォルダの右クリックメニュー、またはタブ0枚時の新規カード作成画面から作成できる
- カード名の制約（作成・リネーム共通）：
  - `/`（スラッシュ）を含む名前は入力を受け付けない（エラー表示）
  - 同じカードフォルダにすでに同名カードが存在する場合はエラーを表示し、別名への変更を促す
  - 拡張子なしで入力した場合は `.md` を自動で付与する
- サイドバー・タブ・その他UIでのカード名表示では `.md` 拡張子を隠す（実カードには `.md` が付いたまま）
- サイドバーでは `.md` カードを主役として表示する
- 添付画像はMarkdown本文から参照される補助ファイルとして扱い、画像ファイル単体をサイドバー上の主要な管理対象にはしない
- PDFなど、Markdown本文の添付画像ではない形式はカードブックに存在しても表示しない

### 削除
- 削除したカードはOSのゴミ箱に移動する（完全削除ではない）
- ゴミ箱から復元することで元に戻せる
- 削除前に確認ダイアログを表示する（「○○ をゴミ箱に移動しますか？」）
- カードツリーの右クリックメニュー、またはコマンドパレット（⌘⇧P）から削除できる
- カードツリーで複数選択している場合は、選択したカード / カードフォルダをまとめてゴミ箱へ移動できる

### リネーム
- カードツリー上でダブルクリック、または右クリックメニューの「名前を変更」から、その場でインライン編集できる
- 開いているカードはメインエリア上部操作バーのカード名からもリネームできる
- リネーム後、そのカードへの内部リンク（`[[...]]`）を全カードの**本文・プロパティ**で自動更新する（確認なし・即時）
- コードブロック内の `[[...]]` は更新対象外（コードの意図を壊さないため）

### 移動
- カードツリーの右クリックメニュー、またはメインエリア上部操作バーの移動操作から移動先カードフォルダを入力して移動できる
- カードツリーで複数選択している場合は、選択したカード / カードフォルダをまとめて移動できる
- 移動後、そのカードへの内部リンクを自動更新する（リネームと同様。本文・プロパティ対象、コードブロック除外）

### 複製
- カードツリーの右クリックメニュー、またはコマンドパレット（⌘⇧P）から複製できる
- 同じカードフォルダに「カード名 のコピー」として作成される

### その他の右クリック操作
- カードツリー、タブ、右パネルのリンクから、カードブック相対パスやMarkdownリンクをコピーできる
- カードツリー、タブ、右パネルのリンクから、OS上のカードの場所を表示できる
- 分割表示中は、カードツリーやリンクの右クリックメニューから別ペインで開ける

## カードフォルダ操作

### 作成
- カードサイドバーの「カードフォルダ作成」ボタン、またはカードフォルダの右クリックメニューから作成できる
- ネストしたカードフォルダ構造（サブカードフォルダ）をサポートする

### 削除
- カードフォルダを削除すると中のカードごとOSのゴミ箱に移動する
- 削除前に確認ダイアログを表示する
- 確認文言は「このカードフォルダをゴミ箱に移動しますか？カードフォルダ内のカードと添付カードも一緒に移動されます。」を基本とする
- カードフォルダツリーの右クリックメニューから削除できる

### リネーム
- カードと同様に、カードフォルダツリー上でダブルクリック、または右クリックメニューの「名前を変更」から、その場でインライン編集できる
- リネーム後、配下の `.md` カードへの内部リンクを全カードの**本文・プロパティ**で自動更新する（確認なし・即時）
- コードブロック内の `[[...]]` は更新対象外

### 移動
- 右クリックメニューから移動先カードフォルダを入力して移動できる
- 複数選択している場合は、選択したカード / カードフォルダをまとめて移動できる
- 移動後、配下の `.md` カードへの内部リンクを自動更新する（リネームと同様。本文・プロパティ対象、コードブロック除外）

## カードブック構造

- ユーザーがOS上の任意のローカルディレクトリをカードブックとして登録する（Obsidianと同じ方式）
- Relicは、カードブック作成時または既存カードフォルダ登録時にRelic専用カードフォルダを自動作成しない
- Markdown本文から参照される画像は本文の補助ファイルとして扱い、画像ファイル単体を主要な管理対象にはしない
- iCloud Drive / OneDrive / Dropbox などのクラウド同期ディレクトリも、OS上の通常のローカルディレクトリとして扱う
- Relicは各クラウドサービスのAPI連携を持たない。同期はOSまたはクラウドストレージアプリ側の機能として行われる
- 複数のカードブックを登録して切り替えながら使える
- 登録済みカードブックは左レール下部に表示し、クリックするとカードサイドバーとメインエリアの状態がそのカードブックへ切り替わる
- 左レール下部のカードブック項目は、右クリックメニューから名前変更または登録一覧からの削除ができる
- 登録一覧から削除しても、OS上のカードブック用ディレクトリ自体は削除しない
