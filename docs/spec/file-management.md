# spec/file-management.md

ファイル・フォルダ管理機能の詳細仕様書。作成・削除・リネーム・移動・複製など、ファイルシステム操作に関わる機能の振る舞いを定義する。

---

## ファイル操作

### 作成
- サイドバーの「新規ファイル」ボタン、または右クリックメニューから作成できる
- 作成直後はファイル名がインライン編集状態になる
- ファイル名の制約（作成・リネーム共通）：
  - `/`（スラッシュ）を含む名前は入力を受け付けない（インライン編集中にエラー表示）
  - 同じフォルダにすでに同名ファイルが存在する場合はエラーを表示し、別名への変更を促す
  - 拡張子なしで入力した場合は `.md` を自動で付与する
- サイドバー・タブ・その他UIでのファイル名表示では `.md` 拡張子を隠す（実ファイルには `.md` が付いたまま）
- サイドバーには `.md` ファイルのみを表示する（PDF・画像など他の形式はワークスペースに存在しても表示しない）

### 削除
- 削除したファイルはOSのゴミ箱に移動する（完全削除ではない）
- ゴミ箱から復元することで元に戻せる
- 削除前に確認ダイアログを表示する（「○○ をゴミ箱に移動しますか？」）

### リネーム
- サイドバー上でファイル名をダブルクリック、または右クリックメニューからリネームできる
- リネーム後、そのファイルへの内部リンク（`[[...]]`）を全ファイルの**本文・フロントマター**で自動更新する（確認なし・即時）
- コードブロック内の `[[...]]` は更新対象外（コードの意図を壊さないため）

### 移動
- サイドバー上でドラッグ&ドロップして別のフォルダに移動できる
- 移動後、そのファイルへの内部リンクを自動更新する（リネームと同様。本文・フロントマター対象、コードブロック除外）

### 複製
- 右クリックメニューから「複製」を選ぶと、同じフォルダに「ファイル名 のコピー」として作成される

## フォルダ操作

### 作成
- サイドバーの「新規フォルダ」ボタン、または右クリックメニューから作成できる
- ネストしたフォルダ構造（サブフォルダ）をサポートする

### 削除
- フォルダを削除すると中のファイルごとOSのゴミ箱に移動する

### リネーム
- ファイルと同様にサイドバー上でリネームできる

### 移動
- ドラッグ&ドロップで別のフォルダ内に移動できる

---

## ワークスペース構造

- ユーザーがMac上の任意のローカルフォルダをワークスペースとして登録する（Obsidianと同じ方式）
- iCloud Drive / OneDrive / Dropbox などのクラウド同期フォルダも、OS上の通常のローカルフォルダとして扱う
- Lorebooksは各クラウドサービスのAPI連携を持たない。同期はOSまたはクラウドストレージアプリ側の機能として行われる
- 複数のワークスペースを登録して切り替えながら使える
- サイドバー上部のドロップダウンでワークスペースを切り替える
  - 現在のワークスペース名 ▼ をクリックするとリストが展開する
  - リストからワークスペースを選ぶと、サイドバーのファイルツリーがそのワークスペースに切り替わる
  - リスト末尾に「＋ ワークスペースを追加」を設置し、既存フォルダを選択して登録できる
