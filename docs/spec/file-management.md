# spec/file-management.md

ファイル・フォルダ管理機能の詳細仕様書。作成・削除・リネーム・移動・複製など、ファイルシステム操作に関わる機能の振る舞いを定義する。

---

## ファイル操作

### 作成
- サイドバーの入力フォームにファイル名を入力して作成できる
- `templates/` フォルダ内に `.md` テンプレートがある場合、新規ファイル作成時にテンプレートを選択できる
- テンプレートを選ぶと、その本文を初期内容として新規ファイルを作成する
- ファイル名の制約（作成・リネーム共通）：
  - `/`（スラッシュ）を含む名前は入力を受け付けない（エラー表示）
  - 同じフォルダにすでに同名ファイルが存在する場合はエラーを表示し、別名への変更を促す
  - 拡張子なしで入力した場合は `.md` を自動で付与する
- サイドバー・タブ・その他UIでのファイル名表示では `.md` 拡張子を隠す（実ファイルには `.md` が付いたまま）
- サイドバーでは `.md` ファイルを主役として表示する
- 添付画像はMarkdown本文から参照される補助ファイルとして扱い、画像ファイル単体をサイドバー上の主要な管理対象にはしない
- PDFなど、Markdown本文の添付画像ではない形式はワークスペースに存在しても表示しない

### 削除
- 削除したファイルはOSのゴミ箱に移動する（完全削除ではない）
- ゴミ箱から復元することで元に戻せる
- 削除前に確認ダイアログを表示する（「○○ をゴミ箱に移動しますか？」）
- ファイルツリーの右クリックメニュー、またはコマンドパレット（⌘⇧P）から削除できる

### リネーム
- ファイルツリー上でダブルクリック、または右クリックメニューの「名前を変更」から、その場でインライン編集できる
- 開いているファイルはサイドバーのリネームバー、またはコマンドパレット（⌘⇧P）からもリネームできる
- リネーム後、そのファイルへの内部リンク（`[[...]]`）を全ファイルの**本文・フロントマター**で自動更新する（確認なし・即時）
- コードブロック内の `[[...]]` は更新対象外（コードの意図を壊さないため）

### 移動
- サイドバー上でドラッグ&ドロップして別のフォルダに移動できる
- 移動後、そのファイルへの内部リンクを自動更新する（リネームと同様。本文・フロントマター対象、コードブロック除外）

### 複製
- ファイルツリーの右クリックメニュー、またはコマンドパレット（⌘⇧P）から複製できる
- 同じフォルダに「ファイル名 のコピー」として作成される

## フォルダ操作

### 作成
- サイドバーの「新規フォルダ」ボタンから作成できる
- ネストしたフォルダ構造（サブフォルダ）をサポートする

### 削除
- フォルダを削除すると中のファイルごとOSのゴミ箱に移動する
- 削除前に確認ダイアログを表示する
- 確認文言は「このフォルダをゴミ箱に移動しますか？フォルダ内のファイルと添付ファイルも一緒に移動されます。」を基本とする
- フォルダツリーの右クリックメニューから削除できる

### リネーム
- ファイルと同様に、フォルダツリー上でダブルクリック、または右クリックメニューの「名前を変更」から、その場でインライン編集できる
- リネーム後、配下の `.md` ファイルへの内部リンクを全ファイルの**本文・フロントマター**で自動更新する（確認なし・即時）
- コードブロック内の `[[...]]` は更新対象外

### 移動
- ドラッグ&ドロップで別のフォルダ内に移動できる
- 移動後、配下の `.md` ファイルへの内部リンクを自動更新する（リネームと同様。本文・フロントマター対象、コードブロック除外）

## ワークスペース構造

- ユーザーがMac上の任意のローカルフォルダをワークスペースとして登録する（Obsidianと同じ方式）
- Relicが機能上必要とするフォルダは、ワークスペース作成時または既存フォルダ登録時に自動作成する
- 添付画像フォルダは `attachments/` 固定とし、画像貼り付け時ではなくワークスペース準備時に作成する
- テンプレートフォルダは `templates/` 固定とし、ワークスペース準備時に作成する
- `templates/` フォルダ直下の `.md` ファイルを新規ファイル用テンプレートとして扱う
- すでに同名フォルダが存在する場合は、そのフォルダを再利用する
- ユーザーにフォルダ名や作成有無は選ばせない
- Markdown本文に添付した画像は、ワークスペース内の `attachments/` に保存する
- iCloud Drive / OneDrive / Dropbox などのクラウド同期フォルダも、OS上の通常のローカルフォルダとして扱う
- Relicは各クラウドサービスのAPI連携を持たない。同期はOSまたはクラウドストレージアプリ側の機能として行われる
- 複数のワークスペースを登録して切り替えながら使える
- サイドバー上部のワークスペース名をクリックすると、登録済みワークスペースのドロップダウンを表示する
  - 現在のワークスペース名を常時表示する
  - 登録済みワークスペースはドロップダウン内に表示する
  - クリックするとサイドバーのファイルツリーがそのワークスペースに切り替わる
