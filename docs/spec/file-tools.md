# spec/file-tools.md

ファイル加工ツールの仕様書。初期リリースに含める。既存ファイルを変更せず、結果は必ず新規ファイルとして出力する。

**原則：既存ファイルは読み取りのみ。操作の結果はすべて新規ファイルとして作成する。**

---

## 条件指定マージ

指定した条件に一致するファイルを1つのファイルに結合して出力する。

- 絞り込み条件：タグ・フロントマターのフィールド値・フォルダ
- 結合順序：ファイル名順・更新日時順・作成日時順から選択
- 各ファイルの区切りとして、ファイル名を見出しとして挿入するかどうかを選択できる
- 出力先フォルダとファイル名を指定して新規ファイルとして保存する
- 出力先に同名ファイルが存在する場合は上書きせず、別名で保存する

---

## 見出しで分割

1つのファイルを指定した見出しレベルで複数ファイルに切り出して出力する。

- 分割の基準とする見出しレベルを指定（H1 / H2 / H3）
- 各見出しのテキストをもとにファイル名を生成する
- ファイル名に使えない文字は、全角・似た文字に置き換える
- 元のファイルは変更しない
- 出力先フォルダを指定して新規ファイルとして保存する

---

## タイトル一覧

条件で絞り込んだファイルのタイトルをリスト化して新規ファイルに出力する。

- 絞り込み条件：タグ・フロントマターのフィールド値・フォルダ（指定なしの場合はワークスペース全体）
- 出力形式：箇条書き（内部リンク付き）
- 例：`- [[ファイル名]]`
- 出力先フォルダとファイル名を指定して新規ファイルとして保存する

---

## 目次生成

フォルダ内のファイルを内部リンク一覧として出力する。フォルダの索引ファイルとして使える。

- 対象フォルダを指定する
- サブフォルダを含めるかどうかを選択できる
- 出力形式：フォルダ階層を反映したネストリスト（内部リンク付き）
- 出力先フォルダとファイル名を指定して新規ファイルとして保存する
- 既存の目次ファイルは自動更新しない
- 出力先に同名ファイルが存在する場合は、別名で新規作成する

---

## 備考

- 上記の機能は初期リリースに含める
- リリース後のアップデートでは、既存機能の使い勝手改善や追加の加工ツールを検討する
- 新しい加工ツールを追加する際も「既存ファイルを変更しない」原則を守ること
