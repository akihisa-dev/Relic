# spec/file-tools.md

カード加工ツールの仕様書。既存カードを変更せず、結果は必ず新規カードとして出力する。

**原則：既存カードは読み取りのみ。操作の結果はすべて新規カードとして作成する。**

---

## 条件指定マージ

指定した条件に一致するカードを1つのカードに結合して出力する。

- 絞り込み条件：タグ・プロパティのフィールド値・カードフォルダ・すべて
- プロパティ絞り込みでは、フィールド名と値を指定する（例：`status` が `draft` のカードだけ）
- 結合順序：カード名順・更新日時順・作成日時順から選択
- 各カードの区切りとして、カード名を見出しとして挿入するかどうかを選択できる
- 出力先カードフォルダとカード名を指定して新規カードとして保存する
- 出力先に同名カードが存在する場合は上書きせず、別名で保存する

---

## 見出しで分割

1つのカードを指定した見出しレベルで複数カードに切り出して出力する。

- 分割の基準とする見出しレベルを指定（H1 / H2 / H3）
- 各見出しのテキストをもとにカード名を生成する
- カード名に使えない文字は `_` に置き換える
- 元のカードは変更しない
- 出力先カードフォルダを指定して新規カードとして保存する

---

## タイトル一覧

条件で絞り込んだカードのタイトルをリスト化して新規カードに出力する。

- 絞り込み条件：カードフォルダ（指定なしの場合はカードブック全体）
- 並び順：カード名順・更新日時順から選択
- 出力形式：箇条書き（内部リンク付き）
- 例：`- [[カード名]]`
- 出力先カードフォルダとカード名を指定して新規カードとして保存する

---

## 目次生成

カードフォルダ内のカードを内部リンク一覧として出力する。カードフォルダの索引カードとして使える。

- 対象カードフォルダを指定する
- サブカードフォルダを含めるかどうかを選択できる
- 出力形式：カードフォルダ階層を反映したネストリスト（内部リンク付き）
- 出力先カードフォルダとカード名を指定して新規カードとして保存する
- 既存の目次カードは自動更新しない
- 出力先に同名カードが存在する場合は、別名で新規作成する

---

## 備考

- 対象機能は上記の4種類とする
- 追加の加工ツールを検討する場合も、この仕様とは別作業として扱う
- 新しい加工ツールを追加する際も「既存カードを変更しない」原則を守ること
