## 確定事項

- **アプリ名**: Relic
- **プラットフォーム**: macOS / Windows（iOS は、デスクトップ版を自分で継続利用し、外出先・モバイルで使いたい場面が明確になったら別アプリとして検討）
- **目的**: 創作設定の作成、保管、閲覧のためのアプリにする
- **フォーマット**: Markdown中心。Markdown内の添付画像はライブプレビューで表示するが、画像管理アプリにはしない
- **カード**: MarkdownファイルをRelic上ではカードと呼び、創作設定の最小単位として扱う
- **カードブック**: ワークスペースをRelic上ではカードブックと呼び、任意のローカルフォルダを登録できる
- **カードフォルダ**: フォルダをRelic上ではカードフォルダと呼び、カードを階層整理する
- **プロパティ**: Markdown内のプロパティを、カードの分類・検索・関連付け・表示のキーとして扱う
- **テキスト機能**: 創作設定カードの作成、保管、閲覧に必要なMarkdown機能を内蔵する
- **拡張性**: プラグインなし。ユーザーが機能を追加・拡張する仕組みは将来も原則持たない
- **機能のオンオフ**: カード加工ツール、プロパティ設定、右パネルを設定でトグルできる
- **同期・履歴管理**: 現行アプリには持たない。必要な場合はOS・クラウド同期フォルダ・Relic外部のツールで扱う
- **クラウド同期フォルダ対応**: iCloud Drive / OneDrive / Dropbox など、OS上でローカルフォルダとして見える場所をカードブックにできる
- **ライブプレビュー**: アプリの根幹
- **Chronicle**: 日付チャートは扱わず、`chronicle` プロパティから生成するChronicleに絞る
- **配布**: まず自分用ビルド。第三者に配る段階でコード署名と自動アップデートを検討し、一般ユーザー向けに継続提供する価値が見えたら配布方法を判断する
- **料金モデル**: 現時点では設計対象外。まず自分用に作って使い、良ければ収益化を検討する
- **開発方法**: Claude Code / Codex を使ったAI駆動開発

## カード機能

| 機能 | 説明 |
|------|------|
| カードブック管理 | 任意のローカルフォルダをカードブックとして登録 |
| カード / カードフォルダ管理 | ネストしたカードフォルダ構造でカードを整理 |
| Markdownエディタ | ライブプレビュー + ソースモード切替 |
| 内部リンク `[[...]]` | カード間をWikiリンクで繋ぐ |
| バックリンク | 自分を参照しているカード一覧を表示 |
| アウトゴーイングリンク | 自分が参照しているカード一覧を表示 |
| タグ | プロパティ `tags:` で横断的に整理・検索 |
| 全文検索 | カードブック内のカードをリアルタイム検索 |
| プロパティ設定 | 固定プロパティ確認とカスタムプロパティの入力能力管理 |
| アウトラインパネル | 見出し構造をサイドパネルで表示 |
| 右パネル | アウトライン / リンク一覧をカードタブ横に表示 |
| コマンドパレット | キーボードでコマンドをすばやく実行 |
| クイックスイッチャー | カード名で素早くカードを開く |
| プロパティ | カード冒頭に分類・検索・関連付け・表示のキーを記述 |
| Chronicle | `chronicle` プロパティから時系列表示 |
| ワードカウント | 文字数・単語数の表示 |
| テーマ | ライト / ダーク / システム追従 |
| 添付画像 | Markdown本文に添付した画像をライブプレビューで表示 |
| カード加工ツール | マージ・分割・タイトル一覧・目次生成 |
| 設定 | テーマ、言語、エディタ表示、機能トグルを管理 |
