# features/markdown.md

マークダウン記法サポートの詳細仕様書。アプリがレンダリング・サポートするマークダウン記法の範囲と挙動を定義する。

**方針：Markdownプレビューは `marked` で扱える標準記法を基準にし、Relicで明示的に必要なObsidian互換記法を追加する。**
すべてのMarkdown方言を網羅するのではなく、Markdown内にテキストとして書ける情報を、Markdownファイルのまま編集・閲覧・検索・可視化・出力へ広げるために必要な記法を優先する。

**画像方針：Markdown内の画像記法は編集・表示対象にし、ワークスペース内の相対パス画像だけ実画像として表示する。** 外部URL画像とワークスペース外参照は画像プレースホルダーとして扱い、Relicは画像管理アプリにはしない。

**図表方針：図表はMarkdown本文中のコードブロックを正本にする。** `mermaid` と `d2` のコードブロックはライブプレビューと通常プレビューでSVG図として表示するが、保存される内容はコードブロックのソース文字列のままとする。

## 標準記法（CommonMark）

| 記法 | 書き方 | 説明 |
|------|--------|------|
| 見出し | `# H1` 〜 `###### H6` | 6段階の見出し |
| 太字 | `**テキスト**` / `__テキスト__` | 太字表示 |
| 斜体 | `*テキスト*` / `_テキスト_` | 斜体表示 |
| 打ち消し線 | `~~テキスト~~` | 打ち消し線 |
| インラインコード | `` `code` `` | インラインのコード表示 |
| コードブロック | ` ```言語名 ` | シンタックスハイライト付きコードブロック |
| 引用ブロック | `> テキスト` | 引用 |
| 箇条書き | `- テキスト` | 順序なしリスト |
| 番号付きリスト | `1. テキスト` | 順序ありリスト |
| チェックボックス | `- [ ] タスク` / `- [x] 完了` | クリックでオン/オフ切り替え可能 |
| 水平線 | `---` | 区切り線 |
| 表（テーブル） | `\| col \|` 記法 | テーブル表示 |
| リンク | `[テキスト](リンク先)` | Markdown標準リンク。相対Markdownパスはワークスペース内ファイルとして開く |
| 添付画像 | `![alt](images/image.png)` | ワークスペース内の相対パス画像だけ実画像として表示 |
| 脚注 | `[^1]` / `[^1]: 注釈` | ページ末尾に注釈を表示 |

Markdownテーブルは保存形式としては通常のテーブル記法のまま扱う。ライブプレビューではセル編集、行列操作、TSV貼り付け・コピーを補助するが、書き戻し後もMarkdown本文はテーブル記法として保存する。
ライブプレビューで表として扱う対象は、見出し行の次に `---`、`:---`、`---:`、`:---:` 形式の区切り行を持つMarkdownテーブルである。
表を書き戻すときは、各行の列数を表内の最大列数に揃え、足りないセルは空欄にし、区切り行は `---` の列に正規化する。
Markdownテーブルのセル値が `#111111` や `` `#111111` `` のようなHEXカラーだけで構成される場合は、通常プレビューとライブプレビューで色見本を併せて表示する。保存されるMarkdown本文は元の色コード文字列のままとする。

Markdownチェックボックスは保存形式としては `- [ ]` / `- [x]` のまま扱う。ライブプレビューでは表示されたチェックボックスをクリックでき、クリック時は該当行のチェック文字だけを書き換える。

Markdown標準リンクのうち、相対Markdownパスはリンク元ファイルを基準にワークスペース内ファイルとして解決する。外部URL、プロトコル付きURL、ページ内アンカーだけのリンクは、ワークスペース内ファイルとしては扱わない。詳細は [links.md](links.md) を参照する。

ライブプレビューのインライン装飾では、インラインコード、Markdown標準リンク、内部リンク、太字、打ち消し線、ハイライト、下線、斜体を行単位で検出する。装飾範囲が重なる場合は先に検出された外側の範囲を優先し、たとえば `[**太字**](path)` はリンクとして表示する。内部リンクに `|表示テキスト` がある場合は表示テキストだけを見せ、リンク先や区切り記号はカーソルまたは選択範囲が触れたときだけ編集対象として見せる。

## 拡張記法（Obsidian互換）

| 記法 | 書き方 | 説明 |
|------|--------|------|
| ハイライト | `==テキスト==` | 黄色のマーカー表示 |
| 下線 | `<u>テキスト</u>` | 下線表示。HTMLはDOMPurifyで安全化し、Relicが明示的に保証するHTML由来の装飾は下線のみとする。 |
| 数式（インライン） | `$式$` | LaTeX記法でインライン数式をレンダリング |
| 数式（ブロック） | `$$式$$` | LaTeX記法でブロック数式をレンダリング |
| 内部リンク | `[[ファイル名]]` | アプリ内ノートへのリンク（詳細は links.md 参照） |
| 内部リンク（エイリアス） | `[[ファイル名\|表示テキスト]]` | リンクの表示テキストをファイル名と別にする |
| 内部リンク（見出し） | `[[ファイル名#見出し]]` | ファイル内の特定の見出しにリンク |
| ブロック参照 | `[[ファイル名^ブロックID]]` | ファイル内の特定の段落にリンク |
| ファイル埋め込み | `![[ファイル名]]` | 別ファイルの内容をその場に展開表示 |
| フロントマター | `---` で囲む YAML | ファイル冒頭のメタデータ（詳細は frontmatter.md 参照） |

## コードブロックのシンタックスハイライト

- 言語名を指定するとその言語のルールで色付けする
- 対応言語：主要な言語を網羅する（Swift, Python, JavaScript, TypeScript, Bash, JSON, YAML, HTML, CSS など）
- 言語未指定の場合はプレーンテキストとして表示
- ライブプレビューの通常コードブロックには、言語名ラベルと本文だけをコピーする操作を表示する
- コードブロックのコピー操作では、コードフェンスや言語名を含めず、コード本文だけをクリップボードへ書き込む
- コピー後は、成功または失敗をボタン上の一時的な文言で表示する
- Mermaid / D2 の図表コードブロックには、通常コードブロック用のヘッダーやコピー操作は表示しない

## 図表コードブロック

`mermaid` と `d2` を指定したコードブロックは、Markdownソースを正本として保持したまま図として表示する。

| 言語名 | 表示 | 実装 |
|--------|------|------|
| `mermaid` | Mermaid図をSVGとして表示 | `mermaid` を遅延読み込みし、`securityLevel: "strict"`、`htmlLabels: false` で初期化する |
| `d2` | D2図をSVGとして表示 | `@terrastruct/d2` を遅延読み込みし、D2描画だけ直列キューで実行する |

- 生成されたSVGは表示前にDOMPurifyでサニタイズする
- 図表コードブロックの言語名は、前後空白を除き、最初の空白区切りトークンを小文字化して判定する。そのため `Mermaid` や `mermaid something` もMermaid図として扱う
- 図表コードブロックのソースは、プレビューHTML内では `data-diagram-source` にURIエンコードして保持する
- 図の表示領域では、ホイールズーム、ドラッグ移動、キーボード操作、全体表示ができる
- ライブプレビューでは「ソースを編集」ボタンで対象コードブロックのソース編集に戻る
- ライブプレビューでは、描画済みの図をSVGとして保存できる
- ライブプレビューでは、描画済みの図のSVG文字列をクリップボードへコピーできる
- 図の描画に失敗した場合は、図表ソースとエラー詳細をコピーできるエラー表示を出す
- 図表レンダラーが文字列を返しても、サニタイズ後にSVGとして表示できる内容が残らない場合は描画失敗として扱う
- `data-diagram-source` は通常プレビューHTML内で図表ソースを保持するための内部属性であり、保存形式には含めない
- SVG保存・SVGコピーは描画済みSVGだけを対象にし、パン/ズーム用の表示ラッパーや一時的な変形状態は含めない
- SVG保存・SVGコピー時は、SVGに `xmlns` がなければ付与し、空のSVGは対象にしない
- D2ブラウザ版はWebAssembly workerを使うため、現行CSPでは `worker-src blob:`、`script-src 'wasm-unsafe-eval'` を許可している。`'unsafe-eval'` は許可しない

## 添付画像

- Markdown本文中の画像記法 `![alt](path)` をサポートする
- 対象はMarkdown記法そのものとし、ワークスペース内の相対パス画像だけ実画像として表示する
- Relicは画像保存用の専用フォルダを自動作成しない
- OS上の対応画像ファイルをエディタ本文へドラッグ&ドロップした場合は、画像ファイルをワークスペース内へ追加し、Markdown画像記法を挿入する
- ライブプレビューでは、ワークスペース内の相対パス画像を実画像として表示し、表示できない画像記法は `alt` またはパスを使った画像プレースホルダーとして表示する
- カーソルまたは選択範囲が画像記法に触れているときは、Markdown記法として編集できる
- 外部URL画像もフェッチせず、画像プレースホルダーとして扱う
- `..` や絶対パスなど、ワークスペース外を参照しうる画像は読み込まず、画像プレースホルダーとして扱う
- 対応画像ファイルはファイルサイドバーに添付ファイルとして表示し、クリックすると画像表示タブで開ける
- 画像の編集・タグ付け・アルバム管理・画像検索は行わない
- 検索とMarkdownリンク候補の主役は `.md` ファイルのままとする

## ファイル埋め込み

- `![[ファイル名]]` で、現在のワークスペース内の別Markdownファイルをその場に表示する
- 埋め込み先はMarkdownファイルだけを対象にし、拡張子がない場合は `.md` を補う
- 埋め込み先の `#見出し` と `^ブロックID` は、読み込み対象ファイルの判定では取り除く
- バックスラッシュは `/` に正規化する
- 空の埋め込み先、絶対パス、URL、`..` でワークスペース外へ出るパス、`.md` 以外の拡張子は埋め込み対象にしない
- 埋め込み対象にできない `![[...]]` はファイルを読み込まず、元の記法をエラー表示として表示する
- 同じ埋め込み先が複数回書かれている場合は、読み込み対象としては重複を除く
- 埋め込み先の読み込み中は、読み込み中表示を出す
- 埋め込み先を読み込めない場合は、エラー表示を出す
- 埋め込み表示は一段階までとする。埋め込み先の本文にさらに `![[...]]` があっても展開しない
- 20,000文字を超える埋め込み先ファイルは全文表示せず、大きすぎることを示す通知を表示する
- 埋め込み内のリンクをクリックした場合は、通常の内部リンクと同じようにリンク先を開く
- 埋め込み内では直接編集しない。編集する場合は元ファイルを開く

## プレビューHTMLの安全化

- プレビューHTMLはDOMPurifyで安全化する
- Relicのプレビュー機能で追加保持する属性は、`checked`、`class`、`data-diagram-language`、`data-diagram-source`、`data-target`、`id` とする
- Markdown見出しには、見出しテキストを小文字化し、空白を `-` に置き換え、URIエンコードした `id` を付ける
- HTMLや数式のレンダリングでエラーが起きても、アプリ全体を止めず、該当箇所だけエラー表示または元テキストに近い表示へ倒す

## 実装ライブラリ

- **編集・ライブプレビューの中心**: CodeMirror 6
- **Markdown解析**: marked
- **脚注拡張**: marked-footnote
- **HTML安全化**: DOMPurify
- **シンタックスハイライト**: highlight.js
- **数式レンダリング**: KaTeX（軽量・高速。一般的な数式記号をカバー）
- **図表レンダリング**: Mermaid、D2

## LaTeX 対応範囲

KaTeX がサポートする標準的なコマンドに準拠する。複雑な LaTeX マクロや特殊パッケージは対象外。
