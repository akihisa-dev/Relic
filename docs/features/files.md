# features/files.md

ファイル・フォルダ管理機能の詳細仕様書。作成・削除・リネーム・移動・複製など、ファイルシステム操作に関わる機能の振る舞いを定義する。

---

## ファイル操作

### 作成
- ファイルサイドバーの新規ファイル操作、フォルダの右クリックメニュー、またはタブ0枚時の新規ファイル作成画面から作成できる
- 新規ファイル作成画面や新規ファイルボタンから名前なしで作成する場合、アプリが既定名で空のMarkdownファイルを作成する
- ファイルサイドバーから新規ファイルを作成した場合は、作成後にそのMarkdownファイルを現在フォーカス中のペインで開く
- ファイルサイドバーから新規ファイルを作成するときは、左レールからタブ列へ短いオープン表示を出す
- ファイルサイドバーから新規フォルダを作成するときは、操作ボタンからファイルツリーへ短い作成表示を出す
- ファイルサイドバー上部のファイル作成・フォルダ作成操作は、処理中だけ該当ボタンを無効にする
- ファイル名の制約（作成・リネーム共通）：
  - `/`（スラッシュ）を含む名前は入力を受け付けない（エラー表示）
  - 同じフォルダにすでに同名ファイルが存在する場合はエラーを表示し、別名への変更を促す
  - 拡張子なしで入力した場合は `.md` を自動で付与する
- サイドバー・タブ・その他UIでのファイル名表示では `.md` 拡張子を隠す（実ファイルには `.md` が付いたまま）
- サイドバーでは `.md` ファイルを主役として表示する
- 添付画像はMarkdown本文から参照される補助ファイルとして扱い、画像ファイル単体をサイドバー上の主要な管理対象にはしない
- PDFなど、Markdown本文の添付画像ではない形式はワークスペースに存在しても表示しない
- 隠しファイル・隠しフォルダ（名前が `.` で始まる項目）はファイルツリーに表示しない
- ファイル作成と複製では、同名ファイルが既にある場合に上書きしない
- Markdownファイルの新規作成、複製、保存、検索置換、チャート編集、ファイル加工ツールの出力、PDF/SVG保存は、一時ファイルへ書き込んでから最終パスへ置き換える安全書き込みを行う

### 削除
- 削除したファイルはOSのゴミ箱に移動する（完全削除ではない）
- ゴミ箱から復元することで元に戻せる
- 削除前に確認ダイアログを表示する（「○○ をゴミ箱に移動しますか？」）
- ファイルツリーの右クリックメニュー、またはコマンドパレットから削除できる
- ファイルツリーで複数選択している場合は、選択したファイル / フォルダをまとめてゴミ箱へ移動できる

### リネーム
- ファイルツリー上でダブルクリック、または右クリックメニューの「名前を変更」から、その場でインライン編集できる
- 開いているファイルは、ファイルタブ本文上部の編集可能なファイルタイトルからもリネームできる
- リネーム後、そのファイルへの内部リンク（`[[...]]`）を全ファイルの**本文・フロントマター**で自動更新する
- 更新対象のファイル数またはリンク数が多い場合は、実行前に確認ダイアログを表示する
- コードブロック内の `[[...]]` は更新対象外（コードの意図を壊さないため）
- リネーム前に対象タブに未保存変更がある場合は、先に保存を試みる。保存に失敗した場合はリネームしない

### 移動
- ファイルツリーの右クリックメニューから移動先フォルダを入力して移動できる
- ファイルまたはフォルダをファイルツリー上でドラッグ&ドロップして移動できる
- ファイル行へドロップした場合は、そのファイルと同じ親フォルダへ移動する
- フォルダ行へドロップした場合は、そのフォルダの中へ移動する
- 空フォルダの内容エリア、展開済みフォルダ内の余白、同じ親フォルダ、子孫フォルダへのドロップでは移動しない
- ファイルツリーで複数選択している場合は、選択したファイル / フォルダをまとめて移動できる
- 移動後、そのファイルへの内部リンクを自動更新する（リネームと同様。本文・フロントマター対象、コードブロック除外）
- 更新対象のファイル数またはリンク数が多い場合は、実行前に確認ダイアログを表示する

### 複製
- ファイルツリーの右クリックメニュー、またはコマンドパレットから複製できる
- 同じフォルダに「ファイル名 のコピー」として作成される
- コピー名候補が既に多く埋まっている場合、無制限に探索せずエラーとして停止する

### その他の右クリック操作
- ファイルツリー、タブ、右パネルのリンクから、ワークスペース相対パスやMarkdownリンクをコピーできる
- ファイルツリー、タブ、右パネルのリンクから、OS上のファイルの場所を表示できる
- 分割表示中は、ファイルツリーやリンクの右クリックメニューから別ペインで開ける
- ファイルツリーのフォルダ右クリックメニューから、そのフォルダ配下だけの展開・折りたたみ、または全フォルダの展開・折りたたみができる
- ファイルツリーのファイル右クリックメニューには、複製を表示する。フォルダには複製を表示しない
- ファイルツリーのMarkdownリンクコピーは、拡張子 `.md` を外した `[[path/to/file]]` 形式をコピーする
- ファイルツリーの右クリックメニューは画面内に収まる位置へ補正して表示し、メニュー外クリックまたはEscapeで閉じる

## フォルダ操作

### 作成
- ファイルサイドバーの「フォルダ作成」ボタン、またはフォルダの右クリックメニューから作成できる
- ネストしたフォルダ構造（サブフォルダ）をサポートする

### 削除
- フォルダを削除すると中のファイルごとOSのゴミ箱に移動する
- 削除前に確認ダイアログを表示する
- 確認文言は「このフォルダをゴミ箱に移動しますか？フォルダ内のファイルと添付ファイルも一緒に移動されます。」を基本とする
- フォルダツリーの右クリックメニューから削除できる

### リネーム
- ファイルと同様に、フォルダツリー上でダブルクリック、または右クリックメニューの「名前を変更」から、その場でインライン編集できる
- リネーム後、配下の `.md` ファイルへの内部リンクを全ファイルの**本文・フロントマター**で自動更新する
- 更新対象のファイル数またはリンク数が多い場合は、実行前に確認ダイアログを表示する
- コードブロック内の `[[...]]` は更新対象外

### 移動
- 右クリックメニューから移動先フォルダを入力して移動できる
- 複数選択している場合は、選択したファイル / フォルダをまとめて移動できる
- 移動後、配下の `.md` ファイルへの内部リンクを自動更新する（リネームと同様。本文・フロントマター対象、コードブロック除外）
- 更新対象のファイル数またはリンク数が多い場合は、実行前に確認ダイアログを表示する

## ワークスペース構造

- ユーザーがOS上の任意のローカルフォルダをワークスペースとして登録する（Obsidianと同じ方式）
- Relicは、ワークスペース作成時または既存フォルダ登録時にRelic専用フォルダを自動作成しない
- Markdown本文から参照される画像は通常のファイル参照として扱い、画像ファイル単体を主要な管理対象にはしない
- iCloud Drive / OneDrive / Dropbox などのクラウド同期フォルダも、OS上の通常のローカルフォルダとして扱う
- Relicは各クラウドサービスのAPI連携を持たない。同期はOSまたはクラウドストレージアプリ側の機能として行われる
- 複数のワークスペースを登録して切り替えながら使える
- 登録済みワークスペースは左レール下部に表示し、クリックするとファイルサイドバーとメインエリアの状態がそのワークスペースへ切り替わる
- 登録済みワークスペースが0件でもアクティブワークスペースがある場合は、そのワークスペースを左レール下部に表示する
- 既存ワークスペースを開く操作、新規ワークスペース作成、登録済みワークスペース切り替えでは、前ワークスペースの開きタブを閉じる
- ワークスペース未選択時は、ファイルサイドバーに既存フォルダを開く主要操作と、新規ワークスペース作成の副次操作を表示する
- ワークスペース選択中も、ファイルサイドバー下部から既存フォルダを開く操作と新規ワークスペース作成操作を実行できる
- 既存フォルダを開く処理中または新規ワークスペース作成中は、両方のワークスペース操作を一時的に無効にし、処理中の表示にする
- 左レール下部のワークスペース項目は、ダブルクリックまたは右クリックメニューから名前変更できる
- ワークスペース名変更は、入力欄からフォーカスが外れたとき、またはEnterで確定する。ただしIME変換中のEnterでは確定しない
- ワークスペース名変更中にEscapeを押した場合、または空文字・変更前と同じ名前になった場合は、保存せず編集状態だけを終了する
- 左レール下部のワークスペース項目は、右クリックメニューまたは項目横の削除ボタンから登録一覧だけ削除できる
- 登録一覧から削除しても、OS上のワークスペースフォルダ自体は削除しない
- 登録済みワークスペースフォルダが削除・移動済みでも、アプリは状態取得を止めず、ファイルツリーを空として扱う
- アプリ設定に壊れたワークスペース登録（空ID、空の名前、相対パス、重複IDなど）が含まれる場合は、読み込み時に無効な登録だけを除外する
- 最後に開いていたワークスペースIDが登録一覧に存在しない場合は、起動時にアクティブワークスペースなしとして扱う
- ワークスペースIDは英数字、ハイフン、アンダースコアだけを有効とし、パス区切りや前後空白を含むIDは設定読み込み・切り替え・名前変更・ワークスペース別設定パス生成で受け付けない

## ファイルツリー選択とピン留め

- ファイルツリーでは、ファイルとフォルダを選択対象にする
- 通常クリックでは、その項目だけを選択する
- `Shift` を押しながらクリックすると、最後に選択した項目からクリックした項目までを範囲選択する
- macOSでは `⌘`、Windows/Linuxでは `Ctrl` を押しながらクリックすると、選択状態を個別に切り替える
- 複数選択中の項目を右クリックまたはドラッグした場合、選択中のファイル / フォルダをまとめて操作対象にする
- ファイルとフォルダは、ファイルツリー行のピンボタンまたは右クリックメニューからピン留め・解除できる
- ピン留めした項目は、ファイルサイドバー上部のピン留め欄に、ワークスペース別に保存された順番で表示する
- ピン留めされたパスが現在のファイルツリーに存在しない場合、その項目はピン留め欄に表示しない
- 保存済みのピン留めパスは、読み込み時にワークスペース相対パスだけへ正規化し、絶対パスやワークスペース外へ出るパスは除外する
- 開いているファイルはファイルツリー上で開いている状態として表示する
- ファイルを開いている途中は、短時間だけファイルツリー行に開き中の表示を付ける
- 同じファイルを開く処理が完了する前にもう一度クリックした場合は、その保留中の読み込みを取り消す
- ファイルツリーが空の場合は、Markdownファイルがないことを表示する

## パス安全性

- アプリ内のファイル操作は、ワークスペース内の正規化済み相対パスだけを対象にする
- 絶対パス、ワークスペース外へ出る `..`、空白だけのパス、ドットだけのパス、不正な形式のパスは処理前に拒否する
- ワークスペース内に見えるシンボリックリンクでも、実体がワークスペース外にある場合は、読み込み・書き込み・移動・削除・検索置換・チャート書き戻し・ファイル加工ツールの入出力対象にしない
- 新規保存先は、存在する親フォルダの実体がワークスペース内である場合だけ許可する。親フォルダの確認が権限エラーなどで失敗した場合は安全側で拒否する
- 子フォルダだけが読めない場合、ファイルツリー全体を失敗させず、その子フォルダを空として扱う
