# features/links.md

このファイルは内部リンク・バックリンク・タグ機能の詳細仕様書です。
`[[...]]` リンク・バックリンク一覧・フロントマター `tags:` による整理・オートコンプリートなど、
ノート間の接続に関わる機能の振る舞いをここに定義します。

---

## 内部リンク `[[...]]`

- リンクの対象は現在開いているワークスペース内のファイルのみ。複数ワークスペース間のリンクは非対応。
- `[[ファイル名]]` 記法でファイル間を接続する内部リンクを作成する
- リンク先ファイルが存在しない場合：
  - リンクとして表示しつつ、色や下線などのスタイルで「未作成」であることを示す
  - クリックすると自動で新規ファイルを作成し、そのファイルを開く（作成場所：リンクが書かれているファイルと同じフォルダ）
- `[[` を入力した時点で、ファイル名のオートコンプリートをカーソル直下にドロップダウンリストで表示する
- パス指定 `[[フォルダ名/ファイル名]]` の書き方をサポートする（手打ち・補完どちらも可）
- 同じファイル名が複数フォルダに存在する場合、補完候補に `フォルダ名/ファイル名` の形式で両方を表示し、ユーザーが選択する。選択したリンクはパス付き `[[フォルダ名/ファイル名]]` で挿入される

### 対応する記法バリエーション

| 記法 | 動作 |
|------|------|
| `[[ファイル名]]` | ファイルを開く |
| `[[ファイル名\|表示テキスト]]` | 表示テキストを変えてファイルを開く |
| `[[ファイル名#見出し]]` | ファイルを開き、指定した見出しの位置にスクロール |
| `[[ファイル名^ブロックID]]` | ファイルを開き、指定したブロックの位置にスクロール |
| `![[ファイル名]]` | ファイルの内容をその場に展開して表示（埋め込み） |

### ブロック参照について

- ツールバーのブロックID挿入ボタンで、カーソル位置の段落末尾に `^abc123` のようなIDを自動生成して挿入できる
- テキストを複数段落にまたがって選択している場合、選択範囲内の各段落末尾にIDを挿入できる
- 既に段落末尾にブロックIDがある場合は重複挿入しない
- ブロックIDは手入力してもよい
- 既存の手書きIDはリンク解決・リネーム追跡の対象として読み取る

### ファイル埋め込みについて

- `![[ファイル名]]` は一段階だけ展開表示する
- 埋め込み先の本文にさらに `![[...]]` があっても、その内側の埋め込みは展開しない
- 大きすぎるファイルは全文表示せず、元ファイルを開いて確認する導線を表示する
- 埋め込み内リンクは通常の内部リンクと同じように開く
- 埋め込み内では直接編集せず、編集する場合は元ファイルを開く

## バックリンク・アウトゴーイングリンク

- **バックリンク一覧**：そのファイルを参照しているファイルの一覧
- **アウトゴーイングリンク一覧**：そのファイルが参照しているリンクの一覧
- バックリンク・アウトゴーイングリンクはナビゲーション補助として位置づける
- 両一覧ともに、右パネル（アウトラインパネルと同じ領域）に表示する
- エクスポート・印刷の対象には含まれない

---

## タグ

- タグはフロントマターの固定プロパティ `tags:` に記述する
- 本文中の `#タグ名` 記法はタグとして扱わない
- 本文中で概念・人物・資料などを参照する場合は、インラインリンク `[[...]]` を使う
- タグを使って横断的な検索・絞り込みが可能
- **階層タグをサポート**：`親/子` の形式で `/` によるネストが可能（例：`キャラ/主人公`）
- **日本語タグをサポート**：日本語・漢字・ひらがな・カタカナを含むタグ名が使用可能
- タグ入力時は、ワークスペース内の既存 `tags:` から候補を出し、表記揺れを抑える

---

## フロントマター（YAML）

フロントマターの詳細仕様は [frontmatter.md](frontmatter.md) を参照。

- ファイル先頭の `---` ブロックをフロントマターとして認識する
- `tags:` フィールドはRelicの固定プロパティとしてタグ検索・タグ一覧に使われる（詳細は frontmatter.md 参照）

## フロントマターテンプレート

- フロントマターテンプレートは、複数の能力付きフィールド名をまとめるアプリ設定上の保存済みセットとして扱う
- 現行UIでは、テンプレート管理画面やテンプレート適用操作は表示しない
- テンプレートはアプリ設定として保存し、ワークスペース内の `templates/` フォルダや `.md` テンプレートファイルには依存しない
- 新規ファイル作成時に本文テンプレートを選択する機能は、現行アプリには持たない
- 詳細は [frontmatter.md](frontmatter.md) を参照する
