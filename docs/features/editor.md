# features/editor.md

エディタ機能の詳細仕様書。テキスト編集に関わるモード・表示・操作の振る舞いを定義する。

---

## モード

| モード | 説明 |
|--------|------|
| ライブプレビュー | マークダウン記法をリアルタイムにレンダリングして表示する |
| ソースモード | Markdown記法とフロントマターYAMLをそのまま表示する |
| タイプライターモード | カーソル行が画面中央付近に来るようにスクロールする。他のモードと併用可能 |

- **デフォルト**：ライブプレビュー
- ライブプレビューを基本の編集画面とし、必要に応じてソースモードへ切り替えられる
- ソースモードでは、フロントマターをフォーム化せず、Markdownファイル内のYAML構文として表示・編集する
- 別ペインの専用プレビューモードは持たない
- タイプライターモードは独立してオン/オフ切り替え可能
- 本文が1MiBを超える、または1行が80,000文字を超えるMarkdownは、編集性能を優先してソース表示で開き、大きいファイルであることを通知する
- **ライブプレビュー中の記法表示**：カーソルまたは選択範囲が触れている装飾範囲だけ記法（`**`、`==`、`#` 等）を表示する。このとき太字・ハイライト・見出しなどのレンダリング自体は維持する（Obsidianスタイル）
- `mermaid` または `d2` を指定したコードブロックは、ライブプレビューで図として表示する。図の操作中は表示を維持し、Markdownソースを編集する場合は図ブロック内の「ソースを編集」ボタンで明示的に編集状態へ入る。
- カーソルのある現在行は、ソースモードとライブプレビューのどちらでも薄くハイライトする。

## 見出し折りたたみ

エディタ本文内のMarkdown見出しは、見出し行の左側に表示する開閉ボタンで折りたためる。

- 対象は `#` 〜 `######` の見出し行
- 折りたたみ範囲は、その見出しの次行から、同じ階層以上の次の見出しの直前までとする
- 折りたたみ中は見出し行だけを残し、見出し末尾に省略表示を出す
- 折りたたみ操作はライブプレビューとソースモードの両方で使える
- 折りたたみ状態はMarkdown本文へ書き込まず、アプリ内設定にも保存しない
- ファイルを開き直したときは、すべて展開状態に戻る
- 右パネルのアウトラインは、見出し折りたたみの操作対象にはしない

## 表示設定

| 設定項目 | デフォルト | 説明 |
|----------|-----------|------|
| フォントサイズ | 16px | エディタの文字サイズを変更できる |
| 行間 | 1.7 | 行の間隔を広げたり狭めたりできる |
| エディタ最大幅 | 標準（660px） | プリセットから選んで設定する（詳細は下記） |
| 行番号表示 | オフ | 設定でオン/オフを切り替えられる |
| フォント | システムフォント | プリセット4種から選択（下記参照） |
| スペルチェック | オン | Electron / OS標準のスペルチェックと連携。設定でオフにできる |

### エディタ最大幅

テキストカラムの最大幅をプリセットから選んで設定できる。ウィンドウを広げてもテキストが伸びすぎず読みやすい幅に収まる。

| プリセット名 | 幅 | 1行の目安文字数（日本語） |
|---|---|---|
| 狭め | 550px | 約30文字 |
| 標準 | 660px | 約36文字 |
| 広め | 800px | 約44文字 |
| 制限なし | — | ウィンドウ全幅 |

- **デフォルト**：標準（660px）

### フォントプリセット

エディタのフォントはプリセット4種から選べる。

| 種類 | macOSの表示名 | Windowsの表示名 | 特徴 |
|---|---|---|---|
| システムフォント | OS標準UIフォント | OS標準UIフォント | OS標準。日本語・英語ともに読みやすい |
| ゴシック | ヒラギノ角ゴシック | 游ゴシック | 角ゴシック系。本文をすっきり読める |
| ヒラギノ明朝 | ヒラギノ明朝 | 游明朝 | 明朝体。縦組みコンテンツや日本語長文に向く |
| Menlo | Menlo | Consolas | 等幅フォント。コードや技術メモ向き |

- **デフォルト**：システムフォント

## 自動保存

- 保存方式：**自動保存のみ**（手動保存なし）
- タイミング：入力停止後 **1秒** で自動保存
- ファイルタブを閉じる前、ファイル名変更前など、未保存の本文を失う可能性がある操作では即時保存を試みる
- 即時保存に失敗した場合は、対象操作を進めず、タブと本文を維持してエラーを表示する
- アプリウィンドウを閉じる前にも、開いているファイルタブの未保存本文や外部変更衝突を確認し、必要な保存確認が通らない場合はウィンドウを閉じない
- 保存中または保存待ちの本文がある状態で外部変更を検出した場合は、衝突として扱い、自動保存を止める
- 衝突中は保存成功扱いにせず、衝突後に残っていた保存待ち本文を追加保存しない
- ワークスペース変更通知を受けた場合は、現在のワークスペース状態を再取得し、ファイルツリー、ピン留め、開いているMarkdownタブの外部変更状態を更新する
- 変更通知のワークスペースIDが現在のアクティブワークスペースと異なる場合は、その通知を反映しない
- 開いているMarkdownファイルがファイルツリーから消えている場合は、そのファイルタブを閉じる
- 開いているMarkdownファイルの外部本文が保存済み本文と同じ場合は、タブ本文を変更しない
- 開いているMarkdownファイルの外部本文が現在本文と同じ場合は、その内容を保存済みとして扱い、未保存状態を解消する
- 未編集の開きタブで外部本文だけが変わっている場合は、外部本文をタブへ自動反映する
- 編集中の開きタブで外部本文が変わっている場合は、Relic側本文を保持したまま外部変更衝突として扱う
- 衝突通知では、外部版を読み込むか、Relic側の本文を保存するかを選べる
- 外部版を選ぶと、外部変更後の本文でタブ内容を更新する
- Relic側を選ぶと、現在タブの本文を保存する。保存に失敗した場合は衝突状態を維持する

## ステータスバー

画面下部にステータスバーを表示する。以下の情報を常時表示する：

- フォーカス中ペインで開いている現在のMarkdownファイルの文字数
- フォーカス中ペインで開いている現在のMarkdownファイルの単語数
- フォーカス中ペインで開いている現在のMarkdownファイルの保存状態
- アプリ全体のステータスバーにはアプリ名を表示する
- フォーカス中ペインでMarkdownファイルが開かれている場合は、アプリ全体のステータスバーにも保存状態を表示する
- フォーカス中ペインでMarkdownファイルが開かれていない場合、アプリ全体のステータスバーの文字数・単語数は0として表示する
- 各ファイルペイン下部にも、そのペインで開いているファイルの文字数・単語数を表示する
- 文字数は本文文字列の長さで数える
- 単語数は空白区切りで数え、空文字は除外する

## Markdown操作メニュー

- **表示**：エディタ本文の右クリックメニューとして表示する
- **レイアウト**：クリップボード系 / 書式系 / リスト系 / 見出し・ブロック系 / 挿入系の操作をまとまりごとに配置する
- 現行UIには、エディタ上部に常時表示されるMarkdown書式ツールバーはない
- ライブプレビュー表のセル上では、表専用の右クリックメニューを優先し、通常のMarkdown操作メニューは開かない
- 選択範囲を右クリックした場合は、その選択範囲を対象にして操作する
- 選択範囲を右クリック後にメニュー操作を行う場合、対象範囲を一時的にハイライトし、メニューを閉じるとハイライトを解除する
- メニュー外側のクリック、またはEscapeで閉じる

## Markdown操作

右クリックメニューに以下の操作を配置する。操作を選ぶと選択中のテキストに対応するMarkdown記法を適用する（選択なしの場合はカーソル位置に記法を挿入する）。
選択範囲を記法で包む操作では、挿入後も記法の内側のテキストを選択状態にする。
対象エディタは、アクティブなDOM要素、ブラウザ選択範囲、フォーカス中エディタ、非空選択範囲を持つエディタ、最後に操作対象だったエディタの順で決める。

### クリップボード系

| 操作 | 動作 |
|------|------|
| コピー | 選択範囲、または右クリック時に保持していた選択範囲の本文をクリップボードへコピーする |
| 切り取り | コピーと同じ対象をクリップボードへコピーし、本文から削除する |
| 貼り付け | 右クリック位置または選択範囲にクリップボード本文を挿入する。Clipboard APIで読めない場合はブラウザ標準の貼り付けへ委ねる |
| 全選択 | 現在のエディタ本文全体を選択する |

### 書式系

| ボタン | 適用される記法 |
|--------|--------------|
| 太字 | `**テキスト**` |
| 斜体 | `*テキスト*` |
| 打ち消し線 | `~~テキスト~~` |
| ハイライト | `==テキスト==` |
| 下線 | `<u>テキスト</u>` |
| インラインコード | `` `テキスト` `` |

- 選択範囲がない場合は、各記法の内側にプレースホルダー文字列を入れる

### ブロック系

| ボタン | 適用される記法 |
|--------|--------------|
| 見出し | H1〜H6をドロップダウンで選択して適用 |
| 引用 | `> テキスト` |
| コードブロック | 空のコードフェンス ` ``` ` を挿入する |
| 水平線 | `---` を挿入 |

- 見出しと引用は、現在行の先頭に記法を付け、行が空の場合はプレースホルダー文字列を入れる
- コードブロックと水平線は、現在行の後ろにブロックとして挿入する
- 見出しメニューを閉じる場合は、約130msの退場表示を出してから閉じる

### リスト系

| ボタン | 適用される記法 |
|--------|--------------|
| 箇条書き | `- テキスト` |
| 番号付きリスト | `1. テキスト` |
| チェックボックス | `- [ ] テキスト` |

- 単一行で同じ種類のリスト操作をもう一度実行すると、リスト記法を外す
- 単一行で別のリスト操作を実行すると、箇条書き・番号付きリスト・チェックボックス間で変換する
- 複数行選択時は、選択範囲内の非空行だけにリスト記法を適用する
- 複数行選択時の番号付きリストは、空行を数えずに連番を振る
- 複数行選択時に対象の非空行がすべて同じ種類のリストなら、そのリスト記法をまとめて外す

### 挿入系

| ボタン | 動作 |
|--------|------|
| リンク | `[テキスト](URL)` を挿入。URLの入力ダイアログを表示する |
| 内部リンク | `[[` を挿入し、ファイル名のオートコンプリートを起動する |
| 表 | 行数・列数を指定するダイアログを表示し、テーブルのMarkdownを挿入する |

- Markdownリンクは、選択範囲がある場合はその文字列をリンクテキストにし、選択範囲がない場合はプレースホルダー文字列をリンクテキストにする
- MarkdownリンクのURL入力でEnterを押すと挿入し、Escapeを押すと入力を閉じる
- MarkdownリンクのURL入力ダイアログを閉じる場合は、約130msの退場表示を出してから閉じる
- 内部リンクは、選択範囲がある場合は `[[選択範囲]]` で包み、選択範囲がない場合は `[[]]` を挿入してカーソルを角括弧の内側に置く
- 表挿入の行数・列数は1以上に丸め、未入力または数値として扱えない場合は3行3列として挿入する
- 表の列見出しは、表示言語に応じた既定ラベルを列番号付きで入れる
- 表挿入の入力でEnterを押すと挿入し、Escapeを押すと入力を閉じる
- 表挿入ダイアログを閉じる場合は、約130msの退場表示を出してから閉じる

### ライブプレビュー表操作

ライブプレビュー中のMarkdownテーブルは、表として表示したままセル単位で編集できる。

- セル入力後、Markdown本文のテーブル記法へ書き戻す
- Enterで下の行へ移動し、最終行では新しい行を追加する
- Tab / Shift+Tabで隣のセルへ移動し、末尾では新しい行を追加する
- 上下キーで同じ列の上下セルへ移動する
- 左右キーは、入力カーソルがセルの端にある場合だけ左右セルへ移動する
- TSVを貼り付けると、複数セルへ展開する
- TSV貼り付けが現在の表の行数や列数を超える場合は、必要な行や列を追加して展開する
- タブや改行を含まない単一テキストの貼り付けは、通常のセル入力として扱い、表範囲へ展開しない
- ドラッグでセル範囲を選択し、コピー時はTSVとしてコピーする
- Delete / Backspaceで選択範囲のセル内容だけを空にし、行列構造は維持する
- 表外側の追加ボタン、行列ハンドル、右クリックメニューから行列の追加・削除・移動・コピー・貼り付け・並べ替えができる
- ヘッダー行は表の見出しとして扱い、行の削除、上下移動、ドラッグ移動の対象にしない
- 表がヘッダー行と本文1行だけの場合、本文行の削除はできない
- 表が1列だけの場合、列削除はできない
- 行コピーはコピーした行を選択行の下へ貼り付け、列コピーはコピーした列を選択列の右へ貼り付ける。行と列のコピー内容は別種別として扱い、種別が合わない貼り付け操作は実行しない
- 列ソートはヘッダー行を固定し、本文行だけを対象に、数値を考慮した大文字小文字非区別の昇順または降順で並べ替える

### ライブプレビューチェックボックス操作

ライブプレビュー中のMarkdownチェックボックスは、表示されたチェックボックスをクリックして本文の `- [ ]` / `- [x]` を切り替えられる。

- チェック状態だけを書き換え、行の本文やリスト構造は維持する
- 大文字の `X` を含む既存チェックも、クリック後は現在の状態に合わせて `[ ]` または `[x]` として保存する

### ライブプレビュー図表操作

ライブプレビュー中の `mermaid` / `d2` コードブロックは、図として表示したまま閲覧操作できる。

- 「全体表示」で図全体が表示領域に収まる倍率へ戻す
- ホイールでズーム、ドラッグで移動できる
- `+` / `-` で拡大縮小、`0` または `f` で全体表示、矢印キーで移動できる
- ソースを変更する場合は「ソースを編集」ボタンを押して、対象コードブロックのMarkdownソースを編集する
- 図の描画に失敗した場合は、エラー表示から図表ソースと詳細エラーをコピーできる
- 図表レンダラーが文字列を返しても、サニタイズ後にSVGとして表示できる内容が残らない場合は描画失敗として扱う
- 図の描画に成功している場合は、図ごとにSVGとして保存できる
- 図の描画に成功している場合は、図ごとにSVG文字列をクリップボードへコピーできる
- SVG保存の初期ファイル名が空になる場合は、SVG用の既定名 `relic-diagram.svg` を使う

### リスト編集キー操作

- リスト行でEnterを押すと、同じリスト種別の次の項目を作る
- 番号付きリストでEnterを押すと、次の番号を自動で入れる
- チェックボックスでEnterを押すと、次の未チェック項目を作る
- 空のリスト項目でEnterを押すと、リスト記法を外して通常行に戻す
- 選択行でTabを押すと字下げし、Shift+Tabで字下げを戻す
- `Alt+ArrowUp` / `Alt+ArrowDown` で選択行を上下に移動する

## 現在ファイルの出力

開いているMarkdownファイルを、現在のプレビュー内容にもとづいて出力できる。

| 操作 | 動作 |
|------|------|
| 印刷 | 印刷用ウィンドウを開き、OS標準の印刷画面を表示する |
| PDFとして保存 | 保存先を選び、A4向けのPDFとして保存する |

- 出力対象は、フォーカス中ペインのアクティブなMarkdownファイルとする
- MarkdownをHTMLへレンダリングし、出力専用の白背景スタイルで印刷/PDF化する
- 出力HTMLには、RelicのアプリUIや編集用コントロールを含めず、レンダリング済みMarkdown本文だけを含める
- 出力HTMLは、外部スクリプトや外部リソースを前提にしない印刷用HTMLとして生成する
- 出力タイトルは、明示タイトル、本文最初のH1、ファイルパス由来のファイル名、表示ファイル名、既定名 `Relic` の順で決める
- 保存ファイル名の初期値は、ファイルパス由来のファイル名、表示ファイル名、本文最初のH1、既定名 `relic-preview` の順で決める
- 保存ファイル名の初期値は、OSで扱いにくい文字や末尾のドット・空白を取り除き、長すぎる場合は短くする
- `mermaid` / `d2` 図は、可能な限りSVGとして描画した状態で出力する
- PDF保存では、拡張子 `.pdf` を付け、最終ファイルへ直接書き込まず一時ファイル経由で安全に保存する
- PDF保存では、保存ダイアログでユーザーが保存先とファイル名を選ぶ
- PDF保存ダイアログをキャンセルした場合、または保存先パスが返らない場合は、エラー表示を出さず、PDF生成もファイル書き込みも行わない
- PDF保存では、生成用の非表示出力ウィンドウでHTMLを読み込み、A4、背景印刷あり、上下48px・左右52px相当の余白でPDF化する
- PDF保存用ウィンドウではインラインスクリプトを許可しない
- 印刷では、印刷用ウィンドウ内に「OS標準の印刷画面を開く」ボタンと「閉じる」ボタンを表示する。印刷時はこのボタン類を印刷対象に含めない
- 印刷用ウィンドウは、起動後にOS標準の印刷画面を自動で開く
- 印刷用ウィンドウは白背景の別ウィンドウとして開き、初期サイズは幅780px・高さ900pxとする
- 印刷用ウィンドウでは、印刷操作バーを動かすためのインラインスクリプトだけを許可し、出力HTMLのCSPも印刷操作バー用に差し替える
- 出力用ウィンドウは、外部ページ遷移、新しいウィンドウ作成、権限要求を許可しない
