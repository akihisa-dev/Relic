# features/diagrams.md

Diagram / 図解機能の詳細仕様書。
Markdownファイルを正本として扱いながら、図形とLineを専用キャンバスで編集する振る舞いを定義する。

---

## 基本方針

- Diagramは通常の `.md` ファイルとして保存する
- Diagramの正本はMarkdownファイルの本文であり、画面上だけに存在する図解データは作らない
- Diagramファイルは、ファイル先頭のYAMLフロントマターで判定する
- 図解用の保存形式は、Relic独自のバイナリや外部サービスではなく、Markdown内に書かれたYAMLとする
- DiagramファイルにDiagram以外の自由本文は混ぜない
- 壊れたDiagramファイルは、通常のDiagramキャンバス編集対象にしない
- Diagramの一時的な表示位置、zoom、選択状態はMarkdownファイルへ保存しない
- Relationship、構造ツリー、自由図というDiagramファイルカテゴリは持たない
- 旧 `type: relationship`、`type: why-tree`、`type: free-drawing` はDiagramファイルとして扱わない

Diagramファイルとして扱うフロントマターの `type` は `diagram` だけとする。

```yaml
---
type: diagram
formatVersion: 1
title: 図解ファイル
---
```

フロントマターで扱うDiagram共通フィールドは `type`、`formatVersion`、任意の `title` だけとする。
他のフロントマター項目が混ざっているMarkdownは、Diagramファイルとしては扱わない。
`formatVersion` がない既存の `type: diagram` ファイルは旧形式として読み込み、画面上では現行形式として扱う。
Relicが対応していない将来の `formatVersion` は、旧形式として誤読せずDiagramファイルとして扱わない。

## Diagramサイドバー

- 左レールのDiagram入口からDiagramサイドバーを開く
- Diagram入口は機能トグルの対象ではなく、初期状態で表示する
- ワークスペース未選択時は、ファイルサイドバーと同じく既存フォルダを開く操作と新規ワークスペース作成操作を表示する
- ワークスペース選択中は、Diagram作成操作、編集中Diagramの図形パレット、Diagramファイル一覧を表示する
- Diagramファイル一覧には、ワークスペースファイルインデックスで `diagram` と判定されたMarkdownファイルを表示する
- Diagramファイル一覧の項目をクリックすると、そのDiagramファイルを開く
- Diagramファイル一覧の項目は右クリックメニューからゴミ箱へ移動できる
- 開いているファイルがDiagramファイルの場合、Diagramファイル一覧より上に図形パレットを表示する
- 図形パレットは、開始/終了、処理、判断、入出力を「基本フロー」、領域を「構造」として用途別に表示する
- 図形パレットの各項目には、図形の輪郭と図形名を常時表示する
- 図形はクリック、Enter、Spaceで現在表示しているキャンバス中央へ追加できる
- 図形をドラッグしてキャンバスへドロップすると、その位置へ追加できる
- キャンバス上で図形を追加する入口は、Diagramサイドバーの図形パレットと、NodeがないDiagramの開始パネルに集約する
- キャンバス上のNodeには接続用ポートを表示し、ポートから別のNodeへドラッグするとLineを追加する

Diagramサイドバーから作成できるDiagramは、図解ファイル1種類だけとする。

- 図解ファイルの既定ファイル名は `図解ファイル.md` 系とする
- 同名ファイルがある場合は、既存ファイルを上書きせず、番号付きの別名を使う
- 作成したDiagramファイルは初期本文を書き込んだうえで開く

## Diagramファイルの表示

- Diagramファイルを通常ファイルタブとして開く
- Diagramファイルでは通常Markdown用の大きなタイトル行を使わず、コンパクトヘッダーにファイル名変更ボタンと既存のソースモード切替アイコンを表示する
- ソースモードがオフの場合、通常のMarkdownエディタではなくDiagramキャンバスを表示する
- ソースモードをオンにすると、同じコンパクトヘッダーを維持したままDiagramファイルのMarkdown本文を直接編集できる
- Diagramファイルでもファイルタイトルのリネーム、タブ操作、自動保存は通常Markdownファイルと同じ扱いにする
- Diagramキャンバスでの編集は、Markdown本文を書き換え、既存の自動保存に乗せる
- Diagramキャンバスには、Mermaidソースをコピーする操作を表示しない
- Diagramファイルのステータスバーには、通常の文字数・単語数ではなくDiagram内容の件数と現在状態を短く表示する
- Diagramキャンバスの拒否理由表示は、画面表示と読み上げ通知の両方で扱う
- Diagramとして読めない形式の場合は、Diagramキャンバスに無効なファイルであること、原因の要約、エラーコード、分類、詳細、取得できる場合は行番号と列番号を表示する
- 無効なDiagram表示では、詳細エラーをコピーでき、ソースモードでMarkdown本文を開ける
- 無効なDiagram表示だけでは、Markdown本文の自動修復、自動削除、自動保存を行わない
- PDF保存時は、DiagramファイルをMarkdownソースではなく図解表示として出力し、図形のアウトラインを維持する

## 保存形式

DiagramのMarkdownは以下の構造で保存する。

```yaml
---
type: diagram
formatVersion: 1
title: 図解ファイル
---

nodes: []
lines: []
```

| フィールド | 内容 |
|-----------|------|
| `formatVersion` | Diagram保存形式のバージョン。現行形式は `1` |
| `nodes` | Diagram上のNode一覧 |
| `nodes[].id` | Diagramファイル内で一意のNode ID |
| `nodes[].shape` | 図形の種類。`terminator`、`process`、`decision`、`input-output`、`area` のいずれか |
| `nodes[].text` | 領域図形では領域名、それ以外の図形では図形に付くラベルとして表示・保存する自由テキスト |
| `nodes[].x` / `nodes[].y` | Node位置 |
| `nodes[].width` / `nodes[].height` | Nodeの固定サイズ |
| `nodes[].layer` | 図形ごとの固定表示レイヤー。領域図形は `0`、領域図形以外の図形本体は `1` とする。図形に付くラベルは表示上の固定レイヤー `2` とし、別フィールドとして保存しない |
| `lines` | Node同士をつなぐLine一覧 |
| `lines[].id` | Diagramファイル内で一意のLine ID |
| `lines[].from` / `lines[].to` | 接続するNode ID |
| `lines[].label` | Line中央に表示するLineラベル。図形に付くラベルとは別に扱う |

- `nodes` と `lines` は旧形式読み込み時に未指定の場合、空配列として扱う
- 旧形式で `nodes[].shape` が未指定の場合は、`process` として扱う
- `nodes[].text` は文字列として扱い、空文字を許可する
- 領域図形の `nodes[].layer` は常に `0` として扱う
- `0` は領域図形専用のレイヤーとし、領域図形以外の `nodes[].layer` は常に `1` として扱う
- 図形に付くラベルは表示上の固定レイヤー `2` とし、図形本体より上に表示する
- 旧形式で `nodes[].layer` が未指定の場合、領域図形は `0`、領域図形以外は `1` として扱う
- 現行形式では `label` 図形を扱わない。`nodes[].shape` が `label` のファイルは不正なDiagramとして扱う
- 旧形式で現在ルールと異なる `nodes[].layer` が保存されている場合は、読み込み時に領域図形 `0`、領域図形以外 `1` へ合わせて扱う
- Lineの表示レイヤーは接続先の図形関係から表示時に決め、MarkdownへLine専用のレイヤーは保存しない
- Diagramは循環、多対多、横断関係を許可する
- 同じNode一対のLineは最大2本まで作れる
- 同じ向きのLineは1本だけ作れる
- 自分自身へ向かうLineは作らない

## 操作

- 図形パレットから図形をクリック、Enter、Spaceで選ぶと、現在表示しているキャンバス中央へ図形を追加できる
- 図形一覧から図形をドラッグし、キャンバスへドロップすると、その位置へ図形を追加できる
- ドラッグ中は、ドロップ予定位置と追加予定サイズをキャンバス上に表示する
- 追加位置が既存図形と重なる場合は、近くの空いているグリッド位置へずらす
- NodeがないDiagramでは、キャンバス中央に開始パネルを表示し、図形選択メニューから最初の図形を追加できる
- 開始パネルは画面上の固定表示として扱い、pan、zoom、選択状態と同じくMarkdownへ保存しない
- 追加できる図形は、開始/終了、処理、判断、入出力、領域とする
- 新規図形には、図形種類に応じた初期テキストを入れる
- 領域図形以外の新規図形は、初期サイズを横5マス、縦2マスとする。現行グリッドでは `160 x 64` として扱う
- 領域図形は、他の図形を囲む背景領域として使う大きめの長方形とし、初期レイヤーは `0` とする
- 領域図形は、完全に内側へ入っている他の図形を内包しているものとして扱う
- 領域図形を移動すると、完全に内包されている図形も同じ距離だけ移動する
- 領域図形の塗りは半透明にし、領域名と枠線は読み取れる濃さを維持する
- 領域図形の `nodes[].text` は領域名ラベルとして左上に表示し、領域内の本文として中央には表示しない
- 領域図形以外の図形では、`nodes[].text` を図形に付くラベルとして中央に表示する
- 図形に付くラベルは、Lineラベルとは別の表示要素として扱い、常に図形本体より上に表示する
- Nodeには移動できることを示すカーソルを表示し、Node上にポインターを重ねたとき、またはNode選択時に接続用ポートとリサイズ用グリップを表示する
- Nodeの接続用ポートから別のNodeへドラッグすると、接続元Nodeから接続先NodeへLineを追加する
- Nodeのテキストは文字入力できることを示すカーソルを表示し、ダブルクリックまたはキーボード操作で直接編集に入る
- 判断図形から出るLineは最大2本までとする
- 判断図形からLineを追加する場合、Lineラベルは標準選択肢として `YES`、`NO` の順に自動入力する。Lineラベルは追加後にユーザーが自由に編集できる
- 既存ファイルなどで判断図形から出るLineが3本以上ある場合、3本目以降は表示・選択・Lineラベル編集・削除などの操作対象にしない
- 判断図形へ入るLineは、入ってくるLine群の平均方向にもっとも近い1つの角へ集約する
- 判断図形から出るLineは、Lineの行き先方向に近い角から出す。入ってくるLineがある場合は、入力用に決まった角を避けて出る角を選ぶ。2本の出力Lineがある場合は、それぞれ別々の角から出す
- 判断図形へ入るLineがない場合、入力用の角は予約せず、出るLineは上下左右4つの角から行き先方向に近い角を選ぶ
- 通常時のNodeはNode内側をドラッグして移動できる
- Nodeをダブルクリックするとテキスト編集に入り、編集確定時にMarkdown本文へ書き戻す
- Diagramの表示層は固定とし、領域図形は `0`、領域以外の図形本体は `1`、図形に付くラベルは `2` として扱う
- Diagramでは図形のレイヤーを前面または背面へ変更する操作を表示しない
- 判断図形や入出力図形などの非矩形図形では、選択中やドラッグ中も図形外側の四角い領域枠を表示しない
- Node移動、Nodeリサイズ、Line作成、Line削除、Line方向変更、Node削除、Lineラベル編集はMarkdown本文へ書き戻す
- pan、zoom、選択状態、Undo/Redo履歴、コピー状態はDiagramファイルごとの画面状態として扱い、別ファイルへ持ち越さない
- キャンバスの固定ツールバーには、Undo、Redo、全体表示、縮小、現在倍率、拡大、100%表示を標準的なアイコン中心で常時表示する
- Nodeを右クリック、またはNode選択中にコンテキストメニューキーか `Shift+F10` を押すと、コピー、複製、図形種類変更、削除の操作を表示する
- 複数Nodeを右クリック、または複数Node選択中にコンテキストメニューキーか `Shift+F10` を押すと、コピー、複製、整列、等間隔配置、削除の操作を表示する
- Lineを右クリック、またはLine選択中にコンテキストメニューキーか `Shift+F10` を押すと、方向反転、削除の操作を表示する
- Lineを選択すると、Lineの始点と終点に付け替え用ハンドルを表示する
- LineラベルはLine中央のラベルを直接操作して編集に入る
- ステータスバーは、未選択時は図形数・線数、選択時や編集中は現在状態だけを短く1行で表示する
- 画面上には操作説明文を常時表示せず、拒否理由や読み上げに必要な状態だけを表示する
- Diagramキャンバス全体はインタラクティブ領域として扱い、図形の読み上げ名には内部識別子ではなく画面上の図形名、選択状態、接続可否を含める
- 図形種類変更は、開始/終了、処理、判断、入出力の相互変更だけを許可し、領域図形への変更と領域図形からの変更は許可しない
- 図形はShiftクリックで複数選択でき、複数選択中はまとめて移動、削除、複製、コピー、貼り付け、整列、等間隔配置できる
- 複数図形を複製または貼り付ける場合、Node IDは新しく採番し、選択範囲内のNode同士を結ぶLineだけを複製する
- Lineを選択すると、Lineの始点と終点に付け替え用ハンドルを表示する
- Lineの始点または終点ハンドルを別の図形へドラッグすると、Line IDとLineラベルを維持したまま接続元または接続先を変更する
- Line端点の付け替え中は変更後の接続をプレビューし、接続可能な図形と接続できない図形を区別して表示する
- キャンバスでは、自己接続、同じ向きの重複Line、同じNode一対の3本目、判断図形からの3本目の通常Line、Line方向反転後または端点付け替え後の重複を拒否し、拒否理由を画面に表示する
- 無効なLine端点付け替え、Escape、または図形以外でポインターを離した場合はMarkdown本文を書き換えず、元のLine ID、Lineラベル、接続を維持する
- Diagramキャンバス上では、`Cmd/Ctrl+Z` でUndo、`Cmd/Ctrl+Shift+Z` でRedo、`Cmd/Ctrl+D` で複製、`Cmd/Ctrl+C` でコピー、`Cmd/Ctrl+V` で貼り付け、`L` でLine作成開始と確定、矢印キーで選択図形の移動または表示位置の移動を行う
- Tabキーで図形へ移動でき、選択図形ではEnterまたはF2でテキスト編集に入る。入力欄内のキー操作はDiagram全体のショートカットとして扱わない
- 空白部分をドラッグすると表示位置を動かせる
- Shiftキーを押しながら空白部分をドラッグすると、範囲内の図形をまとめて選択できる
- ホイール操作でカーソル位置を基準に拡大縮小できる

pan、zoom、選択状態、ドラッグ中の一時位置は画面だけの状態とし、Markdownファイルへ保存しない。

## ファイル操作との関係

- Diagramファイルはファイル参照Nodeを持たない
- ファイル名変更、ファイル移動、フォルダ名変更、フォルダ移動によるDiagram内参照更新は行わない
- Diagramファイルも `.md` ファイルであるため、全文検索の対象に含める
- Diagramファイル内のYAML本文は、Diagramの保存形式として検索対象になる
- DiagramサイドバーのDiagram一覧は、ワークスペースファイルインデックスを使う
- Diagramファイルは通常Markdownファイルと同じ `.md` ファイルだが、ファイルツリーでは通常Markdownと同じファイルとして表示する

## Markdown図表コードブロックとの違い

Diagram / 図解ファイルは、`mermaid` / `d2` の図表コードブロックとは別の機能である。

| 項目 | Diagram / 図解ファイル | 図表コードブロック |
|------|-------------------------|--------------------|
| 保存場所 | 1つの `.md` ファイル全体 | 通常Markdown本文のコードブロック |
| 正本 | フロントマターとYAML本文 | コードブロック内のMermaid/D2ソース |
| 編集画面 | Diagramキャンバス | Markdownエディタ内のライブプレビュー |
| 用途 | 図形とLineを直接配置する図解 | テキストで書いた図表のプレビュー |
