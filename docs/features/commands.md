# features/commands.md

コマンドパレットとクイックスイッチャーの仕様。

---

## 目的

キーボード操作で、ファイル操作とアプリ操作をすばやく実行できるようにする。

---

## コマンドパレット

アプリ内の主要操作を検索して実行するUI。

### 起動

- macOSでは `⌘⇧P`、Windows/Linuxでは `Ctrl+Shift+P` で開く
- メインエリア上にモーダルとして表示する
- サイドバーの表示状態に関係なく開ける

### 表示内容

- 入力欄
- 入力に一致するコマンド一覧
- 各コマンドの名前
- ショートカットがある場合は、利用中のOSに合わせたショートカット表記
- 一致するコマンドがない場合は、候補なしの表示を出す

### 対象コマンド

対象カテゴリは以下とする。

| カテゴリ | 例 |
|----------|----|
| ファイル操作 | 新規ファイル作成、アクティブファイルの名前変更導線、複製、削除 |
| 表示操作 | サイドバー表示切り替え、右パネル表示切り替え、分割表示 |
| エディタ操作 | タイプライターモード |
| 検索・移動 | ファイルサイドバーの検索欄へ移動、クイックスイッチャーを開く |
| 設定 | 設定サイドバー表示へ切り替える |

### 挙動

- 入力に応じて候補を絞り込む
- Enterで選択中のコマンドを実行する
- 上下キーで選択中のコマンドを移動する
- Escapeで閉じる
- モーダル外側をクリックした場合も閉じる
- コマンド実行後は原則として閉じる
- 検索コマンドを実行した場合、ファイルサイドバーが閉じていれば自動で開き、検索欄へフォーカスする
- 設定コマンドを実行した場合、サイドバーを設定表示へ切り替える。左レールの設定ボタンとは異なり、メインエリアの設定パネルタブは開かない
- アクティブファイル名変更の導線はファイルサイドバーを開く。実際のインラインリネームはファイルツリー、またはファイルタブ本文上部のファイルタイトルで行う

---

## クイックスイッチャー

ファイル名でノートをすばやく開くUI。

### 起動

- macOSでは `⌘P`、Windows/Linuxでは `Ctrl+P` で開く
- コマンドパレットからも開ける
- メインエリア上にモーダルとして表示する

### 表示内容

- 入力欄
- 入力に一致するファイル一覧
- ファイル名
- 一致した `aliases` がある場合は、該当alias
- 同名ファイルがある場合はフォルダパス
- 一致するファイルがない場合は、候補なしの表示を出す

### 対象ファイル

- 現在のワークスペース内の `.md` ファイルのみを対象にする
- サイドバーに表示しないファイル形式は対象外にする
- ファイル名、ワークスペース相対パス、フロントマター `aliases` を検索対象にする

### 挙動

- 入力に応じて候補を絞り込む
- 表示する候補は最大50件とする
- Enterで選択中のファイルを開く
- 上下キーで選択中のファイルを移動する
- Escapeで閉じる
- モーダル外側をクリックした場合も閉じる
- 既に開いているファイルを選択した場合は、そのタブをアクティブにする
- 開いていないファイルを選択した場合は、アクティブなペインの新しいタブで開く
- 分割表示中は、現在アクティブなペインのタブ列に開く

---

## コマンドパレットとクイックスイッチャーの関係

- コマンドパレットは「操作」を探す
- クイックスイッチャーは「ファイル」を探す
- 全文検索・タグ検索・正規表現検索は [search.md](search.md) の対象とし、この仕様には含めない
