# features/commands.md

コマンドパレットとクイックスイッチャーの仕様。

---

## 目的

キーボード操作で、ファイル操作とアプリ操作をすばやく実行できるようにする。

---

## コマンドパレット

アプリ内の主要操作を検索して実行するUI。

### 起動

- macOSでは `⌘⇧P`、Windows/Linuxでは `Ctrl+Shift+P` で開く
- メインエリア上にモーダルとして表示する
- サイドバーの表示状態に関係なく開ける
- コマンドパレットを開くときは、クイックスイッチャーを閉じる
- すでに開いている状態で同じショートカットを押すと閉じる

### 表示内容

- 入力欄
- 入力に一致するコマンド一覧
- 各コマンドの名前
- ショートカットがある場合は、利用中のOSに合わせたショートカット表記
- 一致するコマンドがない場合は、候補なしの表示を出す

### 対象コマンド

現行の対象コマンドは以下とする。

| コマンド | 表示条件 | 実行内容 |
|----------|----------|----------|
| 新規ファイル作成 | 常時 | ファイルサイドバーをファイル表示にし、新規ファイル作成状態にする |
| 検索 | 常時 | クイックスイッチャーを開く |
| クイックスイッチャー | 常時 | クイックスイッチャーを開く |
| サイドバー表示切り替え | 常時 | ファイルサイドバーの開閉状態を切り替える |
| 分割表示切り替え | 常時 | メインエリアの左右分割を切り替える |
| 右パネル表示切り替え | 常時 | 右パネルが利用可能な場合だけ右パネルの開閉状態を切り替える |
| 閉じたタブを開く | 閉じたタブの履歴がある場合 | 最後に閉じたタブを開く |
| タイプライターモード切り替え | 常時 | タイプライターモードのオン / オフを切り替える |
| アクティブファイル名変更導線 | フォーカス中ペインのアクティブタブがMarkdownファイルの場合 | ファイルサイドバーをファイル表示にする |
| アクティブファイル複製 | フォーカス中ペインのアクティブタブがMarkdownファイルの場合 | そのMarkdownファイルを複製する |
| アクティブファイル削除 | フォーカス中ペインのアクティブタブがMarkdownファイルの場合 | そのMarkdownファイルの削除確認を開く |
| 設定 | 常時 | サイドバーを設定表示へ切り替える |

### 挙動

- 入力に応じて候補を絞り込む
- 候補検索はコマンド名の部分一致で行い、大文字小文字は区別しない
- 入力内容が変わった場合は、選択中候補を先頭へ戻す
- Enterで選択中のコマンドを実行する
- 上下キーで選択中のコマンドを移動する
- 候補にマウスカーソルを重ねた場合も、その候補を選択状態にする
- Escapeで閉じる
- モーダル外側をクリックした場合も閉じる
- 閉じるときは約130msの退場表示を出してから画面から消す。退場表示中に追加の閉じる操作を受けても二重には処理しない
- コマンド実行後は原則として閉じる
- 新規ファイル作成コマンドは、ワークスペースが開かれていない場合でもコマンド自体は表示する。ただしファイルサイドバー本体はワークスペース未選択時の案内を表示するため、実際のファイル作成入力はワークスペースを開くまで表示されない
- 右パネル表示切り替えコマンドは常にコマンド一覧へ表示し、アウトライン、リンク、復元版を含む右パネル全体を開閉する
- 検索コマンドを実行した場合、クイックスイッチャーを開く
- 設定コマンドを実行した場合、サイドバーを設定表示へ切り替える。左レールの設定ボタンとは異なり、メインエリアの設定パネルタブは開かない
- アクティブファイル名変更の導線はファイルサイドバーを開く。実際のインラインリネームはファイルツリー、またはファイルタブ本文上部のファイルタイトルで行う

## クイックスイッチャー

ファイル名でノートをすばやく開くUI。

### 起動

- macOSでは `⌘P`、Windows/Linuxでは `Ctrl+P` で開く
- コマンドパレットからも開ける
- ファイルサイドバー上部の検索ボタンからも開ける
- メインエリア上にモーダルとして表示する
- クイックスイッチャーをショートカットで開くときは、コマンドパレットを閉じる
- すでに開いている状態で同じショートカットを押すと閉じる

### 表示内容

- 入力欄
- 入力に一致するファイル一覧
- ファイル名
- 一致した `aliases` がある場合は、該当alias
- 同名ファイルがある場合はフォルダパス
- 一致するファイルがない場合は、候補なしの表示を出す

### 対象ファイル

- 現在のワークスペース内の `.md` ファイルのみを対象にする
- サイドバーに表示しないファイル形式は対象外にする
- ファイル名、ワークスペース相対パス、フロントマター `aliases` を検索対象にする

### 挙動

- 入力に応じて候補を絞り込む
- 候補検索はファイル名、ワークスペース相対パス、`aliases` の部分一致で行い、大文字小文字は区別しない
- 表示する候補は最大50件とする
- 同じファイル名の候補が複数ある場合は、候補行に親フォルダパスを表示する
- 入力に一致した `aliases` がある場合は、親フォルダパスよりも一致aliasを優先して表示する
- 入力内容が変わった場合は、選択中候補を先頭へ戻す
- Enterで選択中のファイルを開く
- 上下キーで選択中のファイルを移動する
- 候補にマウスカーソルを重ねた場合も、その候補を選択状態にする
- Escapeで閉じる
- モーダル外側をクリックした場合も閉じる
- 閉じるときは約130msの退場表示を出してから画面から消す。退場表示中に追加の閉じる操作を受けても二重には処理しない
- 既に開いているファイルを選択した場合は、そのタブをアクティブにする
- 開いていないファイルを選択した場合は、アクティブなペインの新しいタブで開く。アクティブなタブがある場合は、その右側に追加する
- 分割表示中は、現在アクティブなペインのタブ列に開く

## コマンドパレットとクイックスイッチャーの関係

- コマンドパレットは「操作」を探す
- クイックスイッチャーは「ファイル」を探す
- コマンドパレットとクイックスイッチャーは同時に表示しない
- 全文検索・タグ検索は [search.md](search.md) の対象とし、この仕様には含めない
