# ナビゲーション仕様

Relicのナビゲーションは、左レール、ファイルサイドバー、メインエリアのタブで構成する。

この文書では、各ナビゲーション要素が「何をできるか」を定義する。
画面内の配置は [../design/screens.md](../design/screens.md)、画面遷移やビュー切り替えの流れは [../design/navigation.md](../design/navigation.md) を正とする。

## 左レール

- ファイルボタンはファイルサイドバーを開閉する
- ファイル加工、フロントマター設定、暦設定、設定は、メインエリアのパネルタブとして開く
- チャートは、メインエリアのチャートタブとして開く
- 既に同じパネルタブまたはチャートタブが開いている場合は、そのタブをアクティブにする
- 左レールからパネルタブを開く場合、ファイルサイドバーの開閉状態や表示中ビューは基本的に変更しない
- 左レールからチャートタブを開く、または既存チャートタブをアクティブにする場合、ファイルサイドバーを閉じる
- パネルタブまたはチャートタブが開いているが非アクティブの場合、左レールの入口は開いている状態として表示する
- パネルタブまたはチャートタブがアクティブな場合、左レールの入口はアクティブ状態として表示する
- 新しくパネルタブまたはチャートタブを開く場合、左レールからタブ列へ短いオープン表示を出す。既存タブをアクティブにするだけの場合は出さない
- ファイルと設定は常に表示する
- ファイル加工、フロントマター設定、暦設定、年表、カレンダーは、対応する機能トグルがオフの場合は表示しない
- 初期状態で表示する入口は、ファイル、カレンダー、設定とする
- ファイル加工、フロントマター設定、暦設定、年表は、初期状態では非表示とする

## タイトルバー

- タイトルバーには、ペインごとのタブ列とタブ行アクションを表示する
- macOSのウィンドウ操作領域と左レール・ファイルサイドバーに重ならないよう、タブ列の左開始位置を調整する
- 右パネルを開いている場合、タブ行アクション領域は右パネル幅を考慮した幅で表示する
- タブ行アクションには、印刷、PDF保存、ソースモード切り替え、分割表示切り替え、右パネルのアウトライン/リンク切り替えを表示する
- 印刷とPDF保存は、フォーカス中ペインのアクティブタブがMarkdownファイルの場合だけ実行できる
- ソースモード切り替えと分割表示切り替えは、オンの場合にアクティブ状態として表示する
- 右パネルのアウトライン/リンク切り替えは、対象ビューが開いている場合にアクティブ状態として表示する
- 右パネル機能トグルがオフの場合、アウトライン/リンク切り替え操作はタイトルバーに表示しない

## ファイルサイドバー

- ファイル・フォルダをツリー表示する
- ワークスペースの作成、登録を行う
- ファイル名、全文、タグ、フロントマターフィールド、正規表現で検索できる
- macOSでは `⌘B`、Windows/Linuxでは `Ctrl+B`、またはボタンで開閉できる
- サイドバー幅はユーザーがドラッグで変更できる。最小幅は180px、最大幅は500pxとする
- 検索コマンドを実行した場合は、ファイルサイドバーを開いて検索入力へフォーカスする
- ワークスペース未選択時は、ファイルツリーや検索ではなく、既存フォルダを開く操作と新規ワークスペース作成操作を表示する
- ワークスペース未選択時は、既存フォルダを開く操作を主要操作として表示し、新規ワークスペース作成操作を副次操作として表示する
- ワークスペース選択中は、ファイル作成、フォルダ作成、全フォルダ展開、全フォルダ折りたたみをファイルサイドバー上部の操作として表示する
- ファイル作成中またはフォルダ作成中は、該当する作成操作を一時的に無効にし、処理中の表示にする
- ワークスペースを開く処理中または作成処理中は、ワークスペースを開く操作と新規ワークスペース作成操作を一時的に無効にし、実行中の操作名を表示する

## ピン留め

- ファイル・フォルダ両方をピン留めできる
- ピン留めしたアイテムはファイルサイドバー上部に固定表示する

## タブ

- ファイルタブ、パネルタブ、チャートタブを開ける
- 枚数制限なしで開ける
- タブが多くなると横にスクロールして切り替える
- 同じMarkdownファイルを再度開く場合は、同じパスのファイルタブを重複作成せず、既存タブを開いたペインへ追加してアクティブにする
- パネルタブは `panel-...`、チャートタブは `chart-...` の安定したIDで扱い、同じ画面を開く場合は既存タブをアクティブにする
- タブはドラッグ&ドロップで同じペイン内の前後、または分割表示中の別ペインへ移動できる
- タブを別ペインへ移動した場合は、移動先ペインをフォーカスし、移動したタブをアクティブにする
- タブを閉じた後は、表示履歴ベースで直前に表示していたタブをアクティブにする
- タブ右クリックメニューから、タブを閉じる、他のタブを閉じる、右側のタブを閉じる、すべてのタブを閉じる操作ができる
- タブ右クリックメニューから、タブをピン留め・解除できる
- ピン留めしたタブにはピンアイコンを表示し、タブ列の先頭側に並べる
- 「他のタブを閉じる」「右側のタブを閉じる」「すべてのタブを閉じる」は、ピン留めしたタブを残す
- ファイルタブの右クリックメニューでは、複製、パスコピー、Markdownリンクコピー、OS上の場所表示もできる
- 分割表示中は、タブ右クリックメニューから別ペインで開ける
- タブが0枚になった場合は、新規ファイル作成画面を表示する

## 分割表示

- 左右2ペインのみ対応
- macOSでは `⌘\`、Windows/Linuxでは `Ctrl+\`、またはボタンで分割・解除できる
- 分割表示中は、ペインごとに独立したタブ列を持つ
- ファイルタブ、パネルタブ、チャートタブはいずれもペイン内のタブとして扱う
- 分割表示を開始するとき、左ペインでアクティブなタブがある場合は、同じタブを右ペインにも表示してアクティブにする
- 分割表示を開始するとき、左ペインにアクティブタブがない場合は、右ペインを空のまま分割状態にする
- 分割表示を解除するときは、右ペインだけにあるタブを左ペインへ統合し、右ペインの最後のアクティブタブが左ペイン側に存在しない場合はそのタブを解除後のアクティブタブにする
- 左右両方に表示されている共有タブを片方のペインで閉じても、もう片方のペインに残っている間はタブ本体を削除しない

## 右パネル

- ファイルタブの右側に開閉可能な右パネルを表示する
- 右パネルには、アウトライン / リンク一覧のどちらか一方だけを表示する
- アウトラインまたはリンクのボタンを押すと、そのビューを選んだ状態で右パネルを開く。すでに同じビューで開いている場合は右パネルを閉じ、最後に選んだビューは保持する
- アウトラインでは、見出し（H1〜H6）の一覧を表示し、クリックで該当箇所にジャンプできる
- リンク一覧では、バックリンク・アウトゴーイングリンクを表示する
- リンク一覧の各リンクは、クリックでリンク先または参照元ファイルを開く
- リンク一覧の各リンクは、右クリックメニューから開く、別ペインで開く、Markdownリンクコピー、パスコピー、OS上の場所表示ができる
- 未作成のアウトゴーイングリンクは、リンクとして表示しつつ未作成であることを示す
- アウトゴーイングリンクは最大1,000件まで表示し、上限に達した場合は一部のみ表示していることをリンク一覧に表示する
- バックリンク読み込み中は、読み込み中表示を出す
- 大きいMarkdownとしてソース表示に倒したファイルでは、編集性能を優先してアウトライン、アウトゴーイングリンク、バックリンクの抽出を行わない
- macOSでは `⌘⇧B`、Windows/Linuxでは `Ctrl+Shift+B`、またはボタンで開閉できる
- 右パネル機能トグルがオフの場合は、タブ行アクションの右パネル切り替え操作を表示しない
- 右パネル機能トグルがオフになった場合、表示中の右パネルも閉じる
- 右パネル機能トグルがオフの場合、ショートカットでも右パネルを表示しない
- 右パネル幅はユーザーがドラッグで変更できる。最小幅は220px、最大幅は520pxとする

## ワークスペース

- 新規ワークスペース作成時は、毎回ユーザーが保存場所を選ぶ
- 登録済みワークスペースは左レール下部から切り替える
- 登録済みワークスペースが0件でもアクティブワークスペースがある場合は、そのワークスペースを左レール下部に表示する
- 登録済みワークスペースもアクティブワークスペースもない場合は、左レール下部のワークスペース一覧を表示しない
- 登録済みワークスペースがあるがアクティブワークスペースがない場合は、左レール下部に登録済みワークスペース一覧を表示し、ファイルサイドバー本体にはワークスペース未選択時の案内を表示する
- ワークスペース項目には、ワークスペース名の先頭1文字を大文字にしたアイコンを表示する。名前が空の場合は `W` を表示する
- ワークスペース項目には、OS上のパスをツールチップとして表示する
- アクティブなワークスペース項目は、選択中として表示する
- ワークスペース切り替え時は、そのワークスペースの登録情報、ファイルツリー、ピン留めなどのワークスペース状態を読み込む
- ただし、別ワークスペースを開く・作成する・切り替える操作では、前ワークスペースの開きタブを閉じる
- 登録済みワークスペースの表示名は、左レール上のダブルクリック、または右クリックメニューから直接変更できる
- ワークスペース名変更は、入力欄からフォーカスが外れたとき、またはEnterで確定する
- ワークスペース名変更中にIME変換している場合、Enterでは確定しない
- ワークスペース名変更中にEscapeを押した場合は取り消す
- ワークスペース名変更で空文字、または変更前と同じ名前になった場合は、保存せず編集状態だけを終了する
- ワークスペース名変更中と変更成功直後の短い時間は、左レールを展開した状態で保つ
- 登録済みワークスペースは、左レールの右クリックメニュー、または項目横の削除ボタンから登録一覧だけ削除できる
- ワークスペース右クリックメニューは、メニュー外クリックまたはEscapeで閉じる

## 通知とオーバーレイ

- コマンドパレット、クイックスイッチャー、リンク右クリックメニュー、短いオープン表示、短い作成表示、トースト通知は、メイン画面上のオーバーレイとして表示する
- リンク右クリックメニューは、メニュー外クリックまたはEscapeで閉じる
- リンク右クリックメニューには、開く、別ペインで開く、Markdownリンクコピー、パスコピー、OS上の場所表示を表示する
- リンク右クリックメニューの別ペインで開く操作は、分割表示中だけ表示する
- トースト通知は、エラー通知と情報通知を表示できる
- トースト通知は、表示後約4秒で閉じ始め、約130msの終了表示後に画面から消える
- トースト通知をクリックした場合は、自動終了待ちを取り消して同じ終了表示で閉じる
- 新しいトースト通知を表示する場合は、既存の表示中・終了中タイマーを取り消し、新しい通知を表示する

## 表示・テーマ

- ダーク／ライトモードはデフォルトでOSの設定に自動追従する
- アプリ設定でダーク／ライトを手動固定することも可能
- テーマがシステム追従の場合、ルート要素の `data-theme` は付けず、CSSの `prefers-color-scheme` によってライト/ダークを切り替える
- テーマをライトまたはダークに固定した場合は、ルート要素の `data-theme` に `light` または `dark` を設定する
- アクセントカラーのユーザー指定は持たず、Relic側で定義した統一デザインのみを使用する
- エディタのフォントはアプリが用意したプリセット数種から選択できる

## キーボードショートカット

ショートカットの主修飾キーは、macOSでは `⌘`、Windows/Linuxでは `Ctrl` とする。画面上のショートカット表記も利用中のOSに合わせて出し分ける。
実行環境がMac系プラットフォームとして判定される場合は `Meta` キーを、それ以外では `Ctrl` キーを主修飾キーとして扱う。

| 操作 | macOS | Windows/Linux |
|------|-------|---------------|
| 検索 | `⌘F` | `Ctrl+F` |
| コマンドパレット | `⌘⇧P` | `Ctrl+Shift+P` |
| クイックスイッチャー | `⌘P` | `Ctrl+P` |
| サイドバー開閉 | `⌘B` | `Ctrl+B` |
| 新規ファイル | `⌘N` | `Ctrl+N` |
| タブを閉じる | `⌘W` | `Ctrl+W` |
| 分割表示 | `⌘\` | `Ctrl+\` |
| 右パネル開閉 | `⌘⇧B` | `Ctrl+Shift+B` |
| タイプライターモード | `⌘⇧T` | `Ctrl+Shift+T` |

## 設定

- 設定パネルでは、テーマ、表示言語、エディタフォント、フォントサイズ、行間、エディタ最大幅、行番号、スペルチェック、フロントマター日付入力順を変更できる
- 設定パネルの変更は、保存ボタンを押す形式ではなく、変更操作ごとに保存を試みる
- テーマはシステム追従、ライト、ダークから選ぶ。初期値はシステム追従とする
- 表示言語はシステム追従、英語、日本語から選ぶ。初期値は英語とする
- 表示言語をシステム追従にした場合、OSまたはブラウザ環境の言語コードが `ja` で始まると日本語、それ以外は英語として表示する
- 翻訳キーが選択中言語に存在しない場合は英語文言へ、それも存在しない場合は翻訳キー文字列そのものへフォールバックする
- エディタフォントの初期値はシステムフォント、フォントサイズは16px、行間は1.7、エディタ最大幅は標準660pxとする
- 行番号表示の初期値はオフ、スペルチェックの初期値はオン、フロントマター日付入力順の初期値は `YYYY-MM-DD` とする
- 機能トグルでは、ファイル加工、フロントマター設定、暦設定、年表、カレンダー、右パネルの表示を切り替えられる
- 初期値は、カレンダーと右パネルをオン、ファイル加工、フロントマター設定、暦設定、年表をオフとする
- 設定パネルには、アプリバージョンと実行プラットフォームを表示する
- 壊れた設定ファイル、配列などオブジェクトではない設定ファイルは、アプリ起動を止めず初期設定として扱う
- 壊れたエディタ数値設定や不明な選択値は、読み込み時に項目ごとの初期値へ戻す
- ワークスペース登録一覧を読み込む場合、IDが安全な文字列でないもの、名前が空のもの、パスが正規化済み絶対パスでないもの、重複IDは除外する
- 前回ワークスペースIDは、読み込み後の登録済みワークスペース一覧に存在する場合だけ復元し、存在しない場合はアクティブワークスペースなしとして扱う
- 機能トグル保存では、全項目が真偽値でそろっている場合だけ保存する
- 設定保存は安全書き込みで行う
- テーマ、表示言語、エディタ設定、機能トグル、カスタムフロントマター定義、フロントマターテンプレート、ワークスペース登録一覧はアプリ全体設定として保存する
- 暦設定、チャート設定、ファイルツリーのピン留めはワークスペース別設定として保存する
- ワークスペース別設定ファイルが壊れている、存在しない、または配列などオブジェクトではない場合、そのワークスペースの暦設定・チャート設定・ピン留めは初期値として扱う
- ワークスペース別設定IDは英数字、 `_`、`-` だけを有効とし、危険なIDでは設定ファイルのパスを作らない。読み込み時は初期設定として扱い、保存時は失敗させる
- ワークスペース別設定のピン留めパスとチャート対象パスは、読み込み時に前後空白と区切り文字を正規化し、ワークスペース相対パスとして安全なものだけを重複除去して復元する
- 旧 `ganttCharts` キーが残っている場合はチャート設定として読み込むが、保存時は現行の `charts` キーだけを書き込む
- チャート設定は現行では年表と日付チャートの固定2件として復元し、保存済み表示名ではなく現行の固定チャート名を使う
