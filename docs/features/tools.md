# features/tools.md

ファイル加工ツールの仕様書。既存ファイルを変更せず、結果は必ず新規ファイルとして出力する。

**原則：既存ファイルは読み取りのみ。操作の結果はすべて新規ファイルとして作成する。**

---

## フォルダ内マージ

指定した条件に一致するファイルを1つのファイルに結合して出力する。

- 絞り込み条件：タグ・フロントマターのフィールド値・フォルダ・すべて
- 絞り込み条件の初期値は「すべて」とする
- フロントマター絞り込みでは、フィールド名と値を指定する（例：`status` が `draft` のファイルだけ）
- タグ絞り込みでは、入力値の先頭に `#` があってもタグ値として扱う
- フロントマター値が真偽値の場合、`true` / `false` のほか、`1` / `0`、`yes` / `no`、`on` / `off` でも照合できる
- 結合順序：ファイル名順・更新日時順・作成日時順から選択
- 結合順序の初期値はファイル名順とする
- ファイル名順はワークスペース相対パスの昇順、更新日時順と作成日時順は新しい順とする
- 各ファイルの区切りとして、ファイル名を見出しとして挿入するかどうかを選択できる
- ファイル名見出しの挿入は初期状態でオンとする
- ファイル名見出しを挿入する場合、各ファイルの直前に拡張子を除いたファイル名をH1見出しとして入れる
- 結合時は各ファイル本文の前後空白を取り除き、ファイル間は水平線 `---` で区切る
- 出力先フォルダとファイル名を指定して新規ファイルとして保存する
- 出力ファイル名が空の場合は、既定名「マージ結果」を使う
- 出力先に同名ファイルが存在する場合は上書きせず、別名で保存する
- 絞り込み候補を作る段階で読めないMarkdownファイルがある場合は、その候補だけを除外して続行する
- 結合対象として確定したファイル本文を読み込めない場合は、安全側で処理を失敗させる

---

## 見出しで分割

1つのファイルを指定した見出しレベルで複数ファイルに切り出して出力する。

- 分割の基準とする見出しレベルを指定（H1 / H2 / H3）
- 見出しレベルの初期値はH2とする
- 各見出しのテキストをもとにファイル名を生成する
- ファイル名に使えない文字は `_` に置き換える
- ファイル名の空白は連続していても1つの空白にまとめ、前後の空白を取り除く
- ファイル名が空になる場合は `untitled` を使う
- 元のファイルは変更しない
- 出力先フォルダを指定して新規ファイルとして保存する
- 分割先ファイル名は各見出しから作るため、画面上に出力ファイル名入力は持たない
- 分割結果には対象見出し行を含め、最初の対象見出しより前にある本文は出力しない
- 指定した見出しレベルの見出しがない場合は、ファイルを作成せずエラーを表示する
- 分割元ファイルが未指定の場合は処理を実行しない
- 分割元ファイルの実体がワークスペース外のシンボリックリンクである場合は処理しない

## タイトル一覧

条件で絞り込んだファイルのタイトルをリスト化して新規ファイルに出力する。

- 絞り込み条件：フォルダ（指定なしの場合はワークスペース全体）
- 絞り込みフォルダの初期値は空欄とし、ワークスペース全体を対象にする
- 並び順：ファイル名順・更新日時順から選択
- 並び順の初期値はファイル名順とする
- ファイル名順は拡張子を除いた表示名の昇順、更新日時順は新しい順とする
- 出力形式：箇条書き（内部リンク付き）
- 例：`- [[ファイル名]]`
- 同じファイル名のMarkdownがワークスペース内に1つだけの場合は、拡張子を除いたファイル名だけの短い内部リンクにする
- 同じファイル名のMarkdownがワークスペース内に複数ある場合は、`[[./フォルダ/ファイル名|ファイル名]]` のようにワークスペースルートからのパスを含め、対象ファイルを一意に指すリンクにする
- リンクの表示名は、拡張子を除いたファイル名だけを使う
- 出力先フォルダとファイル名を指定して新規ファイルとして保存する
- 出力ファイル名が空の場合は、既定名「タイトル一覧」を使う
- 候補作成中に一部のMarkdownファイル情報を取得できない場合は、その候補だけを除外して続行する

---

## 目次生成

フォルダ内のファイルを内部リンク一覧として出力する。フォルダの索引ファイルとして使える。

- 対象フォルダを指定する
- サブフォルダを含めるかどうかを選択できる
- 対象フォルダの初期値は空欄とし、ワークスペースルートを対象にする
- サブフォルダを含める設定は初期状態でオンとする
- 出力形式：フォルダ階層を反映したネストリスト（内部リンク付き）
- 目次内ではフォルダをファイルより先に並べ、同じ階層内では名前順に並べる
- フォルダ行は `**フォルダ名/**` として表示し、ファイル行は内部リンクとして表示する
- 同じファイル名のMarkdownがワークスペース内に1つだけの場合は、拡張子を除いたファイル名だけの短い内部リンクにする
- 同じファイル名のMarkdownがワークスペース内に複数ある場合は、`[[./フォルダ/ファイル名|ファイル名]]` のようにワークスペースルートからのパスを含め、対象ファイルを一意に指すリンクにする
- リンクの表示名は、拡張子を除いたファイル名だけを使う
- 出力先フォルダとファイル名を指定して新規ファイルとして保存する
- 出力ファイル名が空の場合は、既定名「目次」を使う
- 既存の目次ファイルは自動更新しない
- 出力先に同名ファイルが存在する場合は、別名で新規作成する
- サブフォルダを含める指定で一部の子フォルダを読めない場合は、その子フォルダだけを除外して続行する
- 対象フォルダ自体が読めない場合は、安全側で処理を失敗させる

## 備考

- 対象機能は上記の4種類とする
- ファイル加工パネルでは、タイトル一覧生成、目次生成、フォルダ内マージ、見出しで分割の順に表示する
- ワークスペースが開かれていない場合、各ツールの入力欄ではなくワークスペースを開く案内を表示する
- 追加の加工ツールを検討する場合も、この仕様とは別作業として扱う
- 新しい加工ツールを追加する際も「既存ファイルを変更しない」原則を守ること
- 既定名補完後の出力ファイル名は、パス区切り、絶対パス、`.`、`..`、空の名前を受け付けない
- 出力ファイル名に `.md` が含まれる場合も、Markdownファイルとして扱い、必要に応じて末尾を `.md` にそろえる
- 出力先フォルダ、対象フォルダ、分割元ファイル、フォルダ絞り込みは、空文字または正規化済みのワークスペース相対パスだけを受け付ける
- 出力先がワークスペース外の実体を指すシンボリックリンクの場合は保存しない
- 出力先に同名候補が大量に存在する場合、無制限に候補探索を続けずエラーとして停止する
- 出力保存は安全書き込みで行う
