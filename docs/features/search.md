# 検索仕様

## 呼び出し方

- 現行UIでは、ファイルサイドバー上部に常時表示する検索欄は置かない
- ファイルサイドバー上部の検索ボタンまたは `⌘P` は、ファイル名・パス・別名で開くファイルを探すクイックスイッチャーを開く
- クイックスイッチャーは [commands.md](commands.md) の対象とし、全文検索、タグ検索、フロントマター検索とは別機能として扱う
- 全文検索、ファイル名検索、タグ検索、フロントマター検索の処理は、IPC経由で利用できる内部検索処理として保持する
- 検索語句が空、またはワークスペース未選択の場合は検索を実行せず、検索結果・上限通知・検索エラーを空に戻す
- レンダラーからの通常検索要求は、`query`、`mode`、必要に応じて `frontmatterField` を持つ構造化入力として扱う
- IPC境界では現行の構造化入力だけを受け付け、検索語句または検索種別が不正な場合、または `frontmatterField` が文字列でも未指定でもない場合は検索を実行しない

## 検索結果の表示

- 内部検索処理をUIから呼び出す導線では、ファイルサイドバー内に検索結果一覧を表示する
- ファイル名と一致内容を表示し、クリックで該当ファイルを開く
- 全文検索では一致行番号と一致行を表示する
- ファイル名検索、タグ検索、フロントマター検索では該当ファイル名と一致理由を表示する
- 検索中は検索結果領域に読み込み中の表示を出す
- 検索エラーがある場合は、検索結果よりエラー表示を優先する
- 検索結果がない場合は、一致なしの表示を出す
- プロパティ検索で検索語句があり、プロパティ名が未選択の場合は、プロパティ名の入力を促す表示を出す
- 検索結果を開いている途中のファイルは、検索結果上で開いている状態のスタイルを付ける
- 検索結果は最大500件まで表示し、上限に達した場合は結果が省略されたことを表示する
- 2MiBを超えるMarkdownファイルは検索対象外にし、スキップした件数を表示する。2MiBちょうどのMarkdownファイルは検索対象に含める
- サイズ確認や本文読み込みに失敗したMarkdownファイルがあっても検索全体は失敗させず、そのファイルだけをスキップして続行する
- 全文検索で一致した行が空行の場合、結果の一致内容は `(空行)` と表示する

## 対応する検索種別

- 検索対象は現在開いているワークスペース内のみ。複数ワークスペースをまたいだ検索は非対応。
- 検索対象ファイルはワークスペース内のMarkdownファイルだけとする
- 全文検索：フォルダ内のMarkdownファイルを大文字小文字を区別せずにキーワード検索する。**フロントマターも検索対象に含む**
- ファイル名検索：Markdownファイル名またはフロントマター `aliases` で、大文字小文字を区別せずに絞り込む
- タグ検索：フロントマター `tags:` の値で絞り込む。検索語句の先頭に `#` がある場合は取り除いて照合する
- フロントマター検索：プロパティ名と検索語句を指定し、対象プロパティの値で大文字小文字を区別せずに絞り込む
### フロントマター検索

- フロントマター検索では、検索語句とは別にプロパティ名を指定する
- プロパティ名は、固定プロパティから選ぶ。ワークスペース内Markdownに偶然存在する未登録フィールドは候補にしない
- 選択中のプロパティ名が固定プロパティに存在しなくなった場合は、選択を空に戻す
- 検索語句は、対象プロパティの既存値候補がある場合は入力候補として表示する
- 候補値はワークスペース内の既存値を重複除去してまとめ、空文字は除外する
- UIから検索できるプロパティは、固定プロパティに限る
- IPC経由で未登録プロパティ名を指定された場合は、検索エラーにせず結果0件として扱う
- 対象プロパティの値が配列の場合は、配列内のいずれかの値に検索語句が含まれれば一致とする
- 対象プロパティの値が真偽値の場合、`true` / `1` / `yes` / `on` は真、`false` / `0` / `no` / `off` は偽として照合する
- 対象プロパティが存在しない、または値が `null` の場合は一致しない

## 置換

現行UIには、検索結果画面から一括置換を実行する通常操作は置かない。

内部実装として、単一ファイル置換、置換プレビュー、一括置換の処理を持つ。

- 単一ファイル置換は、対象がMarkdownファイルで、検索語句が空でない場合だけ実行する
- 置換は通常文字列または正規表現で実行できる
- 正規表現が無効な場合は、置換を行わずエラーを返す
- 単一ファイル置換と一括置換の書き込みは安全書き込みで行う
- 置換プレビューでは、読めないMarkdownファイルがあってもプレビュー全体を失敗させず、そのファイルだけをスキップする
- 置換プレビューの表示は最大500件とするが、一致ファイルの競合検知用スナップショットはワークスペース走査の最後まで収集する。`truncated` は表示結果が省略された場合だけを表す
- 一括置換の実書き込みでは、読み込みや書き込みに失敗した場合は安全側で処理を失敗させる
