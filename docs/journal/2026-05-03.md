# 2026-05-03

## やったこと

### クラウド同期フォルダの位置づけを整理

- 「iCloud連携」という表現を廃止し、Lorebooksは各クラウドサービスのAPI連携を持たない方針を明記
- iCloud Drive / OneDrive / Dropbox などは、OS上の通常のローカルフォルダとしてワークスペースにできる、という扱いに統一
- AIが「iCloud機能を内蔵している」と誤読しやすい入口文書・技術スタック・アーキテクチャ図・索引の記述を修正

### P0：技術選定（全4問を解消）

- **パッケージマネージャ** → pnpm に決定
- **ビルド・配布ツール** → Electron Forge に決定
- **テストフレームワーク** → Vitest に決定
- **検索インデックス** → インメモリ再構築（起動ごとに作り直す）に決定

### P1：ファイル管理（全7問を解消）

- **削除時の確認ダイアログ** → 確認あり（「○○をゴミ箱に移動しますか？」）に変更
- **ファイル名の禁止文字・重複** → どちらもエラーを出して別名入力を促す
- **`.md` 拡張子の扱い** → 自動付与・表示上は隠す。サイドバーは `.md` のみ表示（PDF等は非表示）
- **複製時の命名規則** → 「ファイル名 のコピー」（既存仕様と一致を確認）
- **リネーム・移動時のリンク自動更新** → 確認なし・即時。本文＋フロントマター対象、コードブロック除外
- **テンプレートフォルダの表示** → ツリーに通常フォルダとして表示

### P1：内部リンク・タグ（全6問を解消）

- **同名ファイルの解決方法** → 補完でパス付きリンクを両方提示し、ユーザーが選択
- **パス指定リンク** → `[[フォルダ/ファイル名]]` 記法をサポート
- **未作成リンクの新規作成先** → リンク元ファイルと同じフォルダ
- **バックリンクの位置づけ** → ナビゲーション補助として右パネルに表示（エクスポート対象外）
- **タグの種類** → 階層タグ（`#親/子`）・日本語タグともにサポート
- **本文タグとフロントマタータグの重複** → マージして1件として扱う。本文タグ機能は存続
