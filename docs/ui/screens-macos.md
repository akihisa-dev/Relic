# ui/screens-macos.md

macOS版の画面一覧と各画面の構成。

---

## 基本レイアウト原則

- **2カラム固定**：左サイドバー＋右メインエリア
- **メインエリアは絶対に遷移しない**：常にエディタを表示する。設定・検索・Gitなどの操作でメインエリアが切り替わることは禁止
- **サイドバーで完結**：ファイルツリー以外のあらゆる操作（設定・検索・Gitなど）はサイドバー内のビュー切り替えで行う

## サイドバー

左側の固定パネル。ビューを切り替えることで複数の役割を担う。

サイドバーのさらに左端に縦アイコンナビを置き、ファイルツリー・検索・Git・設定を切り替える。

- 各ビューはアイコンボタンで切り替える
- アイコンボタンにはツールチップを表示する
- サイドバー幅はユーザーがドラッグで変更できる
- サイドバーを閉じた状態で検索・Git・設定などを開いた場合は、サイドバーを自動で開いて対象ビューを表示する
- ボタン・タブ・ステータス表示・候補ラベルなどの短いUI要素は、原則として折り返さない。表示幅を超える場合は省略表示にし、必要な全文はツールチップや詳細領域で確認できるようにする

### ビュー一覧

| ビュー | 内容 |
|--------|------|
| ファイルツリー | ワークスペース内のフォルダ・ファイルをツリー表示。ピン留め・ワークスペース切り替えも含む |
| 検索 | 検索バーと検索結果一覧。ファイル名・全文・タグ・正規表現検索に対応 |
| Git | コミット履歴・Diff・ブランチ・Gitタグ・手動同期・コンフリクト解決 |
| 設定 | アプリの各種設定（下記参照） |

### 設定ビューの項目

- テーマ（ライト / ダーク / システム追従）
- エディタフォントの選択（プリセットから選択）
- フォントサイズ
- 行間
- エディタ最大幅
- 行番号表示のオン/オフ
- スペルチェックのオン/オフ
- テンプレートフォルダのパス
- GitHubアカウントの接続・切断
- 自動同期のオン/オフ・間隔設定
- 各機能のトグル（GitHub連携 / ファイル加工ツール / フロントマター / 右パネル / フォーカスモード・タイプライターモード）
- フロントマタースキーマ管理（ユーザー定義フィールドの追加・編集・削除。最大13個）
- フロントマター候補定義ファイル `frontmatter.md` の作成ボタン

## メインエリア

常にエディタを表示する。開閉可能な右パネルを持つ。

**固定表示：**
- ツールバー（上部）
- エディタ本体

**開閉可能な右パネル：**
- アウトラインパネル：見出し一覧とジャンプ
- リンクパネル：バックリンク・アウトゴーイングリンク一覧
- 右パネル内では、アウトライン / リンクのどちらか一方だけを表示する
- アウトラインボタン・リンクボタンで表示内容を切り替える

**分割表示時：**
- メインエリアが左右2ペインに分割される
- 各ペインは独立してファイルを表示できる
- タブはペインごとに管理する

## 初回起動時

ワークスペースが未登録の状態で起動したとき、サイドバーに初期設定ビューを表示する。

- 新規ワークスペースを作成（フォルダ名を入力）
- 既存フォルダをワークスペースとして開く
- GitHubリポジトリをクローンして開く

新規ワークスペース作成時は、毎回ユーザーが保存場所を選ぶ。
