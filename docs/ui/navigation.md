# ui/navigation.md

アプリの画面遷移・ビュー切り替えフローを定義するドキュメント。

**前提**：メインエリア（エディタ）は遷移しない。ナビゲーションはサイドバーのビュー切り替えとタブ操作が中心。

---

## 起動フロー

```
アプリ起動
  │
  ├─ ワークスペース登録済み → メイン画面（前回のファイルを復元）
  │
  └─ ワークスペース未登録 → サイドバーに初回設定ビューを表示
        │
        ├─ 新規ワークスペース作成
        ├─ 既存フォルダを開く
        └─ GitHubからクローン
              │
              └─ → メイン画面
```

---

## サイドバーのビュー切り替え

サイドバーは以下のビューを切り替えて表示する。メインエリアは変化しない。
ビュー切り替えは、サイドバー左端の縦アイコンナビで行う。

```
縦アイコンナビ
  ├─ ファイルツリー（デフォルト）
  ├─ 検索
  ├─ Git
  └─ 設定
```

サイドバーを閉じた状態で検索・Git・設定を開いた場合は、サイドバーを自動で開き、指定されたビューを表示する。

---

## ファイルを開く

```
サイドバー（ファイルツリー）でファイルをクリック
  │
  ├─ 新しいタブで開く（デフォルト）
  └─ 既に開いているタブがある場合 → そのタブをアクティブにする
```

---

## タブの操作

```
タブをクリック → そのファイルをメインエリアに表示
タブを閉じる  → 表示履歴ベースで直前に表示していたタブをアクティブにする
              → タブが0枚になった場合は新規ノート作成画面を表示
```

表示履歴は、タブをアクティブにした順に記録する。現在のタブを閉じたら、直近で表示していた未クローズのタブへ戻る。

---

## 分割表示

```
分割ボタンを押す → メインエリアが左右2ペインに分割
  │
  ├─ 左ペイン：元のファイルをそのまま表示
  └─ 右ペイン：新しいファイルを開ける状態になる

分割解除ボタンを押す → 1ペインに戻る（アクティブなペインのファイルを表示）
```

分割表示中は、ペインごとに独立したタブ列を持つ。

---

## 右パネルの開閉

```
アウトラインパネルボタンを押す → 右パネルにアウトラインを表示 / 閉じる
リンクパネルボタンを押す      → 右パネルにリンク一覧を表示 / 閉じる
```

右パネルでは、アウトラインとリンク一覧を同時表示しない。片方を開いている状態でもう片方を開いた場合は、右パネル内の表示内容を切り替える。

---

## 新規ワークスペース作成

```
新規ワークスペース作成を選択
  │
  ├─ フォルダ名を入力
  ├─ 保存場所を選択
  └─ 指定場所にワークスペースフォルダを作成して開く
```

保存場所は毎回ユーザーが選択する。

---

## ワークスペースの切り替え

```
サイドバー上部のワークスペース名をクリック
  │
  └─ ワークスペース一覧が展開
        │
        ├─ 別のワークスペースを選択 → そのワークスペースの前回の状態（タブ・分割・右パネル）を復元する
        └─ 新しいワークスペースを作成
```

---

## 内部リンクのジャンプ

```
エディタ内の [[リンク]] をクリック
  │
  ├─ リンク先が存在する → そのファイルを新しいタブで開く
  └─ リンク先が存在しない → 新規ファイルを作成して開く
```
