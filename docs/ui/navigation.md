# ui/navigation.md

アプリの画面遷移・ビュー切り替えフローを定義するドキュメント。

**前提**: ナビゲーションは左レール、カードサイドバー、メインエリアのタブ操作で行う。カード以外の作業画面も、メインエリアのタブとして開く。

---

## 起動フロー

```
アプリ起動
  │
  ├─ カードブック登録済み → メイン画面（前回のタブ状態を復元）
  │
  └─ カードブック未登録 → カードサイドバーに初回設定ビューを表示
        │
        ├─ 新規カードブック作成
        └─ 既存カードフォルダを開く
              │
              └─ → メイン画面
```

---

## 左レール

```
左レール
  ├─ カード        → カードサイドバーを開閉
  ├─ カード加工    → パネルタブを開く
  ├─ プロパティ  → パネルタブを開く
  ├─ 年表          → 年表タブを開く
  ├─ 年表設定      → パネルタブを開く
  └─ 設定            → パネルタブを開く
```

既に同じパネルタブまたは年表タブが開いている場合は、新規作成せずそのタブをアクティブにする。
カード加工とプロパティは、対応する機能トグルがオフの場合は左レールに表示しない。

---

## カードサイドバー

カードサイドバーは、カードブックとカードの操作を扱う。

```
カードサイドバー
  ├─ カードブック作成 / 登録
  ├─ カードツリー
  ├─ ピン留め
  └─ 検索
```

`⌘B` または左レールのカードボタンで開閉する。検索コマンドを実行した場合は、カードサイドバーを開き、検索入力へフォーカスする。

---

## カードを開く

```
カードサイドバーでカードをクリック
  │
  ├─ 新しいカードタブで開く
  └─ 既に開いているカードタブがある場合 → そのタブをアクティブにする
```

内部リンクをクリックした場合も、リンク先カードをカードタブとして開く。リンク先が存在しない場合は新規カードを作成して開く。

---

## タブの操作

```
タブをクリック → そのタブをアクティブにする
タブを閉じる  → 表示履歴ベースで直前に表示していたタブをアクティブにする
              → タブが0枚になった場合は新規カード作成画面を表示
```

表示履歴は、タブをアクティブにした順に記録する。現在のタブを閉じたら、直近で表示していた未クローズのタブへ戻る。

---

## 分割表示

```
分割ボタンを押す → メインエリアが左右2ペインに分割
  │
  ├─ 左ペイン：元のタブをそのまま表示
  └─ 右ペイン：独立したタブ列を持つ

分割解除ボタンを押す → 1ペインに戻る（アクティブなペインのタブを表示）
```

分割表示中は、ペインごとに独立したタブ列を持つ。カードタブ、パネルタブ、年表タブはいずれもペイン内のタブとして扱う。

---

## 右パネルの開閉

```
アウトラインパネルボタンを押す → 右パネルにアウトラインを表示 / 閉じる
リンクパネルボタンを押す      → 右パネルにリンク一覧を表示 / 閉じる
```

右パネルはカードタブの補助領域であり、アウトラインとリンク一覧を同時表示しない。片方を開いている状態でもう片方を開いた場合は、右パネル内の表示内容を切り替える。
右パネル機能トグルがオフの場合は、上部操作バーの右パネル切り替え操作を表示しない。

---

## カードブックの切り替え

```
左レール下部のカードブック切替を操作
  │
  └─ 別のカードブックを選択 → そのカードブックの前回の状態を復元する
```

---

## 新規カードブック作成

```
新規カードブック作成を選択
  │
  ├─ カードフォルダ名を入力
  ├─ 保存場所を選択
  └─ 指定場所にカードブック用ディレクトリを作成して開く
```

保存場所は毎回ユーザーが選択する。
