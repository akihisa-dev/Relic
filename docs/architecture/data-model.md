# architecture/data-model.md

アプリが扱う主なデータの構造を定義する。

---

## ワークスペース

ユーザーがアプリに登録した任意のローカルフォルダ1つ。保存場所はユーザーが自由に選択できる。

| フィールド | 内容 |
|-----------|------|
| name | フォルダ名（＝ワークスペース名） |
| path | ローカルのフォルダパス |
| gitRepo | 紐付けられたGitリポジトリの情報（未連携の場合はnull） |

---

## ファイル

ワークスペース内の `.md` ファイル1つ。

| フィールド | 内容 |
|-----------|------|
| name | ファイル名（拡張子なし） |
| path | ワークスペースルートからの相対パス |
| frontmatter | ファイル先頭のYAMLメタデータ（下記参照） |
| body | フロントマターを除いた本文テキスト |
| createdAt | 作成日時 |
| updatedAt | 最終更新日時 |

---

## フロントマター

ファイル先頭の `---` で囲まれたYAMLブロック。フィールドの詳細仕様は [spec/frontmatter.md](../spec/frontmatter.md) を参照。

| 種類 | フィールド例 | 内容 |
|------|------------|------|
| 組み込み | `tags`, `aliases` | アプリ機能と連動。削除・型変更不可 |
| 固定システム | `date`, `status`, `publish`, `url`, `author` | アプリが型と専用UIを定義 |
| ユーザー定義 | （ユーザーが命名） | 設定画面でフィールドを追加。最大13個 |
| 自由 | （任意） | ソースモードでYAMLに直接記述。型なし |

---

## フォルダ

ワークスペース内のディレクトリ。ファイルをグループ化する入れ物。

| フィールド | 内容 |
|-----------|------|
| name | フォルダ名 |
| path | ワークスペースルートからの相対パス |
| children | 中に含まれるファイル・フォルダの一覧 |

---

## 内部リンク

`[[ファイル名]]` 記法で定義されるファイル間の接続。

| フィールド | 内容 |
|-----------|------|
| sourceFile | リンクが書かれているファイル |
| targetName | リンク先として指定されたファイル名 |
| targetFile | 実際に存在するリンク先ファイル（存在しない場合はnull） |
| resolved | リンク先ファイルが存在するか否か |

---

## タグ

ファイルに付与されたラベル。本文の `#タグ名` またはフロントマターの `tags:` から収集する。

| フィールド | 内容 |
|-----------|------|
| name | タグ名（`#` を除いた文字列） |
| files | そのタグが付いているファイルの一覧 |

---

## テンプレート

テンプレートフォルダ内に置かれた `.md` ファイル。新規ファイル作成時に適用できる雛形。

| フィールド | 内容 |
|-----------|------|
| name | テンプレート名（ファイル名） |
| path | テンプレートフォルダからのパス |
| content | テンプレートの本文テキスト |

---

## Gitリポジトリ

ワークスペースに紐付けられたGitリポジトリの情報。

| フィールド | 内容 |
|-----------|------|
| localPath | ローカルのリポジトリパス（ワークスペースのパスと同じ） |
| remoteUrl | GitHubのリモートURL |
| currentBranch | 現在のブランチ名 |
| autoSync | 自動同期の設定（オン/オフ・間隔） |

---

## コミット

Gitのコミット1件。コミット履歴の表示・Diffビューで使用する。

| フィールド | 内容 |
|-----------|------|
| hash | コミットを一意に識別するID |
| message | コミットメッセージ |
| author | 作成者名 |
| date | コミット日時 |
| changedFiles | 変更されたファイルの一覧 |

---

## ブランチ

Gitのブランチ1件。

| フィールド | 内容 |
|-----------|------|
| name | ブランチ名 |
| isCurrent | 現在チェックアウト中かどうか |
| isRemote | リモート（GitHub側）にも存在するかどうか |
| lastCommit | 最新のコミット |

---

## Gitタグ

リリース時などにコミットに付けるラベル。`v1.0.0` のようなバージョン番号を付けるために使う。

| フィールド | 内容 |
|-----------|------|
| name | タグ名（例：`v1.0.0`） |
| commit | タグが付いているコミット |
| message | タグのメモ（任意） |

---

## アプリ設定

ユーザーの設定値。アプリのユーザーデータフォルダに保存する。

| フィールド | 内容 |
|-----------|------|
| theme | ライト / ダーク / システム追従 |
| editorFontSize | エディタのフォントサイズ |
| editorLineHeight | 行間 |
| editorMaxWidth | エディタ最大幅のプリセット |
| editorFont | エディタのフォント |
| lineNumbers | 行番号表示のオン/オフ |
| spellCheck | スペルチェックのオン/オフ |
| templateFolder | テンプレートフォルダのパス |
| lastWorkspace | 前回開いていたワークスペース |

---

## エンティティ間の関係

```
Workspace
  ├── Folder（ネスト可）
  │     └── File
  └── File
        ├── Frontmatter
        │     └── Tag（tags: フィールド）
        ├── Tag（本文中の #tag）
        └── InternalLink → File（リンク先）

Workspace
  └── GitRepository
        ├── Branch（複数）
        │     └── Commit（複数）
        └── GitTag → Commit

Template（テンプレートフォルダ内のFile）
```
