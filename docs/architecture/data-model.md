# architecture/data-model.md

アプリが扱う主なデータの構造を定義する。

---

## カードブック

ユーザーがアプリに登録した任意のローカルディレクトリ1つ。保存場所はユーザーが自由に選択できる。

| フィールド | 内容 |
|-----------|------|
| id | アプリ設定内でカードブックを識別するID |
| name | カードフォルダ名（＝カードブック名） |
| path | ローカルのカードフォルダパス |

### カードブック内カードフォルダ

Relicは、カードブック作成時または既存カードフォルダ登録時にRelic専用の必須カードフォルダを自動作成しない。ユーザーが作成した通常のディレクトリとカードをそのまま扱う。

---

## カード

カードブック内の `.md` カード1つ。IPCで読み書きするときは、カード名、カードブック相対パス、Markdown本文全体を扱う。

| フィールド | 内容 |
|-----------|------|
| name | カード名（拡張子なし） |
| path | カードブックルートからの相対パス |
| content | プロパティを含むMarkdown本文全体 |

プロパティや本文部分は、`content` から必要に応じて解析する派生データとして扱う。

---

## Markdown画像記法

Markdown本文に書かれた `![alt](path)` 記法。Relicは画像管理アプリではないため、画像カード単体の整理・タグ付け・検索対象にはしない。現行実装では、画像カードを実画像として読み込まず、ライブプレビューでは `alt` またはパスを使ったプレースホルダーとして扱う。

| フィールド | 内容 |
|-----------|------|
| raw | Markdown本文中の画像記法 |
| alt | Markdown画像記法に書かれた代替テキスト |
| pathOrUrl | Markdown画像記法に書かれた相対パスまたはURL |

---

## プロパティ

カード先頭の `---` で囲まれたYAMLブロック。アプリ設定が一律に付与する属性ではなく、ユーザーが任意のカードに後から追加・削除できるメタ情報として扱う。フィールドの詳細仕様は [spec/frontmatter.md](../spec/frontmatter.md) を参照。

| 種類 | フィールド例 | 内容 |
|------|------------|------|
| 登録済み | （ユーザーが命名） | Relicが入力能力を知っているフィールド。検索・絞り込み・入力補助に使う |
| 固定 | aliases / tags / status / timeline | Relicが意味を知っている予約フィールド。カスタムプロパティには追加しない |
| 未認識 | （任意） | 既存カードや外部エディタで作成されたYAMLキー。壊さず保持し、必要に応じて後から入力能力を割り当てる |

カスタムプロパティでは、ユーザーが任意のフィールド名に任意の入力能力を割り当てる。固定プロパティは、Relicがタグ、状態、Timelineに利用する。

固定プロパティも、デフォルトで各カードに作成しない。カード内に存在する場合に、Relicがその意味を認識して利用する。

---

## カードフォルダ

カードブック内のディレクトリ。カードをグループ化する入れ物。

| フィールド | 内容 |
|-----------|------|
| name | カードフォルダ名 |
| path | カードブックルートからの相対パス |
| children | 中に含まれるカード・カードフォルダの一覧 |

---

## 内部リンク

`[[カード名]]` 記法で定義されるカード間の接続。カードに保存されるのはMarkdown本文中のリンク記法であり、リンク一覧・バックリンクは本文から生成する派生データとして扱う。

| フィールド | 内容 |
|-----------|------|
| sourceCard | リンクが書かれているカード |
| targetName | リンク先として指定されたカード名 |
| targetCard | 実際に存在するリンク先カード（存在しない場合はnull） |
| resolved | リンク先カードが存在するか否か |

---

## タグ

カードに付与されたラベル。プロパティの固定プロパティ `tags:` から収集する。本文中の `#タグ名` はタグとして扱わない。

| フィールド | 内容 |
|-----------|------|
| name | タグ名（`#` を除いた文字列） |
| cards | そのタグが付いているカードの一覧 |

---

## アプリ設定

ユーザーの設定値。Electronの `userData` 配下の `app-settings.json` に保存する。

| フィールド | 内容 |
|-----------|------|
| editorSettings | テーマ、言語、エディタフォント、フォントサイズ、行間、最大幅、行番号、スペルチェック |
| featureToggles | カード加工ツール・プロパティ・右パネルのオン/オフ |
| userDefinedFields | Relicが入力能力を認識するプロパティフィールドの一覧 |
| frontmatterTemplates | アプリ設定に保存されるプロパティテンプレート定義（現行UIでは管理画面なし） |
| lastCardbookId | 前回開いていたカードブックのID |
| cardbooks | アプリに登録済みのカードブック一覧 |

---

## カードブック設定

カードブックごとに異なる設定値。Electronの `userData` 配下の `cardbooks/{cardbookId}.json` に、カードブックと紐付けて保存する。

| フィールド | 内容 |
|-----------|------|
| cardbookPath | 対象カードブックのパス |
| pinnedPaths | カードサイドバーでピン留めしているカードブック相対パス |
| timelineCharts | Timelineの対象カード設定 |

---

## タブと画面状態

タブ、ペイン、サイドバー、右パネルなどの画面状態はレンダラーのZustand storeで扱う。

| 種類 | 内容 |
|------|------|
| カードタブ | カードの `name`、`path`、`content` を持つ編集タブ |
| パネルタブ | カード加工、プロパティ設定、設定を表示するタブ |
| Timelineタブ | Timelineを表示するタブ |
| ペイン | 左右分割時のタブ列とアクティブタブ、表示履歴 |
| UI状態 | カードサイドバー、右パネル、タイプライターモードなどの開閉・選択状態 |

---

## Timeline表示用データ

Timelineはカードとプロパティから生成する派生データであり、Markdown本文とは別の永続データとしてカードブック内に保存しない。

| 種類 | 内容 |
|------|------|
| Timeline | `timeline` プロパティから生成する年または期間の表示行 |

## エンティティ間の関係

```
Cardbook
  ├── CardFolder（ネスト可）
  │     └── Card
  └── Card
        ├── Property
        │     └── Tag（tags: フィールド）
        └── InternalLink → Card（リンク先）
             └── Backlink / OutgoingLink（派生データ）

MarkdownImageSyntax
  └── Image card（Markdown本文中の記法。独立した永続モデルは持たない）
```
