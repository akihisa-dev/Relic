# architecture/data-model.md

アプリが扱う主なデータの構造を定義する。

---

## ワークスペース

ユーザーがアプリに登録した任意のローカルフォルダ1つ。保存場所はユーザーが自由に選択できる。

| フィールド | 内容 |
|-----------|------|
| name | フォルダ名（＝ワークスペース名） |
| path | ローカルのフォルダパス |
| gitRepo | 紐付けられたGitリポジトリの情報（未連携の場合はnull） |

### 必須フォルダ

Relicが機能上必要とするフォルダは、ワークスペース作成時または既存フォルダ登録時に自動作成する。ユーザーにフォルダ名や作成有無は選ばせない。

| フォルダ | 内容 |
|----------|------|
| `attachments/` | Markdown本文に添付した画像の保存場所 |
| `templates/` | 新規ファイル作成時に使うテンプレートの保存場所 |

すでに同名フォルダが存在する場合は、そのフォルダを再利用する。

---

## ファイル

ワークスペース内の `.md` ファイル1つ。

| フィールド | 内容 |
|-----------|------|
| name | ファイル名（拡張子なし） |
| path | ワークスペースルートからの相対パス |
| frontmatter | ファイル先頭のYAMLメタデータ（下記参照） |
| body | フロントマターを除いた本文テキスト |
| createdAt | 作成日時 |
| updatedAt | 最終更新日時 |

---

## 添付画像

Markdown本文から参照される補助ファイル。Relicは画像管理アプリではないため、画像単体の整理・タグ付け・検索対象にはしない。

| フィールド | 内容 |
|-----------|------|
| path | ワークスペースルートからの相対パス |
| referencedBy | 画像を参照しているMarkdownファイル |
| alt | Markdown画像記法に書かれた代替テキスト |

---

## フロントマター

ファイル先頭の `---` で囲まれたYAMLブロック。アプリ設定が一律に付与する属性ではなく、ユーザーが任意のMarkdownファイルに後から追加・削除できるメタ情報として扱う。フィールドの詳細仕様は [spec/frontmatter.md](../spec/frontmatter.md) を参照。

| 種類 | フィールド例 | 内容 |
|------|------------|------|
| 登録済み | （ユーザーが命名） | Relicが入力能力を知っているフィールド。検索・絞り込み・入力補助に使う |
| 未認識 | （任意） | 既存ファイルや外部エディタで作成されたYAMLキー。壊さず保持し、必要に応じて後から入力能力を割り当てる |

フィールド名自体に固定の意味は持たせない。たとえば `status` だから選択式、`date` だから日付、という扱いではなく、ユーザーが任意のフィールド名に任意の入力能力を割り当てる。

`tags` や `aliases` のように既存機能と連動できるキーも、デフォルトで各ファイルに作成しない。Markdownファイル内に存在する場合に、Relicがその意味を認識して利用する。

---

## フォルダ

ワークスペース内のディレクトリ。ファイルをグループ化する入れ物。

| フィールド | 内容 |
|-----------|------|
| name | フォルダ名 |
| path | ワークスペースルートからの相対パス |
| children | 中に含まれるファイル・フォルダの一覧 |

---

## 内部リンク

`[[ファイル名]]` 記法で定義されるファイル間の接続。

| フィールド | 内容 |
|-----------|------|
| sourceFile | リンクが書かれているファイル |
| targetName | リンク先として指定されたファイル名 |
| targetFile | 実際に存在するリンク先ファイル（存在しない場合はnull） |
| resolved | リンク先ファイルが存在するか否か |

---

## タグ

ファイルに付与されたラベル。本文の `#タグ名` またはフロントマターの `tags:` から収集する。

| フィールド | 内容 |
|-----------|------|
| name | タグ名（`#` を除いた文字列） |
| files | そのタグが付いているファイルの一覧 |

---

## テンプレート

テンプレートフォルダ内に置かれた `.md` ファイル。新規ファイル作成時に適用できる雛形。

| フィールド | 内容 |
|-----------|------|
| name | テンプレート名（ファイル名） |
| path | ワークスペース相対パス（例: `templates/日記.md`） |
| content | 適用時に読み込むテンプレート本文 |

---

## Gitリポジトリ

ワークスペースに紐付けられたGitリポジトリの情報。

| フィールド | 内容 |
|-----------|------|
| localPath | ローカルのリポジトリパス（ワークスペースのパスと同じ） |
| remoteUrl | GitHubのリモートURL |
| currentBranch | 現在のブランチ名 |

---

## コミット

Gitのコミット1件。コミット履歴の表示・Diffビューで使用する。

| フィールド | 内容 |
|-----------|------|
| hash | コミットを一意に識別するID |
| message | コミットメッセージ |
| author | 作成者名 |
| date | コミット日時 |
| changedFiles | 変更されたファイルの一覧 |

---

## ブランチ

Gitのブランチ1件。

| フィールド | 内容 |
|-----------|------|
| name | ブランチ名 |
| isCurrent | 現在チェックアウト中かどうか |
| isRemote | リモート（GitHub側）にも存在するかどうか |
| lastCommit | 最新のコミット |

---

## Gitタグ

リリース時などにコミットに付けるラベル。`v1.0.0` のようなバージョン番号を付けるために使う。

| フィールド | 内容 |
|-----------|------|
| name | タグ名（例：`v1.0.0`） |
| commit | タグが付いているコミット |
| message | タグのメモ（任意） |

---

## アプリ設定

ユーザーの設定値。Electronの `userData` 配下の `app-settings.json` に保存する。

| フィールド | 内容 |
|-----------|------|
| theme | ライト / ダーク / システム追従 |
| editorFontSize | エディタのフォントサイズ |
| editorLineHeight | 行間 |
| editorMaxWidth | エディタ最大幅のプリセット |
| editorFont | エディタのフォント |
| lineNumbers | 行番号表示のオン/オフ |
| spellCheck | スペルチェックのオン/オフ |
| featureToggles | GitHub連携・ファイル加工ツール・フロントマター・右パネルのオン/オフ |
| userDefinedFields | Relicが入力能力を認識するフロントマターフィールドの一覧 |
| lastWorkspace | 前回開いていたワークスペース |

---

## ワークスペース設定

ワークスペースごとに異なる設定値。Electronの `userData` 配下の `workspaces/{workspaceId}.json` に、ワークスペースと紐付けて保存する。

| フィールド | 内容 |
|-----------|------|
| workspacePath | 対象ワークスペースのパス |
| autoSync | 自動同期の設定（オン/オフ・間隔） |
| gitRepo | GitHub接続・リモートURL・ブランチなどのリポジトリ設定 |

---

## エンティティ間の関係

```
Workspace
  ├── Folder（ネスト可）
  │     └── File
  └── File
        ├── Frontmatter
        │     └── Tag（tags: フィールド）
        ├── Tag（本文中の #tag）
        └── InternalLink → File（リンク先）

Workspace
  └── GitRepository
        ├── Branch（複数）
        │     └── Commit（複数）
        └── GitTag → Commit

Template（テンプレートフォルダ内のFile）

AttachmentImage
  └── File（Markdown本文から参照）
```
