# architecture/overview.md

Relicのアーキテクチャ全体像。主要コンポーネントの構成と責務を定義する。

---

## 全体構成

Electron アプリは「メインプロセス」「preload境界」「レンダラープロセス」に分かれる。

```
┌─────────────────────────────────────────────┐
│              Renderer Process               │
│  左レール・カードサイドバー・タブ・右パネル │
│  （CodeMirror 6 / 画面描画）                 │
├─────────────────────────────────────────────┤
│              Preload / contextBridge         │
│  window.relic API と型付きIPC呼び出し          │
├─────────────────────────────────────────────┤
│                  IPC 通信                   │
│  （レンダラー ↔ メイン間のメッセージング）    │
├─────────────────────────────────────────────┤
│               Main Process                  │
│  カード読み書き・検索/リンク/タグ生成・設定保存 │
│  （Node.js）                                │
└─────────────────────────────────────────────┘
         ↕
  Local Cardbook
  Directory
```

---

## メインプロセスの責務

カードシステム、設定保存、検索・バックリンク・タグ・別名・Timeline表示用データの生成はメインプロセスが担う。レンダラーはpreloadで公開された `window.relic` APIだけを通して操作する。

| コンポーネント | 責務 |
|--------------|------|
| Card Operations | カードとカードフォルダの読み書き・作成・削除・移動・リネーム・複製 |
| Cardbook Service | カードブック登録、切り替え、一覧削除、名前変更、前回カードブック復元 |
| Search / Backlinks / Tags / Aliases | カードを読み取り、検索結果、バックリンク、タグ一覧、別名索引、プロパティ値候補を生成 |
| Timeline | `timeline` プロパティからTimeline表示用データを生成 |
| Tool Actions | マージ、見出し分割、タイトル一覧、目次生成を実行し、結果を新規カードとして保存 |
| Settings Service | アプリ設定・カードブック設定・プロパティテンプレート設定・入力能力設定の読み書き |
| IPC Handlers | レンダラーへ公開する操作単位を検証し、active cardbookとメインプロセスの処理へ接続する |

アウトゴーイングリンクとアウトラインは、開いているカードタブの内容からレンダラー側で導出する。カードブック全体を読む必要があるバックリンク、検索、Timelineはメインプロセスに要求する。

---

## レンダラープロセスの責務

画面の描画とユーザー操作の受け付けを担う。

| コンポーネント | 責務 |
|--------------|------|
| LeftRail | カードサイドバーや各種パネルタブを開く入口 |
| CardsSidebar | カードツリー表示、カード/カードフォルダ操作、ピン留め、カード検索、カードブック作成/登録 |
| CardTab | カードの編集単位。CodeMirrorエディタ、ライブプレビュー、ソースモード、タイプライターモードを含む |
| PanelTab | カード加工、プロパティ設定、設定を表示する補助タブ |
| ChartTab | Timelineを表示するタブ |
| RightPanel | アウトライン、バックリンク、アウトゴーイングリンクを表示する補助パネル |
| Editor | Markdown編集本体。ライブプレビュー、ソースモード、タイプライターモードを含む |
| Toolbar | メインエリア上部の共有エディタツールバー。書式挿入ボタン等を配置 |
| SplitView | タブ表示領域の左右2ペイン分割表示 |
| StatusBar | 文字数・単語数の表示 |

## データの流れ（基本）

1. ユーザーがカードを開く → レンダラーがメインにIPCで要求
2. メインがカードを読み取り → レンダラーに内容を返す
3. レンダラーがカードタブを開き、CodeMirror 6 に内容をセット → 編集可能になる
4. ユーザーが編集 → 入力停止後1秒でメインに自動保存要求
5. メインがカードに書き込み → 次の検索・リンク・タグ・Timeline読み取りでは更新後のMarkdownを正として生成する
6. アウトラインとアウトゴーイングリンクは、アクティブなカードタブの内容からレンダラー側で再導出する
7. Timelineのバー編集はメインへIPCで要求し、メインが対象Markdownのプロパティを書き換えた後、Timelineを再生成する

## ストレージ

| 種別 | 場所 | 内容 |
|------|------|------|
| カード | ユーザーが登録した任意のローカルディレクトリ内 | ユーザーが作成した `.md` カード |
| アプリ設定 | Electronのユーザーデータディレクトリ内の `app-settings.json` | エディタ設定・機能トグル・入力能力・プロパティテンプレート・登録カードブック |
| カードブック設定 | Electronのユーザーデータディレクトリ内の `cardbooks/{cardbookId}.json` | カードブックパス・ピン留め・Timeline設定 |
| 検索・リンク・タグ・Timeline表示用データ | メモリ上または要求時生成 | カードとプロパティから生成される派生データ。カードブック内には保存しない |

ローカルディレクトリは、iCloud Drive / OneDrive / Dropbox などのクラウド同期ディレクトリでもよい。Relicは各クラウドサービスのAPI連携を持たず、OS上の通常のディレクトリとして読み書きする。
