# architecture/overview.md

Relicのアーキテクチャ全体像。主要コンポーネントの構成と責務を定義する。

---

## 全体構成

Electron アプリは「メインプロセス」と「レンダラープロセス」の2層に分かれる。

```
┌─────────────────────────────────────────────┐
│              Renderer Process               │
│  UI・エディタ・サイドバー・パネル類           │
│  （CodeMirror 6 / 画面描画）                 │
├─────────────────────────────────────────────┤
│                  IPC 通信                   │
│  （レンダラー ↔ メイン間のメッセージング）    │
├─────────────────────────────────────────────┤
│               Main Process                  │
│  ファイル読み書き・Git操作・インデックス管理  │
│  （Node.js / isomorphic-git）               │
└─────────────────────────────────────────────┘
         ↕                    ↕
  Local Workspace        GitHub (HTTPS)
  Folder
```

---

## メインプロセスの責務

ファイルシステムやネットワークへのアクセスはすべてメインプロセスが担う。

| コンポーネント | 責務 |
|--------------|------|
| FileSystem | Markdownファイルの読み書き・作成・削除・移動・リネーム |
| GitManager | コミット・プッシュ・プル・ブランチ管理・自動同期・コンフリクト検出 |
| LinkIndex | `[[...]]` 内部リンク・バックリンク・アウトゴーイングリンクの索引を管理 |
| TagIndex | フロントマターの `tags:` フィールドの索引を管理 |
| SearchEngine | 全文検索・ファイル名検索・タグ検索のインデックス管理と検索実行 |
| TemplateManager | `templates/` フォルダの自動作成、Markdownテンプレート一覧の取得、新規ファイル作成時のテンプレート適用 |

---

## レンダラープロセスの責務

画面の描画とユーザー操作の受け付けを担う。

| コンポーネント | 責務 |
|--------------|------|
| Editor | Markdown編集。ライブプレビューとタイプライターモードを含む |
| Toolbar | エディタ上部の操作バー。書式挿入ボタン等を配置 |
| FrontmatterForm | ファイル先頭のフロントマターをフォーム形式で表示・編集 |
| Sidebar | ファイルツリー表示・ワークスペース切り替え・ピン留め |
| TemplateSelector | 新規ノート作成時に `templates/` 内のMarkdownテンプレートを選ぶ |
| TabBar | 複数ファイルのタブ管理 |
| SplitView | エディタエリアの左右2ペイン分割表示 |
| SearchPanel | 検索UI・結果一覧表示 |
| OutlinePanel | 見出し一覧表示とジャンプ |
| LinksPanel | 右パネルに表示するバックリンク一覧・アウトゴーイングリンク一覧 |
| GitPanel | コミット履歴・Diff表示・ブランチ管理・コンフリクト解決UI |
| StatusBar | 文字数・単語数の表示 |

---

## データの流れ（基本）

1. ユーザーがファイルを開く → レンダラーがメインにIPCで要求
2. メインがファイルを読み取り → レンダラーに内容を返す
3. レンダラーが CodeMirror 6 にセット → 編集可能になる
4. ユーザーが編集 → 入力停止後1秒でメインに自動保存要求
5. メインがファイルに書き込み → LinkIndex・TagIndex を更新

## ストレージ

| 種別 | 場所 | 内容 |
|------|------|------|
| Markdownファイル | ユーザーが登録した任意のローカルフォルダ内 | ユーザーが作成した `.md` ファイル |
| 添付画像 | ワークスペース内の `attachments/` | Markdown本文から参照される画像 |
| テンプレート | ワークスペース内の `templates/` | 新規ファイル作成時に使う雛形 |
| アプリ設定 | Electronのユーザーデータフォルダ内の `app-settings.json` | テーマ・フォントサイズ等の設定値 |
| ワークスペース設定 | Electronのユーザーデータフォルダ内の `workspaces/{workspaceId}.json` | GitHub接続・リモートURL・ブランチなど |
| GitHub認証情報 | macOS Keychain | OAuthで取得した認証情報 |
| 検索インデックス | メモリ上（起動時に再構築） | 全文検索・タグ索引 |

ローカルフォルダは、iCloud Drive / OneDrive / Dropbox などのクラウド同期フォルダでもよい。Relicは各クラウドサービスのAPI連携を持たず、OS上の通常のフォルダとして読み書きする。
