# architecture/overview.md

Relicのアーキテクチャ全体像。主要コンポーネントの構成と責務を定義する。

---

## 全体構成

Electron アプリは「メインプロセス」と「レンダラープロセス」の2層に分かれる。

```
┌─────────────────────────────────────────────┐
│              Renderer Process               │
│  左レール・ファイルサイドバー・タブ・右パネル │
│  （CodeMirror 6 / 画面描画）                 │
├─────────────────────────────────────────────┤
│                  IPC 通信                   │
│  （レンダラー ↔ メイン間のメッセージング）    │
├─────────────────────────────────────────────┤
│               Main Process                  │
│  ファイル読み書き・索引生成・設定保存        │
│  （Node.js）                                │
└─────────────────────────────────────────────┘
         ↕
  Local Workspace
  Folder
```

---

## メインプロセスの責務

ファイルシステム、設定保存、検索・リンク・タグなどの索引生成はメインプロセスが担う。レンダラーはpreloadで公開されたIPC APIだけを通して操作する。

| コンポーネント | 責務 |
|--------------|------|
| FileSystem | Markdownファイルとフォルダの読み書き・作成・削除・移動・リネーム・複製 |
| LinkIndex | `[[...]]` 内部リンク・バックリンク・アウトゴーイングリンクの索引を管理 |
| TagIndex | フロントマターの `tags:` フィールドの索引を管理 |
| SearchEngine | 全文検索・ファイル名検索・タグ検索・フロントマター検索のインデックス管理と検索実行 |
| Chronicle / Graph | フロントマターや内部リンクから年表・日付チャート・グラフ表示用データを生成 |
| SettingsService | アプリ設定・ワークスペース設定・フロントマターテンプレート設定・入力能力設定の読み書き |
| IPC Handlers | レンダラーへ公開する操作単位を検証し、メインプロセスの処理へ接続する |

---

## レンダラープロセスの責務

画面の描画とユーザー操作の受け付けを担う。

| コンポーネント | 責務 |
|--------------|------|
| LeftRail | ファイルサイドバーや各種パネルタブを開く入口 |
| FilesSidebar | ファイルツリー表示・ワークスペース切り替え・ピン留め・ファイル検索 |
| FileTab | Markdownファイルの編集単位。CodeMirrorエディタ、ライブプレビュー、ソースモード、タイプライターモードを含む |
| PanelTab | ダッシュボード、グラフ、ファイル加工、フロントマター設定、設定を表示する補助タブ |
| ChartTab | 年表 / 日付チャートを表示するタブ |
| RightPanel | アウトライン、バックリンク、アウトゴーイングリンクを表示する補助パネル |
| Editor | Markdown編集本体。ライブプレビュー、ソースモード、タイプライターモードを含む |
| Toolbar | エディタ上部の操作バー。書式挿入ボタン等を配置 |
| SplitView | タブ表示領域の左右2ペイン分割表示 |
| StatusBar | 文字数・単語数の表示 |

---

## データの流れ（基本）

1. ユーザーがファイルを開く → レンダラーがメインにIPCで要求
2. メインがファイルを読み取り → レンダラーに内容を返す
3. レンダラーがファイルタブを開き、CodeMirror 6 に内容をセット → 編集可能になる
4. ユーザーが編集 → 入力停止後1秒でメインに自動保存要求
5. メインがファイルに書き込み → LinkIndex・TagIndex を更新

## ストレージ

| 種別 | 場所 | 内容 |
|------|------|------|
| Markdownファイル | ユーザーが登録した任意のローカルフォルダ内 | ユーザーが作成した `.md` ファイル |
| アプリ設定 | Electronのユーザーデータフォルダ内の `app-settings.json` | エディタ設定・機能トグル・入力能力・フロントマターテンプレート・登録ワークスペース |
| ワークスペース設定 | Electronのユーザーデータフォルダ内の `workspaces/{workspaceId}.json` | ワークスペースパス・ピン留め・チャート設定 |
| 検索・リンク・タグ索引 | メモリ上（起動時やファイル更新時に再構築） | 全文検索・内部リンク・タグ・グラフ・年表表示用データ |

ローカルフォルダは、iCloud Drive / OneDrive / Dropbox などのクラウド同期フォルダでもよい。Relicは各クラウドサービスのAPI連携を持たず、OS上の通常のフォルダとして読み書きする。
