# P11

Relicのマークダウンエディタ改善フェーズの正本。

このフェーズでは、RelicのMarkdown正本という前提を保ちながら、Markdown編集体験、ライブプレビュー、フロントマター編集、図表・表・リンクなどエディタ内操作の改善を進める。

---

## フェーズ固有ルール・制約

AIはこのフェーズを前提にユーザーへ接する。

### 読み込みルール

最初に読む:

- この文書

必要になったときだけ読む:

- プロダクト意図を確認する: `../../project/overview.md`
- Markdownエディタ仕様との整合を見る: `../../features/markdown.md`, `../../features/frontmatter.md`
- 図表やプレビュー仕様を見る: `../../features/editor.md`, `../../features/markdown.md`, `../../features/navigation.md`
- 画面・遷移・見た目を確認する: `../../design/screens.md`, `../../design/navigation.md`, `../../design/system.md`
- アーキテクチャやデータ構造に触れる場合: `../../engineering/architecture.md`, `../../engineering/data-model.md`, `../../engineering/decisions.md`
- 実装規約とテスト方針を確認する: `../coding-rules.md`, `../testing-rules.md`
- 未決定事項を扱う場合: `../questions.md`
- バージョンを切る場合: `../versioning-rules.md`

注意:

- 対象範囲は、ユーザーが明示したMarkdownエディタ改善に限る。
- Markdown保存形式、ローカルファイル操作仕様、AI機能や外部サービス利用方針、画面全体の設計変更は、ユーザー確認なしに変更しない。
- ライブプレビューや装飾表示を改善する場合も、Markdown本文そのものを勝手に変換しない。
- CodeMirror、エディタ状態管理、自動保存、外部変更検知に触れる場合は、既存の未保存本文保護と衝突検知を壊さない。
- 仕様書、設計書、チェックリスト、詳細な作業項目は、ユーザーの明示指示または合意なしに作成しない。
- 小規模修正でも、関連ファイルの更新がユーザー指示範囲を越える場合は、理由と対象を説明してから確認する。
- React Doctorなどの診断ツールは判断材料として使ってよいが、点数を上げるためだけにエディタ本体や状態管理を大きく作り替えない。
- 未決定の問いは仕様書や設計書に書かず、必要になった場合だけ `../questions.md` に一元管理する。

---

## 対象範囲

- Markdown本文編集の操作性改善
- ライブプレビューと通常プレビューの表示・操作改善
- フロントマター編集UIの改善
- 表、リンク、図表、コードブロックなどエディタ内要素の操作改善
- キーボード操作、選択範囲操作、コンテキストメニューの改善
- エディタ周辺の小規模な不具合修正
- 既存仕様に沿ったテスト追加・補強
- 変更に必要な最小限の関連文書更新

対象外:

- ユーザー確認なしのMarkdown保存形式変更
- ユーザー確認なしのエディタ全面刷新
- ユーザー確認なしの画面全体リデザイン
- ユーザー確認なしのローカルファイル操作仕様変更
- ユーザー確認なしのAI機能や外部サービス利用方針変更
- 配布ビルド確認
