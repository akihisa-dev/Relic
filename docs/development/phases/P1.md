# P1

Relicの暦設定の実装フェーズの正本。

このフェーズでは、暦設定の実装に必要な範囲だけを扱う。

---

## フェーズ固有ルール・制約

AIはこのフェーズを前提にユーザーへ接する。

### 読み込みルール

最初に読む:

- この文書

必要になったときだけ読む:

- プロダクト意図を確認する: `../../project/overview.md`
- フロントマター仕様を確認する: `../../features/frontmatter.md`
- チャートビューの画面・遷移・見た目を確認する: `../../design/screens.md`, `../../design/navigation.md`, `../../design/system.md`
- アーキテクチャとデータ構造に触れる場合: `../../engineering/architecture.md`, `../../engineering/data-model.md`, `../../engineering/decisions.md`
- 実装規約とテスト方針を確認する: `../coding-rules.md`, `../testing-rules.md`
- バージョンを切る場合: `../versioning-rules.md`

注意:

- 対象範囲はチャートビューとフロントマターに限る。
- チャートビューとフロントマター以外のUI、仕様、データ構造へ変更を広げない。
- 暦設定の詳細仕様は、ユーザーに1問ずつ確認してから決める。
- 仕様が決まっていない暦単位、保存形式、表示形式、既存日付との関係をAIが推定で実装しない。
- 既存Markdownファイルの本文やフロントマターを書き換える処理は、影響範囲と戻し方を確認してから実装する。

---

## 対象範囲

- チャートビュー
- フロントマター

対象外:

- アプリ全体の思想変更
- チャートビューとフロントマター以外の画面変更
- 暦設定と直接関係しない新機能
- ユーザー確認なしの既存Markdown一括変更
