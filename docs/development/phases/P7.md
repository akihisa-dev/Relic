# P7

RelicのAIワークスペースフェーズの正本。

このフェーズでは、Relicの根底である「Markdownに書ける情報をMarkdownファイルのまま保つ」方針を崩さず、AIがワークスペース内の情報をどのように補助できるかを、ユーザー確認に基づいて小さく定義する。

---

## フェーズ固有ルール・制約

AIはこのフェーズを前提にユーザーへ接する。

### 読み込みルール

最初に読む:

- この文書

必要になったときだけ読む:

- プロダクト意図を確認する: `../../project/overview.md`
- 既存仕様との整合を見る: `../../features/`
- 画面・遷移・見た目を確認する: `../../design/screens.md`, `../../design/navigation.md`, `../../design/system.md`
- アーキテクチャやデータ構造に触れる場合: `../../engineering/architecture.md`, `../../engineering/data-model.md`, `../../engineering/decisions.md`
- 実装規約とテスト方針を確認する: `../coding-rules.md`, `../testing-rules.md`
- 未決定事項を扱う場合: `../questions.md`
- バージョンを切る場合: `../versioning-rules.md`

注意:

- 対象範囲はAIワークスペースの方向性整理、仕様化、最小実装、検証に限る。
- AI補助は、Markdownファイルを正本として扱うRelicの方針を崩さない範囲で検討する。
- AIが生成した内容を、ユーザー確認なしにMarkdownファイルへ書き込む前提にしない。
- AIワークスペースの具体的な機能、画面、データ保持、外部サービス利用、API利用は、ユーザーへ1問ずつ確認し、承認された内容だけを正本化または実装する。
- 未決定の問いは仕様書や設計書に書かず、必要になった場合だけ `../questions.md` に一元管理する。
- 新しい仕様書、設計書、チェックリスト、UI、API連携を勝手に作成・更新しない。

---

## 対象範囲

- AIワークスペースの目的と境界の整理
- AIが参照してよいワークスペース情報の範囲決め
- AIがMarkdownファイルへ提案、要約、整理、生成を行う場合の扱いの整理
- 必要になった場合の最小限の仕様書、設計文書、実装、テスト

対象外:

- ユーザー確認なしのAI機能確定
- ユーザー確認なしの外部AIサービス利用
- ユーザー確認なしのAPIキー保存方式決定
- ユーザー確認なしのMarkdownファイル自動更新
- ユーザー確認なしの画面全体リデザイン
- 配布ビルド確認
