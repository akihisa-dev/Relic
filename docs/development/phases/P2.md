# P2

Relicのマークダウンエディタとファイルビュー関連フェーズの正本。

このフェーズでは、マークダウンエディタとファイルビュー関連の実装に必要な範囲だけを扱う。

---

## フェーズ固有ルール・制約

AIはこのフェーズを前提にユーザーへ接する。

### 読み込みルール

最初に読む:

- この文書

必要になったときだけ読む:

- プロダクト意図を確認する: `../../project/overview.md`
- エディタ仕様を確認する: `../../features/editor.md`, `../../features/markdown.md`
- ファイルビュー、ファイル操作、検索、リンクを確認する: `../../features/files.md`, `../../features/search.md`, `../../features/links.md`
- 画面・遷移・見た目を確認する: `../../design/screens.md`, `../../design/navigation.md`, `../../design/system.md`
- アーキテクチャ、データ構造、エディタエンジンに触れる場合: `../../engineering/architecture.md`, `../../engineering/data-model.md`, `../../engineering/editor-engine.md`, `../../engineering/decisions.md`
- 実装規約とテスト方針を確認する: `../coding-rules.md`, `../testing-rules.md`
- バージョンを切る場合: `../versioning-rules.md`

注意:

- 対象範囲はマークダウンエディタとファイルビュー関連に限る。
- 対象外の画面、チャート、フロントマター暦設定へ変更を広げない。
- 詳細仕様は、ユーザーに1問ずつ確認してから決める。
- 既存Markdownファイルの本文やメタデータを書き換える処理は、影響範囲と戻し方を確認してから実装する。

---

## 対象範囲

- マークダウンエディタ
- ファイルビュー関連

対象外:

- アプリ全体の思想変更
- マークダウンエディタとファイルビュー以外の画面変更
- 暦設定、チャートビュー、フロントマター暦設定の追加変更
- ユーザー確認なしの既存Markdown一括変更
