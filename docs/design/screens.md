# design/screens.md

デスクトップ版の画面一覧と各画面の構成。

この文書では、画面内に「何をどこへ置くか」を定義する。
ナビゲーション要素の機能仕様は [../features/navigation.md](../features/navigation.md)、画面遷移やビュー切り替えの流れは [navigation.md](navigation.md) を正とする。

---

## 基本レイアウト原則

- **左レール + プライマリサイドバー + Cowork用セカンダリサイドバー + タブ式メインエリア**を基本構造にする
- **プライマリサイドバー**は、ファイルモードではワークスペースの作成・登録、ファイルツリー、検索、ピン留めを扱い、CoworkモードではAIチャット履歴を扱う
- **セカンダリサイドバー**は、CoworkのAIチャット本文と入力欄を扱う
- **メインエリア**はタブを表示する作業領域とし、Markdownファイル、チャート、ファイル加工、フロントマター設定、暦設定、設定をタブとして開く
- **右パネル**はMarkdownファイルタブの補助領域として扱い、アウトラインとリンク一覧だけを表示する

## 全体レイアウト

```
┌──────────────────────────────────────────────────────────────┐
│  タイトルバー                                                 │
├────┬──────────────┬──────────────┬────────────────┬───────────┤
│左  │              │              │ ペインのタブ行  │           │
│レ  │ プライマリ   │ セカンダリ   │────────────────│ 右パネル  │
│ー  │ サイドバー   │ サイドバー   │ タブ行アクション│ 開閉式    │
│ル  │ （可変幅）   │ （Cowork用） │────────────────│           │
│    │              │              │ メインエリア    │           │
│    │              │              │ ファイル /      │           │
│    │              │              │ パネル / チャート│          │
├────┴──────────────┴──────────────┴────────────────┴───────────┤
│  ステータスバー                                               │
```

- メインウィンドウの初期サイズは `1240 x 820` とする
- メインウィンドウの最小サイズは `640 x 400` とする
- macOSでは隠しタイトルバーを使い、ウィンドウ操作ボタンは左上に残す
- Windowsではアプリ用アイコンを設定し、メニューバーは自動で隠す

## 左レール

左端の縦アイコン領域。プライマリサイドバーの開閉、Cowork表示、メインエリアへ開くパネルタブやチャートタブの入口を持つ。

| 入口 | 動作 |
|------|------|
| ファイル | ファイルサイドバーを開閉する |
| Cowork | プライマリサイドバーにAIチャット履歴を表示し、セカンダリサイドバーにAIチャットを開く |
| ファイル加工 | メインエリアにファイル加工ツールのパネルタブを開く |
| フロントマター | メインエリアにフロントマター設定のパネルタブを開く |
| 暦設定 | メインエリアに暦設定のパネルタブを開く |
| 年表 | メインエリアに年表チャートタブを開く |
| カレンダー | メインエリアに日付チャートタブを開く |
| 設定 | メインエリアに設定のパネルタブを開く |

- 各ボタンにはツールチップを表示する
- ファイル、Cowork、設定は常に表示する
- ファイル加工、フロントマター、暦設定、年表、カレンダーは、対応する機能トグルがオフの場合は入口を表示しない
- 初期状態では、ファイル、Cowork、カレンダー、設定を表示する
- 初期状態では、ファイル加工、フロントマター、暦設定、年表を表示しない
- 登録済みワークスペースがある場合は、レール下部からワークスペースを切り替えられる

---

## タイトルバー

画面上部のタイトルバーには、ペインごとのタブ列とタブ行アクションを置く。

- 左側は、macOSのウィンドウ操作領域と左レール・プライマリサイドバー・開いているセカンダリサイドバーに重ならない位置からタブ列を始める
- 分割表示中は、左ペインと右ペインのタブ列をタイトルバー内に並べる
- 右側には、印刷、PDF保存、ソースモード切り替え、分割表示切り替え、右パネルのアウトライン/リンク切り替えを置く
- 右パネルが開いている場合、右側の操作領域は右パネル幅に合わせて確保する

---

## プライマリサイドバー

左レールのファイル入口またはCowork入口で開く可変幅のパネル。

ファイルモードで扱うこと:

- ワークスペースの作成、登録
- ワークスペース内のフォルダ・Markdownファイルのツリー表示
- ファイル / フォルダの作成、リネーム、移動、複製、削除
- ピン留めしたファイル / フォルダの表示
- ファイル名、全文、タグ、フロントマターフィールド、正規表現による検索
- 検索結果クリックによる該当ファイルのオープン

Coworkモードで扱うこと:

- AIチャット履歴
- 新規チャット作成
- チャット切り替え
- チャット削除確認
- Codex残り使用量、またはOpenAI API使用量確認の案内

サイドバーは、macOSでは `⌘B`、Windows/Linuxでは `Ctrl+B`、またはレール上のファイルボタンで開閉できる。幅はユーザーがドラッグで変更できる。
登録済みワークスペースの切り替えは左レール下部で扱う。

## メインエリア

タブ式の作業領域。左右2ペイン分割に対応し、ペインごとに独立したタブ列を持つ。

### タブ種別

| 種別 | 内容 |
|------|------|
| ファイルタブ | Markdownファイルを編集する |
| パネルタブ | ファイル加工、フロントマター設定、暦設定、設定を表示する |
| チャートタブ | `chronicle0`〜`chronicle9` と暦設定をもとに年表、`plannedDate` / `actualDate` をもとにカレンダーを表示する |

### タブ行アクション

- 印刷
- PDFとして保存
- ソースモード切り替え
- 分割表示切り替え
- 右パネルのアウトライン / リンク表示切り替え

各ペインのタブ列右側に表示する。印刷とPDF保存は、フォーカス中ペインのアクティブなMarkdownファイルがある場合に実行できる。右パネル機能トグルがオフの場合は、右パネルのアウトライン / リンク表示切り替えを表示しない。
ファイル名変更はファイルタブ本文上部のファイルタイトルで扱い、ファイル移動はファイルツリーの操作で扱う。

---

## ファイルタブ

Markdownファイルを表示・編集するタブ。

**固定表示:**
- ファイルタイトル
- エディタ本体
- エディタ右上のフロントマター固定プロパティ追加 `+` ボタン
- ステータスバーの文字数・単語数

**表示モード:**
- ライブプレビュー
- ソースモード
- タイプライターモード

フロントマターがある場合は、ライブプレビューではファイル先頭のYAMLブロックを初期状態で折りたたんだフォームとして控えめに表示する。折りたたみ時はプロパティ件数を表示し、隠れたYAML行の空白や行番号は残さない。クリックで展開して値編集や削除を行える。ソースモードではフォーム化せず、Markdown内のYAMLとして表示する。
エディタ右上の `+` ボタンはフロントマターの有無に関係なく同じ位置に表示し、固定プロパティ追加メニューを開く。
フロントマターがない場合は選択した固定プロパティ付きでファイル先頭にフロントマターを作成し、既存フロントマターがある場合はその末尾に追加する。
フロントマターの各プロパティ行には、このファイルからそのプロパティを削除する操作を表示する。

---

## パネルタブ

Markdownファイル以外の作業画面をメインエリア内のタブとして表示する。

| パネル | 内容 |
|--------|------|
| ファイル加工 | マージ、分割、タイトル一覧、目次生成を実行する |
| フロントマター設定 | 固定プロパティの説明、カスタムプロパティの入力能力、候補を管理する |
| 暦設定 | `chronicle0` のメイン暦名、`chronicle1`〜`chronicle9` のサブ暦名とメイン暦開始年を管理する |
| 設定 | テーマ、フォント、行間、最大幅、言語、行番号、スペルチェック、AI接続方式、機能トグルを管理する |

## Coworkセカンダリサイドバー

プライマリサイドバーの右、メインエリアの左に開くAIチャット用の補助領域。

- AIチャット本文
- ユーザー発言の編集と再送信
- メッセージ入力欄
- 送信ボタン
- AI応答生成中の中断ボタン
- 幅変更ハンドル
- 閉じるボタン

セカンダリサイドバーを閉じても、AIチャット履歴やAI系データは削除されない。プライマリサイドバーを閉じても、セカンダリサイドバーは開いたままにできる。

---

## 右パネル

ファイルタブの右側に開閉表示される補助領域。

- アウトラインパネル: 見出し一覧とジャンプ
- リンクパネル: バックリンク・アウトゴーイングリンク一覧
- 右パネル内では、アウトライン / リンクのどちらか一方だけを表示する
- 右パネル機能トグルがオフの場合は、タブ行アクションの右パネル切り替え操作を表示しない

フロントマターの入力能力管理やファイル加工、設定は右パネルではなく、メインエリアのパネルタブで扱う。

---

## 初回起動時

ワークスペースが未登録、または登録済みワークスペースはあるがアクティブワークスペースがない状態で起動したとき、プライマリサイドバーのファイルモードに初期設定ビューを表示する。

- 既存フォルダをワークスペースとして開く
- 新規ワークスペースを作成

新規ワークスペース作成時は、毎回ユーザーが保存場所を選ぶ。
既存フォルダを開く操作と新規ワークスペース作成操作は、どちらもダイアログをキャンセルした場合、現在のワークスペース登録状態を変えない。
