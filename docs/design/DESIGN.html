Visual Design System

Overview

このデザインシステムは、情報の構造化・可読性・集中体験を重視したナレッジワーク向けインターフェースである。

装飾的な要素を最小限に抑えながら、強いタイポグラフィ、明確な境界線、幾何学的な構成を用いて情報階層を視覚化する。

デザインの目的は「美しく見せること」ではなく、「情報構造を理解しやすくすること」にある。

Design Principles

Structure First

すべてのUI要素は視覚的な装飾よりも構造の表現を優先する。

情報の階層、関係性、状態を明確に伝えることを最重要視する。

Strong Typography

タイポグラフィはUIの主要な表現手段である。

サイズ、ウェイト、余白によって情報の優先順位を示す。

色や装飾への依存を避ける。

Geometric Consistency

直線、矩形、境界線を基本要素とする。

過度な曲線や有機的表現は避ける。

Calm Technology

未来的でありながら過剰に派手ではない。

知的で落ち着いた作業環境を提供する。

Screen Structure

デスクトップ版の画面一覧と各画面の構成。

この文書では、画面内に「何をどこへ置くか」を定義する。 ナビゲーション要素の機能仕様は ../features/navigation.md、画面遷移やビュー切り替えの流れはこの文書の「Navigation Flow」を正とする。

基本レイアウト原則

  • 左レール + プライマリサイドバー + Cowork用セカンダリサイドバー + タブ式メインエリアを基本構造にする
  • プライマリサイドバーは、ファイルモードではワークスペースの作成・登録、ファイルツリー、検索、ピン留めを扱い、CoworkモードではAIチャット履歴を扱う
  • セカンダリサイドバーは、CoworkのAIチャット本文と入力欄を扱う
  • メインエリアはタブを表示する作業領域とし、Markdownファイル、チャート、ファイル加工、フロントマター設定、暦設定、設定をタブとして開く
  • 右パネルはMarkdownファイルタブの補助領域として扱い、アウトラインとリンク一覧だけを表示する

左レール

左端の縦アイコン領域。プライマリサイドバーの開閉、Cowork表示、メインエリアへ開くパネルタブやチャートタブの入口を持つ。

入口 動作
ファイル ファイルサイドバーを開閉する
Cowork プライマリサイドバーにAIチャット履歴を表示し、セカンダリサイドバーにAIチャットを開く
ファイル加工 メインエリアにファイル加工ツールのパネルタブを開く
フロントマター メインエリアにフロントマター設定のパネルタブを開く
暦設定 メインエリアに暦設定のパネルタブを開く
年表 メインエリアに年表チャートタブを開く
カレンダー メインエリアに日付チャートタブを開く
設定 メインエリアに設定のパネルタブを開く
  • 各ボタンにはツールチップを表示する
  • ファイル、設定は常に表示する
  • Cowork、ファイル加工、フロントマター、暦設定、年表、カレンダーは、対応する機能トグルがオフの場合は入口を表示しない
  • 初期状態では、ファイル、Cowork、カレンダー、設定を表示する
  • 初期状態では、ファイル加工、フロントマター、暦設定、年表を表示しない
  • 登録済みワークスペースがある場合は、レール下部からワークスペースを切り替えられる

タイトルバー

画面上部のタイトルバーには、ペインごとのタブ列とタブ行アクションを置く。

  • 左側は、macOSのウィンドウ操作領域と左レール・プライマリサイドバー・開いているセカンダリサイドバーに重ならない位置からタブ列を始める
  • 分割表示中は、左ペインと右ペインのタブ列をタイトルバー内に並べる
  • 右側には、印刷、PDF保存、ソースモード切り替え、分割表示切り替え、右パネルのアウトライン/リンク切り替えを置く
  • 右パネルが開いている場合、右側の操作領域は右パネル幅に合わせて確保する

プライマリサイドバー

左レールのファイル入口またはCowork入口で開く可変幅のパネル。

ファイルモードで扱うこと:

  • ワークスペースの作成、登録
  • ワークスペース内のフォルダ・Markdownファイルのツリー表示
  • ファイル / フォルダの作成、リネーム、移動、複製、削除
  • ピン留めしたファイル / フォルダの表示
  • ファイル名、全文、タグ、フロントマターフィールド、正規表現による検索
  • 検索結果クリックによる該当ファイルのオープン

Coworkモードで扱うこと:

  • AIチャット履歴
  • 新規チャット作成
  • チャット切り替え
  • チャット削除確認
  • Codex残り使用量、またはOpenAI API使用量確認の案内

サイドバーは、macOSでは ⌘B、Windows/Linuxでは Ctrl+B、またはレール上のファイルボタンで開閉できる。幅はユーザーがドラッグで変更できる。 登録済みワークスペースの切り替えは左レール下部で扱う。

メインエリア

タブ式の作業領域。左右2ペイン分割に対応し、ペインごとに独立したタブ列を持つ。

タブ種別

種別 内容
ファイルタブ Markdownファイルを編集する
パネルタブ ファイル加工、フロントマター設定、暦設定、設定を表示する
チャートタブ chronicle0〜chronicle9 と暦設定をもとに年表、plannedDate / actualDate をもとにカレンダーを表示する

タブ行アクション

  • 印刷
  • PDFとして保存
  • ソースモード切り替え
  • 分割表示切り替え
  • 右パネルのアウトライン / リンク表示切り替え

各ペインのタブ列右側に表示する。印刷とPDF保存は、フォーカス中ペインのアクティブなMarkdownファイルがある場合に実行できる。右パネルのアウトライン機能トグルがオフの場合はアウトライン切り替えを表示せず、右パネルのリンク機能トグルがオフの場合はリンク切り替えを表示しない。 ファイル名変更はファイルタブ本文上部のファイルタイトルで扱い、ファイル移動はファイルツリーの操作で扱う。

ファイルタブ

Markdownファイルを表示・編集するタブ。

固定表示:

  • ファイルタイトル
  • エディタ本体
  • エディタ右上のフロントマター固定プロパティ追加 + ボタン
  • ステータスバーの文字数・単語数

表示モード:

  • ライブプレビュー
  • ソースモード
  • タイプライターモード

フロントマターがある場合は、ライブプレビューではファイル先頭のYAMLブロックを初期状態で折りたたんだフォームとして控えめに表示する。折りたたみ時はプロパティ件数を表示し、隠れたYAML行の空白や行番号は残さない。クリックで展開して値編集や削除を行える。ソースモードではフォーム化せず、Markdown内のYAMLとして表示する。 エディタ右上の + ボタンはフロントマターの有無に関係なく同じ位置に表示し、固定プロパティ追加メニューを開く。 フロントマターがない場合は選択した固定プロパティ付きでファイル先頭にフロントマターを作成し、既存フロントマターがある場合はその末尾に追加する。 フロントマターの各プロパティ行には、このファイルからそのプロパティを削除する操作を表示する。

パネルタブ

Markdownファイル以外の作業画面をメインエリア内のタブとして表示する。

パネル 内容
ファイル加工 マージ、分割、タイトル一覧、目次生成を実行する
フロントマター設定 固定プロパティの説明、カスタムプロパティの入力能力、候補を管理する
暦設定 chronicle0 のメイン暦名、chronicle1〜chronicle9 のサブ暦名とメイン暦開始年を管理する
設定 テーマ、フォント、行間、最大幅、言語、行番号、スペルチェック、AI接続方式、機能トグルを管理する

Coworkセカンダリサイドバー

プライマリサイドバーの右、メインエリアの左に開くAIチャット用の補助領域。

  • AIチャット本文
  • ユーザー発言の編集と再送信
  • メッセージ入力欄
  • 送信ボタン
  • AI応答生成中の中断ボタン
  • 幅変更ハンドル
  • 閉じるボタン

セカンダリサイドバーを閉じても、AIチャット履歴やAI系データは削除されない。プライマリサイドバーを閉じても、セカンダリサイドバーは開いたままにできる。

右パネル

ファイルタブの右側に開閉表示される補助領域。

  • アウトラインパネル: 見出し一覧とジャンプ
  • リンクパネル: バックリンク・アウトゴーイングリンク一覧
  • 右パネル内では、アウトライン / リンクのどちらか一方だけを表示する
  • 右パネルのアウトライン機能トグルがオフの場合はアウトライン切り替え操作を表示せず、右パネルのリンク機能トグルがオフの場合はリンク切り替え操作を表示しない

フロントマターの入力能力管理やファイル加工、設定は右パネルではなく、メインエリアのパネルタブで扱う。

初回起動時

ワークスペースが未登録、または登録済みワークスペースはあるがアクティブワークスペースがない状態で起動したとき、プライマリサイドバーのファイルモードに初期設定ビューを表示する。

  • 既存フォルダをワークスペースとして開く
  • 新規ワークスペースを作成

新規ワークスペース作成時は、毎回ユーザーが保存場所を選ぶ。 既存フォルダを開く操作と新規ワークスペース作成操作は、どちらもダイアログをキャンセルした場合、現在のワークスペース登録状態を変えない。

Navigation Flow

アプリの画面遷移・ビュー切り替えフローを定義するドキュメント。

前提: ナビゲーションは左レール、プライマリサイドバー、Cowork用セカンダリサイドバー、メインエリアのタブ操作で行う。Markdownファイル以外の作業画面も、メインエリアのタブとして開く。

この文書では、画面遷移やビュー切り替えの「流れ」を定義する。 ナビゲーション要素の機能仕様は ../features/navigation.md、画面内の配置はこの文書の「Screen Structure」を正とする。

起動フロー

アプリ起動
  │
  ├─ 前回ワークスペースIDが有効 → メイン画面（ワークスペース状態を読み込む）
  │
  └─ アクティブワークスペースなし → プライマリサイドバーのファイルモードに初回設定ビューを表示
        │
        ├─ 新規ワークスペース作成
        └─ 既存フォルダを開く
              │
              └─ → メイン画面

登録済みワークスペースがあっても、前回ワークスペースIDが登録一覧に存在しない場合は、アクティブワークスペースなしとして起動する。この場合、左レール下部には登録済みワークスペース一覧を表示できるが、プライマリサイドバー本体は初回設定ビューを表示する。

左レール

左レール
  ├─ ファイル        → ファイルサイドバーを開閉
  ├─ Cowork          → AIチャット履歴を表示し、AIチャットを開く
  ├─ ファイル加工    → パネルタブを開く（機能トグルON時のみ）
  ├─ フロントマター  → パネルタブを開く（機能トグルON時のみ）
  ├─ 暦設定          → パネルタブを開く（機能トグルON時のみ）
  ├─ 年表            → 年表チャートタブを開く（機能トグルON時のみ）
  ├─ カレンダー      → 日付チャートタブを開く（機能トグルON時のみ）
  └─ 設定            → パネルタブを開く

既に同じパネルタブまたはチャートタブが開いている場合は、新規作成せずそのタブをアクティブにする。 ファイル、設定は常に表示する。Cowork、ファイル加工、フロントマター、暦設定、年表、カレンダーは、対応する機能トグルがオフの場合は左レールに表示しない。 初期状態では、ファイル、Cowork、カレンダー、設定を表示し、ファイル加工、フロントマター、暦設定、年表は表示しない。

プライマリサイドバー

プライマリサイドバーは、ファイルモードではワークスペースとMarkdownファイルの操作を扱い、CoworkモードではAIチャット履歴を扱う。

プライマリサイドバー
  ├─ ファイルモード
  │   ├─ ワークスペース作成 / 登録
  │   ├─ ファイルツリー
  │   ├─ ピン留め
  │   └─ 検索
  └─ Coworkモード
      ├─ AIチャット履歴
      ├─ 新規チャット
      ├─ チャット削除確認
      └─ AI使用量表示

macOSでは ⌘B、Windows/Linuxでは Ctrl+B、または左レールのファイルボタンで開閉する。検索コマンドを実行した場合は、ファイルサイドバーを開き、検索入力へフォーカスする。

Cowork

左レールのCoworkボタンを押す
  │
  ├─ プライマリサイドバーをAIチャット履歴表示にする
  └─ セカンダリサイドバーをAIチャット表示で開く
        │
        ├─ チャットを選ぶ → 選択中チャットの会話へ切り替える
        ├─ 新規チャット → 空のチャットを作って選択する
        ├─ メッセージ送信 → AI応答とMarkdown反映結果を会話に表示する
        └─ 応答生成中の中断 → 生成を止める

プライマリサイドバーを閉じても、セカンダリサイドバーは開いたままAIチャットを続けられる。セカンダリサイドバーを閉じた場合も、AIチャット履歴やAI系データは削除しない。

ファイルを開く

ファイルサイドバーでファイルをクリック
  │
  ├─ 新しいファイルタブで開く
  └─ 既に開いているファイルタブがある場合 → そのタブをアクティブにする

内部リンクをクリックした場合も、リンク先ファイルをファイルタブとして開く。リンク先が存在しない場合は新規ファイルを作成して開く。

タブの操作

タブをクリック → そのタブをアクティブにする
タブを閉じる  → 表示履歴ベースで直前に表示していたタブをアクティブにする
              → タブが0枚になった場合は新規ファイル作成画面を表示

表示履歴は、タブをアクティブにした順に記録する。現在のタブを閉じたら、直近で表示していた未クローズのタブへ戻る。

分割表示

分割ボタンを押す → メインエリアが左右2ペインに分割
  │
  ├─ 左ペイン：元のタブをそのまま表示
  └─ 右ペイン：独立したタブ列を持つ

分割解除ボタンを押す → 1ペインに戻る（アクティブなペインのタブを表示）

分割表示中は、ペインごとに独立したタブ列を持つ。ファイルタブ、パネルタブ、チャートタブはいずれもペイン内のタブとして扱う。

右パネルの開閉

アウトラインパネルボタンを押す → 右パネルにアウトラインを表示 / 閉じる
リンクパネルボタンを押す      → 右パネルにリンク一覧を表示 / 閉じる

右パネルはファイルタブの補助領域であり、アウトラインとリンク一覧を同時表示しない。片方を開いている状態でもう片方を開いた場合は、右パネル内の表示内容を切り替える。 右パネルのアウトライン機能トグルがオフの場合はアウトライン切り替え操作を表示せず、右パネルのリンク機能トグルがオフの場合はリンク切り替え操作を表示しない。

現在ファイルの出力

ファイルタブの右クリックメニューで印刷を選ぶ
  │
  └─ 右クリックしたMarkdownファイルから一時PDFを生成
        │
        └─ Chromium PDFプレビューで開く

ファイルタブの右クリックメニューでPDFとして保存を選ぶ
  │
  └─ 右クリックしたMarkdownファイルからPDFを生成
        │
        └─ 保存ダイアログで保存先を選んで保存

印刷とPDF保存は、Markdownファイルのタブ右クリックメニューから実行する。印刷時に生成するPDFはプレビュー用の一時ファイルであり、ユーザーが保存先を選ばない限りワークスペースや任意の保存先には残さない。

ワークスペースの切り替え

左レール下部のワークスペース切替を操作
  │
  └─ 別のワークスペースを選択 → 開いているタブを閉じ、そのワークスペース状態を読み込む

ワークスペース切り替えでは、登録情報、ファイルツリー、ピン留めなどのワークスペース状態を読み込む。前ワークスペースで開いていたタブは持ち越さない。

新規ワークスペース作成

新規ワークスペース作成を選択
  │
  ├─ 保存ダイアログで作成先とフォルダ名を選択
  └─ 指定場所にワークスペースフォルダを作成して開く

保存場所は毎回ユーザーが選択する。 既存フォルダを開く操作や新規ワークスペース作成操作でダイアログをキャンセルした場合は、登録済みワークスペースやアクティブワークスペースを変更しない。 ただし、これらの操作ではダイアログ表示前に開きタブの保存確認を行うため、確認を通過した後にキャンセルした場合も、開きタブは閉じた状態になる。
