Overview

Text is structure. Interface is architecture.

Structure First 情報の階層、関係性、状態を明確に伝えることを最優先する。
Strong Typography サイズ、ウェイト、余白によって情報の優先順位を示す。
Calm Technology 未来的でありながら、過剰に派手ではない作業環境を作る。

Design Principles

Structure First

すべてのUI要素は、装飾よりも構造の表現を優先する。

Strong Typography

タイポグラフィをUIの主要な表現手段として扱う。

Geometric Consistency

直線、矩形、境界線、グリッドを基本要素にする。

Calm Technology

知的で落ち着いた、長く使える作業環境を提供する。

Visual Language

装飾要素は機能的なレイアウトを補助する目的でのみ使用する。装飾が情報を上回ってはならない。

Components

Tabs

明確な境界線を持ち、アクティブ状態は青色ラインで表現する。

Tables

グリッドを強調し、情報密度と読みやすさの両方を保つ。

Buttons

Primaryは青地に白文字、Secondaryは透明背景と境界線を基本にする。

Navigation Flow

Startup

前回ワークスペースIDが有効ならメイン画面へ。未登録なら初回設定ビューを表示する。

Open File

ファイルサイドバーからMarkdownを開き、既存タブがあればそのタブをアクティブにする。

Split View

左右2ペインで独立したタブ列を持ち、分割解除時はアクティブペインへ戻る。
