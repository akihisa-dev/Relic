Visual Design System

Overview

このデザインシステムは、情報の構造化・可読性・集中体験を重視したナレッジワーク向けインターフェースである。

Relicでは、Liquid Charcoalのうち現在採用している原則と、その原則を画面構成、色、UI部品へ適用する仕様をDESIGN.mdの正本へ集約する。

外部で更新される一般的なデザイン原則をリポジトリ内へ複製せず、Relicへ採用する内容だけをDESIGN.mdへ反映する。実装時の具体値と振る舞いは、DESIGN.mdに定義したトークン、コンポーネント、画面仕様を基準にする。

Liquid Charcoalは、Canvas、Liquid、Charcoalの関係によって、Relicの作業画面を読み書きに集中できる状態へ整えるデザイン原則である。

Canvasは情報を置くための基盤であり、Liquidは透明・半透明の層であり、Charcoalは画面に構造、焦点、メリハリを与える炭色の基準である。

色や質感は主役ではなく、Markdown本文、ファイル構造、タブ、アウトライン、プロパティを見分けるための支えとして扱う。

Liquid Charcoalは特定の文化、様式、装飾表現を参照しない。墨色という語は色の参考語であり、書道、墨絵、和風表現を意味しない。

デザインの目的は「美しく見せること」ではなく、「情報構造を理解しやすくすること」にある。

Design Principles

Structure First

すべてのUI要素は視覚的な装飾よりも構造の表現を優先する。

情報の階層、関係性、状態を明確に伝えることを最重要視する。

Clear Typography

タイポグラフィはUIの主要な表現手段である。

サイズ、ウェイト、余白によって情報の優先順位を示す。

文字そのものを装飾せず、本文、見出し、補助情報の差はサイズ、ウェイト、濃度、配置で示す。

Canvas, Liquid, Charcoal

Canvasを情報を書くための基盤として扱う。

Liquidを透明・半透明のレイヤーとして扱い、面同士の距離、重なり、選択状態、補助的なまとまりを示す。

Charcoalを本文、見出し、アクティブ状態、主要操作、現在位置の基準として扱う。

炭色は画面全体を塗る色ではなく、構造とメリハリを与える色として使う。

Thin Boundaries

境界線は1pxを基本にし、パネル、タブ、表、サイドバーの関係を静かに分ける。

境界は情報より強く見えてはならない。境界だけで判別しづらい場合は、Surfaceの明度差やLiquidの透明度差を併用する。

影、グロー、強い質感、装飾的な背景は避ける。

Theme Equality

Light ThemeとDark Themeを同等に扱う。

Light Themeは白いCanvasを基準にし、Charcoalで構造を与える。

Dark Themeは暗いCanvasを基準にし、明るい前景と低濃度のLiquidで構造を与える。

Dark ThemeはLight Themeの単純な反転ではない。

どちらのテーマでも、情報の可読性、現在位置、操作対象、選択状態を明確に示す。

Border System

border-width: 1px;

影による階層表現は最小限に抑える。

Corners

border-radius: 6px;

角丸は画面全体の連続性と静けさを保つために使う。

過度に丸い形状や、強くキャラクター化された形状は避ける。

検索欄、ボタン、選択行、カード、モーダル、オーバーレイなどの操作対象は、コンポーネントとプラットフォームに応じて控えめな角丸を使う。

Functional Colors

有彩色は、状態を伝える必要がある場合に限って使う。

機能色はLiquid Charcoalの印象を作るための色ではなく、エラー、警告、成功、情報を誤認させないための補助である。

用途 Light Theme Dark Theme 使い方
Danger #B42318 #FFB4A9 エラー、削除、破壊的操作
Warning #8A5A00 #E7C16C 警告、注意、未解決状態
Success #2E6B3F #88D49D 成功、完了、正常状態
Info #2F5E8C #A8C7FA 補足情報、通知、案内

機能色は必要な要素に限定して使い、通常のナビゲーション、装飾、ブランド表現には使わない。

Color Usage

Canvasは作業面の基盤である。Light Themeでは白を基準にし、Dark Themeでは暗いCanvasを基準にする。

Liquidは透明・半透明の層である。情報のまとまり、重なり、選択状態、操作対象を示すために使う。

Charcoalは本文、見出し、リンク、アクティブ線、主要ボタン、現在位置に使う。

Charcoalは少なく使うことで意味を持つ。面の塗りつぶしには限定的に使う。

Grayは非アクティブなタブ、選択背景、補助情報、境界線に使う。役割は情報の階層を支えることであり、装飾的な塗り分けではない。

Typography

書体はInter、Noto Sans JP、OSのシステムフォントの順に使用する。

フォントファイルはアプリに同梱しない。ユーザー環境にない場合はOS標準フォントへ自然にフォールバックする。

macOS、Windows、Linuxそれぞれの標準的なUIフォントで自然に見えることを優先する。

font-family: Inter, "Noto Sans JP", system-ui, sans-serif;

Heading

スタイル:

font-weight: 700;
letter-spacing: 0;

Section Labels

font-size: 12px;
font-weight: 700;
letter-spacing: 0;
text-transform: none;

Body

font-size: 14px;
line-height: 1.7;
font-weight: 400;

Surface and Layers

Liquid Charcoalでは、画面を単一の平面として扱わない。

Canvasの上にSurfaceとLiquidが重なり、必要な場所にCharcoalが現れる。

Surfaceは情報を整理するための面であり、装飾用のカードではない。

Liquidレイヤーは透明または半透明で構成する。

Surface同士の区別は、濃い塗り分けではなく、明度差、透明度差、境界線によって示す。

透明性によって文字や操作対象の判別が弱くなる場合は、可読性を優先する。

Motion

Liquid Charcoalの動きは、操作を補助するために存在する。

動きは情報を強調するためではなく、状態の変化を自然に伝えるために使う。

Liquidは炭色を含む流体として、わずかな粘性を持つ。そのため、動きは急激に跳ねたり、強く反発したりしない。

過度な弾性、大きな移動、長いアニメーションは避ける。

動きは滑らかで、静かで、短くあるべきである。

Texture and Light

光は演出ではなく、素材を認識するために存在する。

Liquidは自然な透過と弱い反射を持つ。

Charcoalは光を強く跳ね返さず、沈んだ見え方を持つ。

ハイライト、反射、影は、要素の分離や状態の理解を助ける範囲に留める。

質感は色そのものを変えず、軽い階調、透明度、明度差として扱う。

推奨する質感:

  • Clean
  • Matte
  • Smooth
  • Quiet
  • Monochrome
  • Transparent
  • Lightly layered
  • Slightly viscous

避ける質感:

  • Glossy
  • Metallic
  • Plastic
  • Glassy
  • Neon
  • Wet-looking
  • Noisy
  • Dirty
  • Strongly textured

Decorative Elements

装飾要素は機能的なレイアウトを補助する目的でのみ使用する。

装飾はLiquid Charcoalの質感を誇張するためではなく、情報構造、境界、視線誘導を支えるために使う。

許可される要素:

  • 細い境界線
  • Canvasより一段低いSurface
  • 透明または半透明のLiquidレイヤー
  • Charcoalのアクティブインジケーター
  • 読みやすさを損なわない軽い階調

装飾が情報を上回ってはならない。

強いノイズ、汚れ加工、グロー、ガラス反射、金属光沢、ネオン化、有彩色の強いアクセント、装飾のためのグラデーション、角丸や影による過度なカード化は使用しない。

和風化、書道、墨絵、筆致の表現、紙の繊維や凹凸の再現、液体を過剰に演出するアニメーションも使用しない。

  • Structured
  • Intelligent
  • Calm
  • Focused
  • Durable
  • Functional
  • Observational
  • Clean
  • Monochrome
  • Transparent
  • Lightly layered

避けるべき印象:

  • Playful
  • Organic
  • Decorative
  • Skeuomorphic
  • Overly Minimal
  • Consumer Social App
  • Glossy
  • Metallic
  • Plastic
  • Glassy
  • Neon
  • Wet
  • Colorful
  • Noisy
  • Strongly textured

この空気感は、派手な演出ではなく、情報の配置、余白、細い境界、透明・半透明の層、炭色のコントラストによって作る。

適用対象:

  • アプリUI
  • ドキュメント
  • カード
  • 表
  • 入力欄
  • タブ
  • サイドバー
  • アウトライン
  • プロパティ
  • モーダル

どの制作物でも、素材感は主役ではなく、情報と構造を支える背景として扱う。

要素や機能は現状のまま保ち、視覚表現だけを調整する。

Screen Structure

デスクトップ版の画面一覧と各画面の構成。

この文書では、画面内に「何をどこへ置くか」を定義する。 ナビゲーション要素の機能仕様は ../features/navigation.md、画面遷移やビュー切り替えの流れはこの文書の「Navigation Flow」を正とする。

基本レイアウト原則

  • 左レール + プライマリサイドバー + タブ式メインエリア + 右パネルを基本構造にする
  • プライマリサイドバーは、ファイルモードではワークスペースの作成・登録、ファイルツリー、検索、ピン留めを扱う
  • メインエリアはタブを表示する作業領域とし、Markdownファイル、チャート、ファイル加工、フロントマター設定、暦設定、設定をタブとして開く
  • 右パネルはMarkdownファイルタブの補助領域として扱い、アウトライン、リンク一覧、復元版のいずれか一つを表示する

左レール

左端の縦アイコン領域。プライマリサイドバーの開閉、メインエリアへ開くパネルタブやチャートタブの入口を持つ。

入口 動作
ファイル ファイルサイドバーを開閉する
ファイル加工 メインエリアにファイル加工ツールのパネルタブを開く
フロントマター メインエリアにフロントマター設定のパネルタブを開く
グラフビュー メインエリアにグラフビューチャートタブを開く
暦設定 メインエリアに暦設定のパネルタブを開く
年表 メインエリアに年表チャートタブを開く
設定 メインエリアに設定のパネルタブを開く
  • 各ボタンにはツールチップを表示する
  • ファイル、グラフビュー、設定は常に表示する
  • ファイル加工、フロントマター、暦設定、年表は、対応する機能トグルがオフの場合は入口を表示しない
  • 初期状態では、ファイル、グラフビュー、設定を表示する
  • 初期状態では、ファイル加工、フロントマター、暦設定、年表を表示しない
  • 登録済みワークスペースがある場合は、レール下部からワークスペースを切り替えられる

タイトルバー

画面上部のタイトルバーには、ウィンドウ操作領域とアプリ全体の上部領域を置く。

  • macOSのウィンドウ操作領域と左レール・プライマリサイドバーに重ならないようにする
  • ペインごとのタブ列は、タイトルバーではなくメインエリア上部に表示する
  • ファイル本文上部のファイルタイトル行には、ソースモード切り替えとフロントマター固定プロパティ追加操作を置く
  • PDF保存はMarkdownファイルのタブ右クリックメニューに置く

プライマリサイドバー

左レールのファイル入口で開く可変幅のパネル。

ファイルモードで扱うこと:

  • ワークスペースの作成、登録
  • ワークスペース内のフォルダ・Markdownファイルのツリー表示
  • ファイル / フォルダの作成、リネーム、移動、複製、削除
  • ピン留めしたファイル / フォルダの表示
  • 検索ボタンからのクイックスイッチャー起動

サイドバーは、macOSでは ⌘B、Windows/Linuxでは Ctrl+B、またはレール上のファイルボタンで開閉できる。幅はユーザーがドラッグで変更できる。 登録済みワークスペースの切り替えは左レール下部で扱う。

メインエリア

タブ式の作業領域。左右2ペイン分割に対応し、ペインごとに独立したタブ列を持つ。

タブ種別

種別 内容
ファイルタブ Markdownファイルを編集する
パネルタブ ファイル加工、フロントマター設定、暦設定、設定を表示する
チャートタブ graph でワークスペースのMarkdown、タグ、添付画像のリンクグラフ、chronicle と暦設定で年表を表示する

ファイルタイトル行

  • ファイルタイトル
  • ソースモード切り替え
  • フロントマター固定プロパティ追加

ファイルタイトル行はファイルタブ本文上部に表示する。ファイル名変更はファイルタイトルで扱い、ファイル移動はファイルツリーの操作で扱う。分割表示と右パネル表示は、それぞれの対応ボタンまたはショートカットで切り替える。

ファイルタブ

Markdownファイルを表示・編集するタブ。

固定表示:

  • ファイルタイトル
  • エディタ本体
  • エディタ右上のフロントマター固定プロパティ追加 + ボタン
  • ステータスバーの文字数・単語数

表示モード:

  • ライブプレビュー
  • ソースモード
  • タイプライターモード

フロントマターがある場合は、ライブプレビューではファイル先頭のYAMLブロックを初期状態で折りたたんだフォームとして控えめに表示する。折りたたみ時はプロパティ件数を表示し、隠れたYAML行の空白や行番号は残さない。クリックで展開して値編集や削除を行える。ソースモードではフォーム化せず、Markdown内のYAMLとして表示する。 エディタ右上の + ボタンはフロントマターの有無に関係なく同じ位置に表示し、固定プロパティ追加メニューを開く。 フロントマターがない場合は選択した固定プロパティ付きでファイル先頭にフロントマターを作成し、既存フロントマターがある場合はその末尾に追加する。 フロントマターの各プロパティ行には、このファイルからそのプロパティを削除する操作を表示する。

パネルタブ

Markdownファイル以外の作業画面をメインエリア内のタブとして表示する。

パネル 内容
ファイル加工 タイトル一覧、目次生成、タグ別索引、フォルダ内マージを実行する
フロントマター設定 固定プロパティを常時確認できる一覧と category 候補管理を表示し、YAML書式例は補助情報として開閉表示する
暦設定 年表で使うメイン暦名、サブ暦名、サブ暦のメイン暦開始年を管理する
設定 テーマ、フォント、行間、最大幅、言語、行番号、スペルチェック、機能トグルを管理する

右パネル

ファイルタブの右側に開閉表示される補助領域。

  • アウトラインパネル: 見出し一覧とジャンプ
  • リンクパネル: バックリンク・アウトゴーイングリンク一覧
  • 復元パネル: 現在開いているMarkdownファイルの保存前復元版
  • 右パネル内では、アウトライン / リンク / 復元版のいずれか一つだけを表示する
  • 右パネルのアウトライン機能トグルがオフの場合はアウトライン切り替え操作を表示せず、右パネルのリンク機能トグルがオフの場合はリンク切り替え操作を表示しない
  • フロントマター機能トグルがオフの場合は、左レールのフロントマター設定入口を表示しない

フロントマターの固定プロパティ説明や category 候補管理は、メインエリアのフロントマター設定パネルタブで扱う。個別ファイル内のフロントマター値編集はファイルタブ内のフロントマター表示で扱う。ファイル加工、設定も右パネルではなく、メインエリアのパネルタブで扱う。

初回起動時

ワークスペースが未登録、または登録済みワークスペースはあるがアクティブワークスペースがない状態で起動したとき、プライマリサイドバーのファイルモードに初期設定ビューを表示する。

  • 既存フォルダをワークスペースとして開く
  • 新規ワークスペースを作成

新規ワークスペース作成時は、毎回ユーザーが保存場所を選ぶ。 既存フォルダを開く操作と新規ワークスペース作成操作は、どちらもダイアログをキャンセルした場合、現在のワークスペース登録状態を変えない。

Navigation Flow

アプリの画面遷移・ビュー切り替えフローを定義するドキュメント。

前提: ナビゲーションは左レール、プライマリサイドバー、メインエリアのタブ操作で行う。Markdownファイル以外の作業画面も、メインエリアのタブとして開く。

この文書では、画面遷移やビュー切り替えの「流れ」を定義する。 ナビゲーション要素の機能仕様は ../features/navigation.md、画面内の配置はこの文書の「Screen Structure」を正とする。

起動フロー

アプリ起動
  │
  ├─ 前回ワークスペースIDが有効 → メイン画面（ワークスペース状態を読み込む）
  │
  └─ アクティブワークスペースなし → プライマリサイドバーのファイルモードに初回設定ビューを表示
        │
        ├─ 新規ワークスペース作成
        └─ 既存フォルダを開く
              │
              └─ → メイン画面

登録済みワークスペースがあっても、前回ワークスペースIDが登録一覧に存在しない場合は、アクティブワークスペースなしとして起動する。この場合、左レール下部には登録済みワークスペース一覧を表示できるが、プライマリサイドバー本体は初回設定ビューを表示する。

左レール

左レール
  ├─ ファイル        → ファイルサイドバーを開閉
  ├─ ファイル加工    → パネルタブを開く（機能トグルON時のみ）
  ├─ フロントマター  → パネルタブを開く（機能トグルON時のみ）
  ├─ グラフビュー    → グラフビューチャートタブを開く
  ├─ 暦設定          → パネルタブを開く（機能トグルON時のみ）
  ├─ 年表            → 年表チャートタブを開く（機能トグルON時のみ）
  └─ 設定            → パネルタブを開く

既に同じパネルタブまたはチャートタブが開いている場合は、新規作成せずそのタブをアクティブにする。 ファイル、グラフビュー、設定は常に表示する。ファイル加工、フロントマター、暦設定、年表は、対応する機能トグルがオフの場合は左レールに表示しない。 初期状態では、ファイル、グラフビュー、設定を表示し、ファイル加工、フロントマター、暦設定、年表は表示しない。

プライマリサイドバー

プライマリサイドバーは、ファイルモードではワークスペースとMarkdownファイルの操作を扱う。

プライマリサイドバー
  ├─ ファイルモード
  │   ├─ ワークスペース作成 / 登録
  │   ├─ ファイルツリー
  │   ├─ ピン留め
  │   └─ 検索ボタン

macOSでは ⌘B、Windows/Linuxでは Ctrl+B、または左レールのファイルボタンで開閉する。検索ボタンまたは検索コマンドを実行した場合は、クイックスイッチャーを開く。

ファイルを開く

ファイルサイドバーでファイルをクリック
  │
  ├─ 新しいファイルタブで開く
  └─ 既に開いているファイルタブがある場合 → そのタブをアクティブにする

内部リンクをクリックした場合も、リンク先ファイルをファイルタブとして開く。リンク先が存在しない場合は新規ファイルを作成して開く。

タブの操作

タブをクリック → そのタブをアクティブにする
タブを閉じる  → 表示履歴ベースで直前に表示していたタブをアクティブにする
              → タブが0枚になった場合は新規ファイル作成画面を表示

表示履歴は、タブをアクティブにした順に記録する。現在のタブを閉じたら、直近で表示していた未クローズのタブへ戻る。

分割表示

分割ボタンを押す → メインエリアが左右2ペインに分割
  │
  ├─ 左ペイン：元のタブをそのまま表示
  └─ 右ペイン：独立したタブ列を持つ

分割解除ボタンを押す → 1ペインに戻る（アクティブなペインのタブを表示）

分割表示中は、ペインごとに独立したタブ列を持つ。ファイルタブ、パネルタブ、チャートタブはいずれもペイン内のタブとして扱う。

右パネルの開閉

アウトラインパネルボタンを押す      → 右パネルにアウトラインを表示 / 閉じる
リンクパネルボタンを押す           → 右パネルにリンク一覧を表示 / 閉じる
復元パネルボタンを押す             → 右パネルに現在ファイルの保存前復元版を表示 / 閉じる

右パネルはファイルタブの補助領域であり、アウトライン、リンク一覧、復元版を同時表示しない。いずれかを開いている状態で別の表示を開いた場合は、右パネル内の表示内容を切り替える。 右パネルのアウトライン機能トグルがオフの場合はアウトライン切り替え操作を表示せず、右パネルのリンク機能トグルがオフの場合はリンク切り替え操作を表示しない。

現在ファイルの出力

ファイルタブの右クリックメニューでPDFとして保存を選ぶ
  │
  └─ 右クリックしたMarkdownファイルからPDFを生成
        │
        └─ 保存ダイアログで保存先を選んで保存

PDF保存は、Markdownファイルのタブ右クリックメニューから実行する。ユーザーが保存先を選ばない限りワークスペースや任意の保存先には残さない。

ワークスペースの切り替え

左レール下部のワークスペース切替を操作
  │
  └─ 別のワークスペースを選択 → 開いているタブを閉じ、そのワークスペース状態を読み込む

ワークスペース切り替えでは、登録情報、ファイルツリー、ピン留めなどのワークスペース状態を読み込む。前ワークスペースで開いていたタブは持ち越さない。

新規ワークスペース作成

新規ワークスペース作成を選択
  │
  ├─ 保存ダイアログで作成先とフォルダ名を選択
  └─ 指定場所にワークスペースフォルダを作成して開く

保存場所は毎回ユーザーが選択する。 既存フォルダを開く操作や新規ワークスペース作成操作でダイアログをキャンセルした場合は、登録済みワークスペースやアクティブワークスペースを変更しない。 ただし、これらの操作ではダイアログ表示前に開きタブの保存確認を行うため、確認を通過した後にキャンセルした場合も、開きタブは閉じた状態になる。
