# Visual Design System

## Overview

このデザインシステムは、情報の構造化・可読性・集中体験を重視したナレッジワーク向けインターフェースである。

Relicの色、素材、質感の基礎思想は [LIQUID_CHARCOAL.md](LIQUID_CHARCOAL.md) を正とし、この文書ではその思想をRelicの画面構成、色、UI部品へ適用する。

Relicでは、白は情報を書くための紙面、黒は操作状態と文字の基準、灰色は面の階層と境界として扱う。

質感は写実的な素材再現ではなく、液体のように滑らかな階調と、木炭のように沈んだ黒の対比として扱う。

装飾的な要素を最小限に抑えながら、黒い文字、細い境界線、十分な余白、控えめな選択状態によって情報階層を視覚化する。

デザインの目的は「美しく見せること」ではなく、「情報構造を理解しやすくすること」にある。

---

## Design Principles

### Structure First

すべてのUI要素は視覚的な装飾よりも構造の表現を優先する。

情報の階層、関係性、状態を明確に伝えることを最重要視する。

---

### Clear Typography

タイポグラフィはUIの主要な表現手段である。

サイズ、ウェイト、余白によって情報の優先順位を示す。

大きな見出しは黒い文字で示し、大文字化や装飾線で強調しすぎない。

---

### Quiet Monochrome

色は白、黒、灰色を中心にする。

状態表示や主要操作にも青などの有彩色を使わず、黒い線、黒い面、淡いグレーの背景で示す。

---

### Thin Boundaries

境界線は1pxを基本にし、パネル、タブ、表、サイドバーの関係を静かに分ける。

影による浮遊感は避け、必要な場合も薄い影に留める。

画面全体は、明るい白い面と薄い線で構成する。

---

## Visual Language

### Shapes

#### Primary

- Rectangle
- Line
- Grid
- Section Block

#### Secondary

- Subtle Rounded Rectangle
- Thin Active Indicator
- Divider Line

装飾要素は情報の邪魔にならない範囲で使用する。

---

#### Border System

境界線は重要な情報構造の一部として扱う。

```css
border-width: 1px;
border-color: var(--border-primary);
```

影による階層表現は最小限に抑える。

---

### Corners

```css
corner-radius: 4px;
```

角丸は検索欄、ボタン、選択行、カードなど小さな操作対象にだけ控えめに使用する。

アプリの大きなカラム、タブ境界、左右パネルの境界は直線を基本とする。

---

## Color Palette

### Core Colors

#### Primary Ink

主要アクセントカラー。Liquid Charcoalでは、文字、アクティブ線、主要ボタンに使う黒として扱う。

```css
--color-primary: #111111;
```

用途:

- Active indicators
- Primary buttons
- Links
- Icons
- High-emphasis labels

---

#### Secondary Charcoal

補助アクセントカラー。黒より弱いテキスト、補助アイコン、ホバー状態に使うチャコールグレーとして扱う。

```css
--color-accent: #5F6368;
```

用途:

- Hover States
- Sub Labels
- Boundary Emphasis

---

### Neutral Palette

#### Background

```css
--color-bg: #FFFFFF;
```

エディタ背景、ページ全体の背景色。情報を書くための白い紙面として扱う。

---

#### Surface

```css
--color-surface: #FAFAFA;
```

サイドバー、ツールバー、パネル背景。白い紙面より一段低い淡い面として扱う。

---

#### Surface Alt

```css
--color-surface-alt: #F3F3F2;
```

選択中の項目背景、カード背景。状態を示すための淡いグレーとして扱う。

---

#### Border

```css
--color-border: #DEDEDE;
```

区切り線。パネル、タブ、表、フォームを分ける細い線として扱う。

---

#### Border Strong

```css
--color-border-strong: #B8B8B8;
```

重要な境界。通常境界より少し強いが、黒い主線とは区別する。

---

### Typography Colors

#### Primary Text

```css
--color-text: #111111;
```

本文、見出し、主要ラベルに使う黒として扱う。

---

#### Secondary Text

```css
--color-text-secondary: #5F6368;
```

補助情報、非アクティブなタブ、二次ラベルに使う。

---

#### Muted Text

```css
--color-text-muted: #9A9A9A;
```

さらに弱い説明文、プレースホルダー、補助メタ情報に使う。

---

## Typography

書体はInter、Noto Sans JP、OSのシステムフォントの順に使用する。

フォントファイルはアプリに同梱しない。ユーザー環境にない場合はOS標準フォントへ自然にフォールバックする。

macOS、Windows、Linuxそれぞれの標準的なUIフォントで自然に見えることを優先する。

```css
font-family: Inter, "Noto Sans JP", system-ui, sans-serif;
```

### Heading

スタイル:

```css
font-weight: 700;
letter-spacing: 0;
```

---

### Section Labels

```css
font-size: 12px;
font-weight: 700;
letter-spacing: 0;
text-transform: none;
```

---

### Body

```css
font-size: 14px;
line-height: 1.7;
font-weight: 400;
```

## Layout

### Three Panel Structure

```text
Explorer
Editor
Outline
```

情報構造を常時可視化する。

---

### Grid Alignment

主要なUI要素はグリッドに沿って配置する。

任意の位置調整は避ける。

---

最適な可読性を維持するため、以下を推奨する。

```css

```

## Navigation

### Active Item

選択状態は淡い背景色と左側の黒いインジケーターで表現する。

```css
width: 3px;
background: var(--color-primary);
```

---

### Outline Navigation

アウトラインは番号を付けず、控えめな目次として表示する。

見出し階層は左余白と細い境界線で表現する。

番号ブロックや大きな装飾見出しは使用しない。

例:

```text
Overview
  Goals
  References
    Notes
```

---

## Components

### Tabs

- 明確な境界線
- 過剰な影を使用しない
- アクティブ状態は黒い細線、黒い文字、白い面で表現

---

### Tables

- グリッドを強調
- 行間を広く確保
- 情報密度を保ちながら読みやすさを優先

---

### Buttons

Primary:

```css
background: var(--color-primary);
color: white;
```

Secondary:

```css
background: transparent;
border: 1px solid var(--color-border);
```

---

## Decorative Elements

装飾要素は機能的なレイアウトを補助する目的でのみ使用する。

装飾はLiquid Charcoalの質感を誇張するためではなく、情報構造、境界、視線誘導を支えるために使う。

許可される要素:

- 細い境界線
- 淡いグレー面
- 黒いアクティブインジケーター
- 控えめなグラデーション

装飾が情報を上回ってはならない。

強いノイズ、汚れ加工、グロー、ガラス反射、金属光沢、ネオン化、有彩色の強いアクセントは使用しない。

# Screen Structure

デスクトップ版の画面一覧と各画面の構成。

この文書では、画面内に「何をどこへ置くか」を定義する。
ナビゲーション要素の機能仕様は [../features/navigation.md](../features/navigation.md)、画面遷移やビュー切り替えの流れはこの文書の「Navigation Flow」を正とする。

---

## 基本レイアウト原則

- **左レール + プライマリサイドバー + タブ式メインエリア + 右パネル**を基本構造にする
- **プライマリサイドバー**は、ファイルモードではワークスペースの作成・登録、ファイルツリー、検索、ピン留めを扱う
- **メインエリア**はタブを表示する作業領域とし、Markdownファイル、チャート、ファイル加工、フロントマター設定、暦設定、設定をタブとして開く
- **右パネル**はMarkdownファイルタブの補助領域として扱い、アウトライン、リンク一覧、現在ファイルのフロントマター編集フォームのいずれか一つを表示する

---

## 全体レイアウト

```
┌──────────────────────────────────────────────────────────────┐
│  タイトルバー                                                 │
├────┬──────────────┬────────────────────────┬───────────┤
│左  │              │ ペインのタブ行          │           │
│レ  │ プライマリ   │────────────────────────│ 右パネル  │
│ー  │ サイドバー   │ ファイルタイトル行      │ 開閉式    │
│ル  │ （可変幅）   │────────────────────────│           │
│    │              │ メインエリア            │           │
│    │              │ ファイル / パネル /     │           │
│    │              │ チャート                │           │
├────┴──────────────┴────────────────────────┴───────────┤
│  ステータスバー                                               │
└──────────────────────────────────────────────────────────────┘
```

- メインウィンドウの初期サイズは `1240 x 820` とする
- メインウィンドウの最小サイズは `640 x 400` とする
- macOSでは隠しタイトルバーを使い、ウィンドウ操作ボタンは左上に残す
- Windowsではアプリ用アイコンを設定し、メニューバーは自動で隠す

---

## 左レール

左端の縦アイコン領域。プライマリサイドバーの開閉、メインエリアへ開くパネルタブやチャートタブの入口を持つ。

| 入口 | 動作 |
|------|------|
| ファイル | ファイルサイドバーを開閉する |
| ファイル加工 | メインエリアにファイル加工ツールのパネルタブを開く |
| フロントマター | メインエリアにフロントマター設定のパネルタブを開く |
| グラフビュー | メインエリアにグラフビューチャートタブを開く |
| 暦設定 | メインエリアに暦設定のパネルタブを開く |
| 年表 | メインエリアに年表チャートタブを開く |
| 設定 | メインエリアに設定のパネルタブを開く |

- 各ボタンにはツールチップを表示する
- ファイル、グラフビュー、設定は常に表示する
- ファイル加工、フロントマター、暦設定、年表は、対応する機能トグルがオフの場合は入口を表示しない
- 初期状態では、ファイル、グラフビュー、設定を表示する
- 初期状態では、ファイル加工、フロントマター、暦設定、年表を表示しない
- 登録済みワークスペースがある場合は、レール下部からワークスペースを切り替えられる

---

## タイトルバー

画面上部のタイトルバーには、ウィンドウ操作領域とアプリ全体の上部領域を置く。

- macOSのウィンドウ操作領域と左レール・プライマリサイドバーに重ならないようにする
- ペインごとのタブ列は、タイトルバーではなくメインエリア上部に表示する
- ファイル本文上部のファイルタイトル行には、ソースモード切り替えとフロントマター固定プロパティ追加操作を置く
- PDF保存はMarkdownファイルのタブ右クリックメニューに置く

---

## プライマリサイドバー

左レールのファイル入口で開く可変幅のパネル。

ファイルモードで扱うこと:

- ワークスペースの作成、登録
- ワークスペース内のフォルダ・Markdownファイルのツリー表示
- ファイル / フォルダの作成、リネーム、移動、複製、削除
- ピン留めしたファイル / フォルダの表示
- ファイル名、全文、タグ、フロントマターフィールドによる検索
- 検索結果クリックによる該当ファイルのオープン

サイドバーは、macOSでは `⌘B`、Windows/Linuxでは `Ctrl+B`、またはレール上のファイルボタンで開閉できる。幅はユーザーがドラッグで変更できる。
登録済みワークスペースの切り替えは左レール下部で扱う。

---

## メインエリア

タブ式の作業領域。左右2ペイン分割に対応し、ペインごとに独立したタブ列を持つ。

### タブ種別

| 種別 | 内容 |
|------|------|
| ファイルタブ | Markdownファイルを編集する |
| パネルタブ | ファイル加工、フロントマター設定、暦設定、設定を表示する |
| チャートタブ | `graph` でワークスペースのMarkdown、タグ、添付画像のリンクグラフ、`chronicle` と暦設定で年表を表示する |

### ファイルタイトル行

- ファイルタイトル
- ソースモード切り替え
- フロントマター固定プロパティ追加

ファイルタイトル行はファイルタブ本文上部に表示する。ファイル名変更はファイルタイトルで扱い、ファイル移動はファイルツリーの操作で扱う。分割表示と右パネル表示は、それぞれの対応ボタンまたはショートカットで切り替える。

---

## ファイルタブ

Markdownファイルを表示・編集するタブ。

**固定表示:**
- ファイルタイトル
- エディタ本体
- エディタ右上のフロントマター固定プロパティ追加 `+` ボタン
- ステータスバーの文字数・単語数

**表示モード:**
- ライブプレビュー
- ソースモード
- タイプライターモード

フロントマターがある場合は、ライブプレビューではファイル先頭のYAMLブロックを初期状態で折りたたんだフォームとして控えめに表示する。折りたたみ時はプロパティ件数を表示し、隠れたYAML行の空白や行番号は残さない。クリックで展開して値編集や削除を行える。ソースモードではフォーム化せず、Markdown内のYAMLとして表示する。
エディタ右上の `+` ボタンはフロントマターの有無に関係なく同じ位置に表示し、固定プロパティ追加メニューを開く。
フロントマターがない場合は選択した固定プロパティ付きでファイル先頭にフロントマターを作成し、既存フロントマターがある場合はその末尾に追加する。
フロントマターの各プロパティ行には、このファイルからそのプロパティを削除する操作を表示する。

---

## パネルタブ

Markdownファイル以外の作業画面をメインエリア内のタブとして表示する。

| パネル | 内容 |
|--------|------|
| ファイル加工 | タイトル一覧、目次生成、タグ別索引、フォルダ内マージを実行する |
| フロントマター設定 | 固定プロパティの説明、カスタムプロパティの入力能力、候補を管理する |
| 暦設定 | 年表で使うメイン暦名、サブ暦名、サブ暦のメイン暦開始年を管理する |
| 設定 | テーマ、フォント、行間、最大幅、言語、行番号、スペルチェック、機能トグルを管理する |

---

## 右パネル

ファイルタブの右側に開閉表示される補助領域。

- アウトラインパネル: 見出し一覧とジャンプ
- リンクパネル: バックリンク・アウトゴーイングリンク一覧
- フロントマターパネル: 現在開いているMarkdownファイルのフロントマター編集フォーム
- 右パネル内では、アウトライン / リンク / フロントマターのいずれか一つだけを表示する
- 右パネルのアウトライン機能トグルがオフの場合はアウトライン切り替え操作を表示せず、右パネルのリンク機能トグルがオフの場合はリンク切り替え操作を表示しない
- フロントマター機能トグルがオフの場合は、左レールのフロントマター設定入口と右パネルのフロントマター切り替え操作を表示しない

フロントマターの入力能力管理や固定プロパティ説明は右パネルではなく、メインエリアのフロントマター設定パネルタブで扱う。ファイル加工、設定も右パネルではなく、メインエリアのパネルタブで扱う。

---

## 初回起動時

ワークスペースが未登録、または登録済みワークスペースはあるがアクティブワークスペースがない状態で起動したとき、プライマリサイドバーのファイルモードに初期設定ビューを表示する。

- 既存フォルダをワークスペースとして開く
- 新規ワークスペースを作成

新規ワークスペース作成時は、毎回ユーザーが保存場所を選ぶ。
既存フォルダを開く操作と新規ワークスペース作成操作は、どちらもダイアログをキャンセルした場合、現在のワークスペース登録状態を変えない。

# Navigation Flow

アプリの画面遷移・ビュー切り替えフローを定義するドキュメント。

**前提**: ナビゲーションは左レール、プライマリサイドバー、メインエリアのタブ操作で行う。Markdownファイル以外の作業画面も、メインエリアのタブとして開く。

この文書では、画面遷移やビュー切り替えの「流れ」を定義する。
ナビゲーション要素の機能仕様は [../features/navigation.md](../features/navigation.md)、画面内の配置はこの文書の「Screen Structure」を正とする。

---

## 起動フロー

```
アプリ起動
  │
  ├─ 前回ワークスペースIDが有効 → メイン画面（ワークスペース状態を読み込む）
  │
  └─ アクティブワークスペースなし → プライマリサイドバーのファイルモードに初回設定ビューを表示
        │
        ├─ 新規ワークスペース作成
        └─ 既存フォルダを開く
              │
              └─ → メイン画面
```

登録済みワークスペースがあっても、前回ワークスペースIDが登録一覧に存在しない場合は、アクティブワークスペースなしとして起動する。この場合、左レール下部には登録済みワークスペース一覧を表示できるが、プライマリサイドバー本体は初回設定ビューを表示する。

---

## 左レール

```
左レール
  ├─ ファイル        → ファイルサイドバーを開閉
  ├─ ファイル加工    → パネルタブを開く（機能トグルON時のみ）
  ├─ フロントマター  → パネルタブを開く（機能トグルON時のみ）
  ├─ グラフビュー    → グラフビューチャートタブを開く
  ├─ 暦設定          → パネルタブを開く（機能トグルON時のみ）
  ├─ 年表            → 年表チャートタブを開く（機能トグルON時のみ）
  └─ 設定            → パネルタブを開く
```

既に同じパネルタブまたはチャートタブが開いている場合は、新規作成せずそのタブをアクティブにする。
ファイル、グラフビュー、設定は常に表示する。ファイル加工、フロントマター、暦設定、年表は、対応する機能トグルがオフの場合は左レールに表示しない。
初期状態では、ファイル、グラフビュー、設定を表示し、ファイル加工、フロントマター、暦設定、年表は表示しない。

---

## プライマリサイドバー

プライマリサイドバーは、ファイルモードではワークスペースとMarkdownファイルの操作を扱う。

```
プライマリサイドバー
  ├─ ファイルモード
  │   ├─ ワークスペース作成 / 登録
  │   ├─ ファイルツリー
  │   ├─ ピン留め
  │   └─ 検索
```

macOSでは `⌘B`、Windows/Linuxでは `Ctrl+B`、または左レールのファイルボタンで開閉する。検索コマンドを実行した場合は、ファイルサイドバーを開き、検索入力へフォーカスする。

---

## ファイルを開く

```
ファイルサイドバーでファイルをクリック
  │
  ├─ 新しいファイルタブで開く
  └─ 既に開いているファイルタブがある場合 → そのタブをアクティブにする
```

内部リンクをクリックした場合も、リンク先ファイルをファイルタブとして開く。リンク先が存在しない場合は新規ファイルを作成して開く。

---

## タブの操作

```
タブをクリック → そのタブをアクティブにする
タブを閉じる  → 表示履歴ベースで直前に表示していたタブをアクティブにする
              → タブが0枚になった場合は新規ファイル作成画面を表示
```

表示履歴は、タブをアクティブにした順に記録する。現在のタブを閉じたら、直近で表示していた未クローズのタブへ戻る。

---

## 分割表示

```
分割ボタンを押す → メインエリアが左右2ペインに分割
  │
  ├─ 左ペイン：元のタブをそのまま表示
  └─ 右ペイン：独立したタブ列を持つ

分割解除ボタンを押す → 1ペインに戻る（アクティブなペインのタブを表示）
```

分割表示中は、ペインごとに独立したタブ列を持つ。ファイルタブ、パネルタブ、チャートタブはいずれもペイン内のタブとして扱う。

---

## 右パネルの開閉

```
アウトラインパネルボタンを押す      → 右パネルにアウトラインを表示 / 閉じる
リンクパネルボタンを押す           → 右パネルにリンク一覧を表示 / 閉じる
フロントマターパネルボタンを押す    → 右パネルに現在ファイルのフロントマター編集フォームを表示 / 閉じる
```

右パネルはファイルタブの補助領域であり、アウトライン、リンク一覧、フロントマター編集フォームを同時表示しない。いずれかを開いている状態で別の表示を開いた場合は、右パネル内の表示内容を切り替える。
右パネルのアウトライン機能トグルがオフの場合はアウトライン切り替え操作を表示せず、右パネルのリンク機能トグルがオフの場合はリンク切り替え操作を表示しない。フロントマター機能トグルがオフの場合は、右パネルのフロントマター切り替え操作を表示しない。

---

## 現在ファイルの出力

```
ファイルタブの右クリックメニューでPDFとして保存を選ぶ
  │
  └─ 右クリックしたMarkdownファイルからPDFを生成
        │
        └─ 保存ダイアログで保存先を選んで保存
```

PDF保存は、Markdownファイルのタブ右クリックメニューから実行する。ユーザーが保存先を選ばない限りワークスペースや任意の保存先には残さない。

---

## ワークスペースの切り替え

```
左レール下部のワークスペース切替を操作
  │
  └─ 別のワークスペースを選択 → 開いているタブを閉じ、そのワークスペース状態を読み込む
```

ワークスペース切り替えでは、登録情報、ファイルツリー、ピン留めなどのワークスペース状態を読み込む。前ワークスペースで開いていたタブは持ち越さない。

---

## 新規ワークスペース作成

```
新規ワークスペース作成を選択
  │
  ├─ 保存ダイアログで作成先とフォルダ名を選択
  └─ 指定場所にワークスペースフォルダを作成して開く
```

保存場所は毎回ユーザーが選択する。
既存フォルダを開く操作や新規ワークスペース作成操作でダイアログをキャンセルした場合は、登録済みワークスペースやアクティブワークスペースを変更しない。
ただし、これらの操作ではダイアログ表示前に開きタブの保存確認を行うため、確認を通過した後にキャンセルした場合も、開きタブは閉じた状態になる。
