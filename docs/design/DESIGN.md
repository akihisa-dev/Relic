# Visual Design System

## Overview

このデザインシステムは、情報の構造化・可読性・集中体験を重視したナレッジワーク向けインターフェースである。

Relicでは、Liquid Charcoalのうち現在採用している原則と、その原則を画面構成、色、UI部品へ適用する仕様をこの文書の正本として扱う。

外部で更新される一般的なデザイン原則をリポジトリ内へ複製せず、Relicへ採用する内容だけをこの文書へ反映する。実装時の具体値と振る舞いは、この文書に定義したトークン、コンポーネント、画面仕様を基準にする。

Liquid Charcoalは、読み書きに集中できる作業画面を整えるデザイン原則である。

目的は、情報の読みやすさ、状態の識別、操作の明確さを保ちながら、静かで構造的かつ清潔感のある画面をつくることにある。

色や質感は主役ではない。Markdown本文、ファイル構造、タブ、アウトライン、プロパティなどの情報と操作を見分けるために使う。

Liquid Charcoalは、特定の文化、様式、流行、装飾表現を根拠にしない。

---

## Design Principles

- 情報と操作を、色や質感より優先する。
- 主作業領域は、画用紙のような明るい不透明な面、または黒色の水ガラスのような暗い面にする。
- ナビゲーション、操作面、補助面、前面の層は、画面の階層と連続性を保つために半透明の面を一貫して使ってよい。
- 深い黒は、本文と構造の焦点に限定する。主要操作の実行面は、橙色の水ガラスで示してよい。
- Light ThemeとDark Themeを同等に扱う。
- 橙色のアクセントは、用途付きの複数トークンとして持つ。用途のない色段階は作らない。
- 機能色は、鮮やかな赤、緑、青の3色に固定し、エラー、警告、成功、情報を伝える場合に限って使う。
- 情報の区別には、余白、配置、明度差、透明度差、細い境界を使う。
- 広めの余白、精密な境界、軽い面の重なりによって、清潔で新鮮な印象をつくる。
- 透明感や静けさによって可読性と操作性を損なわない。
- 既存の情報構造を、装飾や不要なコンテナで複雑にしない。

## Themes

Light ThemeとDark Themeは、同じ情報の役割と状態をそれぞれの明度環境で表現する。

Dark ThemeはLight Themeの単純な色反転ではない。どちらのテーマでも、本文、補助情報、現在位置、選択状態、操作対象、境界を明確に区別する。

Dark Themeでは、構造と焦点の役割を明るい前景が担う。

| 項目 | Light Theme | Dark Theme |
|------|-------------|------------|
| 作業面 | 画用紙のような明るい不透明な面 | 黒色の水ガラスのような暗い面 |
| 主な前景 | 深い黒 | 画用紙のような明るい前景 |
| 補助面 | 作業面よりわずかに暗い面 | 作業面よりわずかに明るい面 |
| 半透明の面 | 白を含む低濃度の面 | 暗い色を含む低濃度の面 |
| 境界 | 淡い灰色の細線 | 暗部で判別できる明度差のある細線 |
| 状態の区別 | 濃度、境界、余白、配置 | 濃度、境界、余白、配置 |

## Visual Language

### Shapes

#### Primary

- Canvas
- Surface
- Navigation / Operation Surface
- Auxiliary Surface
- Foreground Surface
- Hairline Border
- Thin Active Indicator

#### Secondary

- Rounded Rectangle
- Divider Line
- Section Block
- Grid

装飾要素は情報の邪魔にならない範囲で使用する。

---

#### Border System

境界線は重要な情報構造の一部として扱う。

```css
border-width: 1px;
border-color: var(--color-border);
```

影による階層表現は最小限に抑える。

---

### Corners

角丸は画面全体の連続性と静けさを保つために使う。

過度に丸い形状や、強くキャラクター化された形状は避ける。

検索欄、ボタン、選択行、カード、モーダル、オーバーレイなどの操作対象は、コンポーネントの大きさとプラットフォームに応じて控えめな角丸を一貫して使う。

## Color Palette

色と質感は、画用紙、黒色の水ガラス、橙色の水ガラスを基準にする。トークンは用途と一緒に定義し、用途のない色段階や装飾だけの色段階は追加しない。

### Accent Palette

橙色のアクセントは機能色ではない。主要操作、視線の起点、主要な区切りなど、限定された役割にだけ使う。濃度、境界、余白、配置、状態差を組み合わせ、意味のない橙色の広い面塗りは行わない。

| 役割 | トークン | Light Theme | Dark Theme | 用途 |
|------|----------|-------------|------------|------|
| アクセント面 | `--color-accent-surface` | `rgba(242, 105, 27, 0.84)` | `rgba(255, 155, 77, 0.82)` | 主要操作、限定された前面のアクセント面 |
| アクセント | `--color-accent` | `#F2691B` | `#FF9B4D` | ブランドの印、視線の起点、主要な区切り |
| 強調境界 | `--color-accent-strong` | `#D95711` | `#FFD0AD` | ホバー、アクティブ、強調境界 |

### Base Tokens

Light ThemeとDark Themeでは同じ役割を保ちながら色値を分ける。色値を機械的に反転せず、本文、補助情報、境界、状態の判別をそれぞれ確認する。

| 役割 | トークン | Light Theme | Dark Theme |
|------|----------|-------------|------------|
| 主要な前景 | `--color-primary` | `#1A1B17` | `#F4F0E6` |
| 作業面 | `--color-bg` | `#F4F0E6` | `#11120F` |
| 半透明の面 | `--color-overlay` | `rgba(255, 255, 255, 0.44)` | `rgba(17, 18, 15, 0.70)` |
| 低濃度の面 | `--color-overlay-subtle` | `rgba(26, 27, 23, 0.06)` | `rgba(244, 240, 230, 0.08)` |
| 補助面 | `--color-surface` | `#FAF7F0` | `#191A16` |
| 選択された面 | `--color-surface-alt` | `#EBE5D9` | `#24251F` |
| 通常の境界 | `--color-border` | `#D8D0C1` | `#3B3C33` |
| 強い境界 | `--color-border-strong` | `#B8AF9F` | `#5B5D52` |
| 本文 | `--color-text` | `#1A1B17` | `#F4F0E6` |
| 補助情報 | `--color-text-secondary` | `#62625B` | `#B8B8AB` |
| 弱い補助情報 | `--color-text-muted` | `#76756C` | `#8E8E82` |

### Functional Colors

機能色は状態を誤認させないための補助であり、通常のナビゲーション、装飾、ブランド表現には使わない。`danger`と`warning`は意味として区別するが、同じ赤の`--color-danger`を使い、文言、形、境界、アイコンなどで意味を分ける。

| 色 | トークン | Light Theme | Dark Theme | 用途 |
|----|----------|-------------|------------|------|
| Red | `--color-danger` | `#E23B30` | `#FF6B5F` | エラー、警告、破壊的操作、未解決 |
| Green | `--color-success` | `#008A5A` | `#45E39B` | 成功、完了、正常状態 |
| Blue | `--color-info` | `#2563EB` | `#68A5FF` | 補足情報、通知、案内 |

機能色だけで状態を伝えず、必要な文言、形、境界、アイコン、位置などを併用する。

## Typography

タイポグラフィは、プラットフォームの標準を基本とする。

文字の階層は役割で定義し、具体的なサイズ、行間、ウェイトは環境に合わせて調整する。長文を無理なく読み続けられる行長と行間を保つ。

小ささや薄さだけで情報を弱めない。補助情報であっても、必要な内容は読めるコントラストを確保する。

```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

| 役割 | 階層の基準 |
|------|------------|
| `display` | 画面の入口や大きな見出し。最も強い階層にする。 |
| `heading` | セクションやまとまりを示し、本文より明確にする。 |
| `body` | 長く読み続ける本文。行長と行間を優先する。 |
| `auxiliary` | 補助情報。小ささや薄さだけで必要な情報を弱めない。 |

`display`、`heading`、`body`、`auxiliary`の関係を一貫させる。文字そのものを装飾せず、サイズ、ウェイト、濃度、配置、余白で階層を示す。

## Surface and Layers

主作業領域は、画用紙のような不透明で安定した面、または黒色の水ガラスのような暗い面にする。ナビゲーション、操作面、補助面、前面の層には、周囲との関係と画面全体の連続性を保つために半透明を使ってよい。

レイヤーは役割で定義する。画面ごとに同じ役割が同じ面にまとまるとは限らず、層数や重なり順を一律に固定しない。

- 作業面：本文や主な編集対象を安定して支える。
- ナビゲーション・操作面：現在位置や操作対象を示す。
- 補助面：プロパティ、補足情報、選択肢などを支える。
- 前面：メニュー、ダイアログ、通知など、一時的な判断を前へ置く。

### Transparency and Blur

透明表現は、情報の階層、面の連続性、前後関係を支える場合に限って使う。

- 低濃度の面は、不透明度6〜8%を基準にする。
- 通常の前面・補助面は、不透明度44〜70%を基準にする。
- アクセント面は、不透明度82〜84%を基準にし、主要操作などの限定された用途だけに使う。
- ブラーは6〜10px程度の弱い範囲に留め、ヘッダー、補助パネル、メニュー、ダイアログなど前後関係が必要な場所だけに使う。
- 文字や操作対象の判別が弱くなる場合は、不透明度を上げるか不透明な面へ置き換える。

面の違いは、濃い塗り分けではなく、明度差、透明度差、境界、余白によって示す。

## Hierarchy and States

画面上の状態は、見た目の変化だけでなく、何が起きているかを明確に伝える。

- 現在位置は、周囲との十分な濃度差または境界で示す。
- 選択状態は、低濃度の面、境界、文字の強さを組み合わせて示す。
- ホバー状態は、操作可能であることを静かに補足する。
- キーボードフォーカスは、選択状態と区別できる明確な輪郭で示す。
- 無効状態は、濃度を下げるだけでなく、操作できないことが分かる表現を併用する。
- 主要操作と補助操作は、橙色などの色だけに頼らず、濃度、境界、サイズ、配置も組み合わせて区別する。

## Motion

動きは、状態の変化と要素の関係を伝えるために使う。

| 用途 | 時間の範囲 | 使い方 |
|------|------------|--------|
| `feedback` | 100〜180ms | ホバー、フォーカス、軽い応答 |
| `surface` | 180〜280ms | 面の切り替え、展開、前面表示 |
| `relationship` | 240〜360ms | 要素の移動や関係の変化 |

通常画面のボタンやスイッチは、ポインターを重ねても補足ツールチップやOS標準の説明を表示しない。右クリックで開くメニュー内のアイコンボタンに限り、ボタン中央の上側へ、画用紙色の明るい角丸面、深い黒の文字、面と三角を連続して囲む細いアウトライン、ボタンを指す三角として補足ツールチップを表示する。ポインターが400ms留まった後に表示し、意図せず通過しただけでは表示しない。表示時は不透明度を0から1、拡大率を0.85から1へ変化させ、変形には短い弾性を付ける。ポインターが離れた時は遅延を設けず、100msで元へ戻す。キーボードフォーカスでは待ち時間を設けずに同じ表示状態へ移り、動きを減らす設定では変化時間を短縮する。

急激な跳ね返り、過度な弾性、大きな移動、長いアニメーションを避ける。動きは滑らかで、静かで、操作に対する軽い応答として感じられる速さにする。

設定のオン／オフスイッチは、状態に応じてスイッチへ `on` 状態を付け、ノブを約400msの短い弾性移動で切り替える。ノブの移動はスイッチ内部の22pxに限定し、トラックとノブの色は約300msで同時に切り替える。

動きを減らす設定が有効な場合は、不要なアニメーションを停止または簡略化する。

## Texture and Light

光と質感は、素材とレイヤーの認識を助ける範囲に限定する。

画用紙は、わずかに暖かい色と均一で控えめな質感を持つ。黒色と橙色の水ガラスは、薄い透過、面の縁に沿う弱い反射、上部の細いハイライト、浅い色の重なりを持つ素材として扱う。文字や操作対象の輪郭がぼやけるほどの屈折や光沢は使わない。

質感は、軽い階調、透明度、明度差として表現する。汚れ、筆致、強いノイズ、金属光沢、鏡面反射、強いグローなどを再現しない。

## Avoided Expressions

- 用途のないアクセントパレットの色段階
- 意味や役割のない橙色の広い面塗り
- 装飾のためのグラデーション
- ネオン、グロー、強い反射
- 強い光沢、強い屈折、鏡面反射
- 強い影や過度なカード化
- 紙の汚れ、筆致、強いノイズ
- 文化や様式を想起させる装飾
- 情報を判別しにくくする透明表現
- 注意を奪う大きな動きや長いアニメーション
- 役割を説明できない装飾、ラベル、アイコン、コンテナ

## Layout

### Spacing

Spacingは4pxを基準とした段階で定義する。値は用途と一緒に使い、用途のない段階は追加しない。

| トークン | 値 | 用途 |
|----------|----|------|
| `--space-1` | 4px | アイコンとラベルなどの近接 |
| `--space-2` | 8px | ラベルと操作対象などの近接 |
| `--space-3` | 12px | 小さなコンポーネント内の間隔 |
| `--space-4` | 16px | 標準的なコンポーネント内の間隔 |
| `--space-5` | 24px | 情報グループ内の間隔 |
| `--space-6` | 32px | グループ間の間隔 |
| `--space-7` | 48px | セクション内の大きな間隔 |
| `--space-8` | 64px | 主要領域間の間隔 |

### Application Structure

```text
Rail
Primary Sidebar
Tabbed Main Area
Right Panel
```

情報構造を常時可視化する。

---

### Grid Alignment

Breakpoint、列数、Gutter、Marginは画面ごとに決める。固定Gridをすべての画面へ一律に適用せず、内容の読みやすさ、操作のしやすさ、現在の情報構造を優先する。

推奨比率は定義しない。見た目の配分より、情報の役割と状態の識別を優先する。

---

### Editor Width

本文の行長と行間を優先し、画面幅、表示モード、ユーザー設定に応じて調整する。固定値をすべての画面へ一律に適用しない。

## Navigation

### Active Item

選択状態は淡いSurfaceまたはLiquidレイヤーと、Charcoalのインジケーターで表現する。

```css
width: 3px;
background: var(--color-primary);
```

現在位置、操作対象、選択状態は十分に分かる濃度で示す。

---

### Outline Navigation

アウトラインは番号を付けず、控えめな目次として表示する。

見出し階層は左余白と細い境界線で表現する。

番号ブロックや大きな装飾見出しは使用しない。

例:

```text
Overview
  Goals
  References
    Notes
```

---

## Components

### Tabs

- 明確な境界線
- 過剰な影を使用しない
- アクティブ状態はCharcoalの細線、Primary Text、Selected Surfaceで表現
- 非アクティブ状態はSecondary TextとHairline Borderで表現

---

### Segmented Controls

- フォントなど相互排他的な設定値は、文字ボタンと単一の橙色ピルで表現する
- 橙色ピルは選択項目の実際の位置と幅へ、幅を変えながら400msで滑らかに移動する
- 選択中の文字は橙色上でも読める黒色、非選択の文字はSecondary Textで表示する
- 全選択肢を薄い境界と補助面のカプセルで囲み、その内側を選択中の橙色ピルが移動する
- キーボードフォーカスはピルと区別できる輪郭で示し、動きを減らす設定では移動を即時にする

### Theme Switch

- すべての対応OSで、タイトルバー左側の同じ位置に52×28pxのテーマ切り替えトグルを置く
- macOSではウインドウ操作ボタンと重ならない位置を保つ
- 戻る・進むボタンはテーマ切り替えの右側へ、小型の左右一組として置く
- 移動先がないボタンは、形を維持したまま前景の濃度を下げて無効状態を示す
- ライト状態は橙色の太陽、ダーク状態は作業面色の月で示し、トラック、輪郭、フォーカスはテーマの既存トークンを使う

### Language Switch

- アプリ全体のステータスバー右端に52×28pxの表示言語切り替えトグルを置く
- テーマ切り替えと同じトラック、22pxの丸いつまみ、色、フォーカス輪郭、移動時間を使う
- 日本語ではつまみを左に置いて `JP`、英語では右に置いて `EN` をつまみ内へ表示する
- キーボード操作と現在状態を支援技術へ伝え、動きを減らす設定では切り替えを即時にする

---

### Tables

- グリッドを強調
- 行間を広く確保
- 情報密度を保ちながら読みやすさを優先

---

### Buttons

Primary:

```css
background: var(--color-accent-surface);
border: 1px solid var(--color-accent);
color: var(--color-primary);
```

Secondary:

```css
background: var(--color-overlay);
border: 1px solid var(--color-border);
```

破壊的操作、警告、成功、情報などを示す場合だけFunctional Colorsを使う。

---

### Card

Cardは、独立した情報のまとまりを示す必要がある場合だけ使う。

- 境界は細く、情報より強く見せない。
- 影は使わないか、弱くする。
- すべての情報をCardで囲まず、余白、面、境界によるグルーピングも使う。
- Cardを過度に入れ子にして、既存の情報構造を複雑にしない。

---

### Accessibility

- 本文、補助情報、操作対象は、Light ThemeとDark Themeの両方で読めるコントラストを確保する。
- 機能色だけで状態を伝えず、文言、形、境界、アイコン、位置などを併用する。
- キーボードフォーカスは、選択状態と区別できる明確な輪郭で示す。
- 無効状態は、濃度を下げるだけでなく、操作できないことが分かる表現を併用する。
- 動きを減らす設定が有効な場合は、不要なアニメーションを停止または簡略化する。

## Decorative Elements

装飾要素は機能的なレイアウトを補助する目的でのみ使用する。

装飾はLiquid Charcoalの質感を誇張するためではなく、情報構造、境界、視線誘導を支えるために使う。

許可される要素:

- 細い境界線
- Canvasより一段低いSurface
- 透明または半透明のLiquidレイヤー
- Charcoalのアクティブインジケーター
- 読みやすさを損なわない軽い階調

装飾が情報を上回ってはならない。

強いノイズ、汚れ加工、グロー、強いガラス反射、金属光沢、ネオン化、有彩色の強いアクセント、装飾のためのグラデーション、角丸や影による過度なカード化は使用しない。

和風化、書道、墨絵、筆致の表現、紙の汚れや強い凹凸の再現、液体を過剰に演出するアニメーションも使用しない。

## 基本レイアウト原則

- **左レール + プライマリサイドバー + タブ式メインエリア + 右パネル**を基本構造にする
- 情報を見る目的ごとの主要な表示単位を**ビュー**と呼び、**ファイルビュー / カードビュー / テーブルビュー / グラフビュー / スフィアビュー / クロニクルビュー**の6つを設ける
- 主要ビューのUI名は、表示幅を抑えるため**ファイル / カード / テーブル / グラフ / スフィア / クロニクル**とし、正式なビュー名と区別する
- **プライマリサイドバー**は、ファイルモードではワークスペースの作成・登録、ファイルツリー、検索、ピン留めを扱う
- **メインエリア**はタブを表示する作業領域とし、Markdownファイル、チャート、設定をタブとして開く
- **右パネル**はMarkdownファイルタブの補助領域として扱い、アウトライン、リンク一覧、復元版のいずれか一つを表示する

- ステータスバー右端にはJP・ENの表示言語切り替えを配置する

## 左レール

左端の縦アイコン領域。プライマリサイドバーの開閉、メインエリアへ開くパネルタブやチャートタブの入口を持つ。

| 入口 | 動作 |
|------|------|
| ファイル | ファイルビューのファイルサイドバーを開閉し、Markdownをファイルタブへ表示する |
| リフレッシュ | 設定のすぐ下に常設し、現在のワークスペースのファイルと表示を最新状態へ更新する。実行中は回転矢印を回転させる |
| カード | メインエリアにカードビューのチャートタブを開く |
| テーブル | メインエリアにフロントマター設定を統合したテーブルビューのチャートタブを開く |
| グラフ | メインエリアにグラフビューのチャートタブを開く |
| スフィア | メインエリアに3次元スフィアビューのチャートタブを開く |
| クロニクル | メインエリアにクロニクルビューの年表を表示するチャートタブを開く |
| 設定 | メインエリアに設定のパネルタブを開く |

- 各ボタンにはツールチップを表示する
- ファイルと設定は常に表示する
- カード、テーブル、グラフ、スフィア、クロニクルは、対応する機能トグルがオフの場合は入口を表示しない
- 初期状態では、ファイルと設定だけを表示する
- 登録済みワークスペースがある場合は、レール下部からワークスペースを切り替えられる

## タイトルバー

画面上部のタイトルバーには、ウィンドウ操作領域とアプリ全体の上部領域を置く。

- macOSのウィンドウ操作領域と左レール・プライマリサイドバーに重ならないようにする
- テーマ切り替えの右側に戻る・進むを並べ、直前または次のタブ表示へ移動できるようにする
- ペインごとのタブ列は、タイトルバーではなくメインエリア上部に表示する
- ファイル本文上部のファイルタイトル行には、ソースモード切り替えとフロントマター固定プロパティ追加操作を置く
- PDF保存はMarkdownファイルのタブ右クリックメニューに置く

## プライマリサイドバー

サイドバー下部の「ワークスペースを開く」行、ファイルツリーのフォルダ、または2件以上のMarkdown複数選択を右クリックした場合に「ファイル加工」を表示する。ワークスペース行では現在のワークスペース全体を対象にする。サブメニューにはタイトル一覧、目次、タグ別索引、ファイルのマージを並べる。単一ファイルでは表示せず、Markdown以外を含む複数選択では理由を添えて無効表示する。サブメニューはポインターとキーボードの両方で操作でき、画面端では開く方向を反転する。

左レールのファイル入口で開く可変幅のパネル。

ファイルモードで扱うこと:

- ワークスペースの作成、登録
- ワークスペース内のフォルダ・Markdownファイルのツリー表示
- ファイル / フォルダの作成、リネーム、移動、複製、削除
- ピン留めしたファイル / フォルダの表示
- 検索ボタンからのクイックスイッチャー起動

サイドバーは、macOSでは `⌘B`、Windows/Linuxでは `Ctrl+B`、またはレール上のファイルボタンで開閉できる。幅はユーザーがドラッグで変更できる。
登録済みワークスペースの切り替えは左レール下部で扱う。

## メインエリア

タブ式の作業領域。左右2ペイン分割に対応し、ペインごとに独立したタブ列を持つ。

### タブ種別

| 種別 | 内容 |
|------|------|
| ファイルタブ | ファイルビューとしてMarkdownファイルを編集する |
| パネルタブ | 設定などの補助画面を表示する |
| チャートタブ | `cards` でカードビュー、`table` でテーブルビュー、`graph` でグラフビュー、`sphere` でスフィアビュー、`chronicle` でクロニクルビューを表示する |

カードビューのチャートタブは、重複する画面見出しを置かず、左側のカード対象一覧と、右側の選択中カードを上端から表示する。一覧には既存表示名だけを並べ、選択中の項目を背景と境界で区別する。一覧の選択変更だけではファイルを開かず、右側の大きなカードを選択した時に元のMarkdownファイルを開く。右列はカードを内側へ置くための別の面や専用余白を持たず、暗い一枚板のカード自体を表示領域とする。カードは右列の横幅を使い、画像枠の横幅に合わせて全体の高さも伸縮させ、狭い幅で高さだけを維持しない。カード上部に明朝系の既存表示名、その下に細い二重枠で額装した代表画像、下部に将来文章を表示できる空欄を置く。画像は枠内で縦横比を保ち、周囲に余白を残して切り取らずに表示する。現段階では文章の入力元や自動抜粋を追加せず、欄だけを確保する。画像の読込前は控えめな補助面と短い読込表現、読込失敗時は画像と同じ寸法を維持した代替表示を使い、カード名と操作を残す。カード画像は選択中の1件だけ読み込み、連続選択時に古い読込結果を新しいカードへ反映しない。前回のカード選択、現在開いているカード対象、一覧の先頭の順で初期選択し、動きを減らす設定へ追従する。対象がない場合は、空であることと`card`プロパティへの画像パス指定方法を同じ空状態で案内する。

テーブルビューのチャートタブは、上部にファイル件数と「列」操作を置き、その下を表本体とする。初期状態ではファイル名列だけを表示し、追加したプロパティ列は同じ文字サイズと行高で整然と並べる。プロパティ列のヘッダーに並び替え専用のドラッグハンドルを置き、ドロップ位置を線で示して確定後の順序をワークスペース設定へ保存する。ファイル名列は左端に固定し、並び替え対象にしない。ヘッダーとファイル名列を固定し、列が表示幅を超えた場合は表本体だけを横スクロールする。ファイル名はリンクとして既存のフォーカス表現を使い、同名の場合だけ親フォルダを控えめな補助文字で示す。セルは長い値を1行で省略し、テーブル内では内容や警告にホバー補足を表示しない。YAMLが不正な行は削除せず、ファイル名の隣にテーマ色へ追従する警告を置く。列選択はツールバーから開く小さなポップオーバー内で検索とチェックを行い、セル編集、行選択、装飾的なカード化は加えない。大量行では表示範囲付近だけを描画し、行高を固定してスクロール位置が変動しないようにする。

固定プロパティの列見出しには、並び替えとは独立した設定操作を置く。設定ポップオーバーは列見出しへ接続して表示し、そのプロパティの意味と書き方を確認できる。`category` では同じポップオーバー内で候補値を追加・削除する。「列」ポップオーバーは、ワークスペース内に実在するプロパティの検索と表示列の選択に限定し、固定プロパティの別一覧は表示しない。独立したフロントマター設定ページや右側の専用設定パネルは追加しない。

グラフビューでノードをホバーすると、ノード周囲のハローが穏やかに明滅し、接続線上の小さな光が注目ノードから接続先へ一方向に流れる。

クロニクルビューのチャートタブでは、`chronicle` の期間を横軸に置く横軸年表を表示する。

スフィアビューのチャートタブでは、グラフビューと同じノードとリンクを暗色または明色の3次元空間へ表示する。リンク線は常時表示するが、密集時にもノードの輪郭を覆わない軽量な細線に抑え、注目ノードへ接続する線だけを立体的に太く強調する。リンクが多い場合は反発力とリンク距離を増やし、接続数の多いノードを追加で押し広げる。大規模または高密度な表示ではノード径を緩やかに縮めつつ、リンク線に埋もれない視認性と選択しやすい大きさを保つ。中心の過密化と少数ノードの極端な離脱を抑えながら球状に配置する。配置が落ち着いた後はノード座標を周期更新せず、表示中はクリック、ホバー、視点操作の対象判定に必要な描画を維持し、非表示または退避時に停止する。球の中心には傾きのない縦の中心軸と水平の赤道リングを、ライト・ダークテーマの両方で背景とノードから明確に識別できるアクセントオレンジの点線として、ノードの配置と描画を始める前に小さく表示する。点線は明るい芯線と幅広い半透明の外縁を重ね、密集したリンクの中でも輪郭を追える太さを画面上で保つ。ノードが球状に広がる間は、少数の外れ値を除いた全体の広がりへ同じガイドを滑らかに追従させ、配置完了時に作り直さない。その後もノードを背景側のガイドへ重ねる。ガイドはノード操作の対象にはしない。初回表示では右上から斜めに見下ろすクォーター視点を使い、縦軸と赤道リングの奥行きから立体であることを把握できるようにする。左ドラッグによる回転を主操作とし、右ドラッグではスフィアを見失わない範囲で視点の中心を上下左右へ移動する。右上には「視点をリセット」を置き、クォーター視点、球の中心、全体が収まる倍率へ戻せるようにする。ノードは1回目の左クリックで選択と接続先の強調を固定し、選択中の同じノードを再度左クリックしたときだけファイル表示またはタグ検索を実行する。背景の左クリックで選択を解除する。常時表示する文字は件数、操作案内、選択中ノード名に絞る。WebGLの失敗はこのタブ内の案内へ閉じ込める。
スフィアの赤道リングと中心軸の周囲には、天球の向きと奥行きを示す極細の点線グリッドを置く。グリッドは通常時に赤道リングより薄く、視点操作中に少し明るく表示し、ノードの意味や操作の判定には使わない。
スフィアのノードは参照数と被参照数の合計を5段階の星の等級として表現し、接続の多い星ほど大きく明瞭に、少ない星ほど小さく控えめにする。等級は描画データの更新時だけに求め、視点操作中は再計算しない。通常のリンク線は等級差を妨げないよう少し薄くし、選択中ノードと接続先の強調は等級より優先する。

年表は完全Canvas 2Dで描画し、単年を丸い節点、期間を開始・終了の節点を結ぶ線として示す。年ラベル、節点、期間線、ファイル名、年または期間の表示を同じCanvasへ描画する。

年は左を過去、右を未来として順序を固定する。年軸は年表項目の有無から独立した等間隔の時間軸とし、項目が存在する年だけを特別な目盛りとして表示しない。縦位置には意味を持たせず、項目、文字、期間線の衝突回避にだけ使う。

年表上部に期間スケールの段階式スライダーを置き、1年、5年、10年、50年、100年、500年単位を切り替える。期間スケールは一目盛りが表す年数と年同士の横方向の距離だけを変更し、ズーム率、文字や節点の大きさ、Markdown、ワークスペース設定へ保存しない。スケール変更時は画面中央の年を維持する。

年表を開いた直後は収束済みの静止配置を、固定の初期ズーム率で表示する。全体表示への自動調整と前回位置の復元は行わず、項目数の中央にある年を画面中央へ置く。配置は表示のたびに多少変わってよい。

空白ドラッグでは地図と同じように表示位置を移動し、解放後は慣性で移動する。年ラベルは画面上端の固定ヘッダーとして表示し、ヘッダーから下へ選択中の期間スケールに沿った縦ガイドを伸ばす。パン範囲は制限しない。ピンチまたは通常のホイール操作は、期間スケールを変えずにポインター位置を中心とした視野のズームだけを行う。縮小時は年ラベルを視認できる間隔へ間引くが、年軸の基本単位と項目の年位置は変えない。

年表は閲覧専用とする。項目は一時的にドラッグでき、近くの項目も反発で少し動くが、解放後または操作の中断後は対応年への引力で正しい位置へ戻り、Markdownには書き戻さない。縮小に応じてファイル名と年・期間表示を徐々に薄くし、ホバー中はノードと期間ラインを発光させ、項目名を14pxへ拡大してカーソルの少し上へ表示する。移動を伴わずにノード、期間線、ファイル名のいずれかをクリックした場合は、クロニクルビューのタブを残して同じペインへファイルタブを開く。

項目には表示ごとに異なる色を割り当て、一回の表示中は維持する。表示可能な項目がない場合は案内文を表示する。キーボードによるパン、ズーム、項目操作は提供しない。

### ファイルタイトル行

- ファイルタイトル
- ソースモード切り替え
- フロントマター固定プロパティ追加

ファイルタイトル行はファイルタブ本文上部に表示する。ファイル名変更はファイルタイトルで扱い、ファイル移動はファイルツリーの操作で扱う。分割表示と右パネル表示は、それぞれの対応ボタンまたはショートカットで切り替える。

## ファイルタブ

Markdownファイルを表示・編集するタブ。

**固定表示:**
- ファイルタイトル
- エディタ本体
- エディタ右上のフロントマター固定プロパティ追加 `+` ボタン
- ステータスバーの文字数・単語数

**表示モード:**
- ライブプレビュー
- ソースモード
- タイプライターモード

フロントマターがある場合は、ライブプレビューではファイル先頭のYAMLブロックを初期状態で折りたたんだフォームとして控えめに表示する。折りたたみ時はプロパティ件数を表示し、隠れたYAML行の空白や行番号は残さない。クリックで展開して値編集や削除を行える。展開中は行、入力欄、配列値、追加領域の高さと余白をコンパクトに揃え、本文より強い表組みに見せない。プロパティ行を主な区切りとし、単一値とクロニクルの入力面へ独立した外枠を重ねず、複数値だけを小さなチップで区別する。ソースモードではフォーム化せず、Markdown内のYAMLとして表示する。
エディタ右上の `+` ボタンはフロントマターの有無に関係なく同じ位置に表示し、固定プロパティ追加メニューを開く。
フロントマターがない場合は選択した固定プロパティ付きでファイル先頭にフロントマターを作成し、既存フロントマターがある場合はその末尾に追加する。
フロントマターの各プロパティ行には、左端にドラッグ専用の並び替えハンドルと、このファイルからそのプロパティを削除する操作を表示する。ハンドル列全体をつかめる範囲とし、一定距離を動かすと行のプレビューがポインターへ追従する。元の行を薄く残して差し込み線を表示し、ドラッグ中はエディタ内の行DOMを移動せず、ドロップ時だけYAMLへ順序を反映する。

---

## パネルタブ

Markdownファイル以外の作業画面をメインエリア内のタブとして表示する。

| パネル | 内容 |
|--------|------|
| 設定 | フォント、行間、最大幅、行番号、スペルチェック、機能トグルを管理する |

---

## 右パネル

ファイルタブの右側に開閉表示される補助領域。

- アウトラインパネル: 見出し一覧とジャンプ
- リンクパネル: バックリンク・アウトゴーイングリンク一覧
- 復元パネル: 現在開いているMarkdownファイルの保存前復元版
- 右パネル内では、アウトライン / リンク / 復元版のいずれか一つだけを表示する
- 右パネルのアウトラインとリンクは常に利用可能とし、それぞれの切り替え操作を表示する

フロントマターの一覧、固定プロパティ説明、`category` 候補管理は、メインエリアのテーブルビューで一体として扱う。個別ファイル内のフロントマター値編集はファイルタブ内のフロントマター表示で扱う。独立したフロントマター設定画面は設けない。ファイル加工は左レール下部の登録済みワークスペース項目、またはファイルツリーの対象を右クリックしたメニューで扱い、設定はメインエリアのパネルタブで扱う。

---

## 初回起動時

ワークスペースが未登録、または登録済みワークスペースはあるがアクティブワークスペースがない状態で起動したとき、プライマリサイドバーのファイルモードに初期設定ビューを表示する。

- 既存フォルダをワークスペースとして開く
- 新規ワークスペースを作成

新規ワークスペース作成時は、毎回ユーザーが保存場所を選ぶ。
既存フォルダを開く操作と新規ワークスペース作成操作は、どちらもダイアログをキャンセルした場合、現在のワークスペース登録状態を変えない。

# Navigation Flow

アプリの画面遷移・ビュー切り替えフローを定義するドキュメント。

**前提**: ナビゲーションは左レール、プライマリサイドバー、メインエリアのタブ操作で行う。Markdownファイル以外の作業画面も、メインエリアのタブとして開く。

この文書では、画面遷移やビュー切り替えの「流れ」を定義する。
ナビゲーション要素の機能仕様は [../features/navigation.md](../features/navigation.md)、画面内の配置はこの文書の「Screen Structure」を正とする。

---

## 起動フロー

```
アプリ起動
  │
  ├─ 前回ワークスペースIDが有効 → メイン画面（ワークスペース状態を読み込む）
  │
  └─ アクティブワークスペースなし → プライマリサイドバーのファイルモードに初回設定ビューを表示
        │
        ├─ 新規ワークスペース作成
        └─ 既存フォルダを開く
              │
              └─ → メイン画面
```

登録済みワークスペースがあっても、前回ワークスペースIDが登録一覧に存在しない場合は、アクティブワークスペースなしとして起動する。この場合、左レール下部には登録済みワークスペース一覧を表示できるが、プライマリサイドバー本体は初回設定ビューを表示する。

---

## 左レール

```
左レール
  ├─ ファイル        → ファイルサイドバーを開閉
  ├─ カード          → カードビューのチャートタブを開く（機能トグルON時のみ）
  ├─ テーブル        → フロントマター設定を統合したチャートタブを開く（機能トグルON時のみ）
  ├─ グラフ          → グラフビューのチャートタブを開く（機能トグルON時のみ）
  ├─ スフィア        → スフィアビューのチャートタブを開く（機能トグルON時のみ）
  ├─ クロニクル      → クロニクルビューの年表を表示するチャートタブを開く（機能トグルON時のみ）
  └─ 設定            → パネルタブを開く
```

既に同じパネルタブまたはチャートタブが開いている場合は、新規作成せずそのタブをアクティブにする。
ファイルと設定は常に表示する。カード、テーブル、グラフ、スフィア、クロニクルは、対応する機能トグルがオフの場合は左レールに表示しない。
初期状態では、ファイルと設定だけを表示する。

---

## プライマリサイドバー

プライマリサイドバーは、ファイルモードではワークスペースとMarkdownファイルの操作を扱う。

```
プライマリサイドバー
  ├─ ファイルモード
  │   ├─ ワークスペース作成 / 登録
  │   ├─ ファイルツリー
  │   ├─ ピン留め
  │   └─ 検索ボタン
```

macOSでは `⌘B`、Windows/Linuxでは `Ctrl+B`、または左レールのファイルボタンで開閉する。検索ボタンまたは検索コマンドを実行した場合は、クイックスイッチャーを開く。

---

## ファイルを開く

```
ファイルサイドバーでファイルをクリック
  │
  ├─ 新しいファイルタブで開く
  └─ 既に開いているファイルタブがある場合 → そのタブをアクティブにする
```

内部リンクをクリックした場合も、リンク先ファイルをファイルタブとして開く。リンク先が存在しない場合は新規ファイルを作成して開く。

---

## タブの操作

```
タブをクリック → そのタブをアクティブにする
タブを閉じる  → ペイン内の表示履歴を基準に直前に表示していたタブをアクティブにする
              → タブが0枚になった場合は新規ファイル作成画面を表示
閉じたタブを開く → 最後に閉じたタブを元のペインと位置へ戻してアクティブにする
タブを右クリック → 閉じたタブの履歴がある場合は「閉じたタブを開く」を選択できる
```

ペイン内の表示履歴は、タブをアクティブにした順に記録する。現在のタブを閉じたら、同じペインで直近に表示していた未クローズのタブへ戻る。閉じたタブはウィンドウ内の専用履歴へ最大20件保持し、コマンドパレットまたは `⌘⇧T` / `Ctrl+Shift+T` で新しいものから復元する。

タイトルバーの戻る・進むは、ファイル、画像、PDF、パネル、チャートを含むタブの閲覧順を前後に移動する。分割表示中は対象タブのペインも復元する。戻った後に別のタブへ移動した場合は、それより先の進む履歴を破棄し、閉じたタブは移動先から除外する。

---

## 分割表示

```
分割ボタンを押す → メインエリアが左右2ペインに分割
  │
  ├─ 左ペイン：元のタブをそのまま表示
  └─ 右ペイン：独立したタブ列を持つ

分割解除ボタンを押す → 1ペインに戻る（アクティブなペインのタブを表示）
```

分割表示中は、ペインごとに独立したタブ列を持つ。ファイルタブ、パネルタブ、チャートタブはいずれもペイン内のタブとして扱う。

---

## 右パネルの開閉

```
アウトラインパネルボタンを押す      → 右パネルにアウトラインを表示 / 閉じる
リンクパネルボタンを押す           → 右パネルにリンク一覧を表示 / 閉じる
復元パネルボタンを押す             → 右パネルに現在ファイルの保存前復元版を表示 / 閉じる
```

右パネルはファイルタブの補助領域であり、アウトライン、リンク一覧、復元版を同時表示しない。いずれかを開いている状態で別の表示を開いた場合は、右パネル内の表示内容を切り替える。
右パネルのアウトラインとリンクは常に利用可能とし、それぞれの切り替え操作を表示する。

---

## 現在ファイルの出力

```
ファイルタブの右クリックメニューでPDFとして保存を選ぶ
  │
  └─ 右クリックしたMarkdownファイルからPDFを生成
        │
        └─ 保存ダイアログで保存先を選んで保存
```

PDF保存は、Markdownファイルのタブ右クリックメニューから実行する。ユーザーが保存先を選ばない限りワークスペースや任意の保存先には残さない。

---

## ワークスペースの切り替え

```
左レール下部のワークスペース切替を操作
  │
  └─ 別のワークスペースを選択 → 開いているタブを閉じ、そのワークスペース状態を読み込む
```

ワークスペース切り替えでは、登録情報、ファイルツリー、ピン留めなどのワークスペース状態を読み込む。前ワークスペースで開いていたタブは持ち越さない。

---

## 新規ワークスペース作成

```
新規ワークスペース作成を選択
  │
  ├─ 保存ダイアログで作成先とフォルダ名を選択
  └─ 指定場所にワークスペースフォルダを作成して開く
```

保存場所は毎回ユーザーが選択する。
既存フォルダを開く操作や新規ワークスペース作成操作でダイアログをキャンセルした場合は、登録済みワークスペースやアクティブワークスペースを変更しない。
ただし、これらの操作ではダイアログ表示前に開きタブの保存確認を行うため、確認を通過した後にキャンセルした場合も、開きタブは閉じた状態になる。
