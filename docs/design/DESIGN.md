# Visual Design System

## Overview

このデザインシステムは、情報の構造化・可読性・集中体験を重視したナレッジワーク向けインターフェースである。

Relicでは、Liquid Charcoalのうち現在採用している原則と、その原則を画面構成、色、UI部品へ適用する仕様をこの文書の正本として扱う。

外部で更新される一般的なデザイン原則をリポジトリ内へ複製せず、Relicへ採用する内容だけをこの文書へ反映する。実装時の具体値と振る舞いは、この文書に定義したトークン、コンポーネント、画面仕様を基準にする。

Liquid Charcoalは、読み書きに集中できる作業画面を整えるデザイン原則である。

目的は、情報の読みやすさ、状態の識別、操作の明確さを保ちながら、静かで構造的かつ清潔感のある画面をつくることにある。

色や質感は主役ではない。Markdown本文、ファイル構造、タブ、アウトライン、プロパティなどの情報と操作を見分けるために使う。

Liquid Charcoalは、特定の文化、様式、流行、装飾表現を根拠にしない。

---

## Design Principles

- 情報と操作を、色や質感より優先する。
- 主作業領域は、情報が安定して読める不透明な面にする。
- ナビゲーション、操作面、補助面、前面の層は、画面の階層と連続性を保つために半透明の面を一貫して使ってよい。
- 深い黒は、本文、現在位置、主要操作など、焦点が必要な場所に限定する。
- Light ThemeとDark Themeを同等に扱う。
- 有彩色のアクセントを基礎トークンとして持たない。
- 機能色は、エラー、警告、成功、情報を伝える場合に限って使う。
- 情報の区別には、余白、配置、明度差、透明度差、細い境界を使う。
- 広めの余白、精密な境界、軽い面の重なりによって、清潔で新鮮な印象をつくる。
- 透明感や静けさによって可読性と操作性を損なわない。
- 既存の情報構造を、装飾や不要なコンテナで複雑にしない。

## Themes

Light ThemeとDark Themeは、同じ情報の役割と状態をそれぞれの明度環境で表現する。

Dark ThemeはLight Themeの単純な色反転ではない。どちらのテーマでも、本文、補助情報、現在位置、選択状態、操作対象、境界を明確に区別する。

Dark Themeでは、構造と焦点の役割を明るい前景が担う。

| 項目 | Light Theme | Dark Theme |
|------|-------------|------------|
| 作業面 | 白い面 | 暗い面 |
| 主な前景 | 深い黒 | 明るい前景 |
| 補助面 | 作業面よりわずかに暗い面 | 作業面よりわずかに明るい面 |
| 半透明の面 | 白を含む低濃度の面 | 暗い色を含む低濃度の面 |
| 境界 | 淡い灰色の細線 | 暗部で判別できる明度差のある細線 |
| 状態の区別 | 濃度、境界、余白、配置 | 濃度、境界、余白、配置 |


---

### Corners

```css

```

検索欄、ボタン、選択行、カード、モーダル、オーバーレイなどの操作対象は、コンポーネントとプラットフォームに応じて控えめな角丸を使う。

アクセントは有彩色ではなく、Charcoalの濃度、境界、余白、状態差によって作る。

### Functional Colors

有彩色は、状態を伝える必要がある場合に限って使う。

機能色はLiquid Charcoalの印象を作るための色ではなく、エラー、警告、成功、情報を誤認させないための補助である。

| 用途 | Light Theme | Dark Theme | 使い方 |
|------|-------------|------------|--------|
| Danger | `#D92D20` | `#FF7A70` | エラー、削除、破壊的操作 |
| Warning | `#B54708` | `#FEC84B` | 警告、注意、未解決状態 |
| Success | `#067647` | `#47CD89` | 成功、完了、正常状態 |
| Info | `#155EEF` | `#84ADFF` | 補足情報、通知、案内 |

機能色は必要な要素に限定して使い、通常のナビゲーション、装飾、ブランド表現には使わない。機能色だけで状態を伝えず、文言、形、境界、アイコンなど、色以外の手掛かりを併用する。

---

### Color Usage

Canvasは作業面の基盤である。Light Themeでは白を基準にし、Dark Themeでは暗いCanvasを基準にする。

Liquidは透明・半透明の層である。情報のまとまり、重なり、選択状態、操作対象を示すために使う。

Charcoalは本文、見出し、リンク、アクティブ線、主要ボタン、現在位置に使う。

Charcoalは少なく使うことで意味を持つ。面の塗りつぶしには限定的に使う。

Grayは非アクティブなタブ、選択背景、補助情報、境界線に使う。役割は情報の階層を支えることであり、装飾的な塗り分けではない。

## Typography

タイポグラフィは、プラットフォームの標準を基本とする。

本文、見出し、補助情報の階層は、サイズ、ウェイト、濃度、配置、余白によって示す。文字そのものを装飾せず、長文を無理なく読み続けられる行長と行間を保つ。

小ささや薄さだけで情報を弱めない。補助情報であっても、必要な内容は読めるコントラストを確保する。

```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

### Heading

スタイル:

```css
font-weight: 700;
letter-spacing: 0;
```

---

### Section Labels

```css
font-size: 12px;
font-weight: 700;
letter-spacing: 0;
text-transform: none;
```

---

### Body

```css
font-size: 14px;
line-height: 1.7;
font-weight: 400;
```

## Surface and Layers

主作業領域は、不透明で安定した面にする。ナビゲーション、操作面、補助面、前面の層には、周囲との関係と画面全体の連続性を保つために半透明を使ってよい。

情報の階層、面の連続性、状態の識別を支える場合に使う。ヘッダー、補助パネル、メニュー、ダイアログなど、前後関係を示す必要がある領域では、弱いブラーを併用してよい。文字や操作対象の判別が弱くなる場合は、不透明度を上げるか不透明な面へ置き換える。

面の違いは、濃い塗り分けではなく、明度差、透明度差、境界、余白によって示す。

## Hierarchy and States

画面上の状態は、見た目の変化だけでなく、何が起きているかを明確に伝える。

- 現在位置は、周囲との十分な濃度差または境界で示す。
- 選択状態は、低濃度の面、境界、文字の強さを組み合わせて示す。
- ホバー状態は、操作可能であることを静かに補足する。
- キーボードフォーカスは、選択状態と区別できる明確な輪郭で示す。
- 無効状態は、濃度を下げるだけでなく、操作できないことが分かる表現を併用する。
- 主要操作と補助操作は、色の追加ではなく、濃度、境界、サイズ、配置で区別する。

---

## Motion

動きは、状態の変化と要素の関係を伝えるために使う。

急激な跳ね返り、過度な弾性、大きな移動、長いアニメーションを避ける。動きは滑らかで、静かで、短くし、操作に対する軽い応答として感じられる速さにする。

動きを減らす設定が有効な場合は、不要なアニメーションを停止または簡略化する。

---

## Texture and Light

光と質感は、素材とレイヤーの認識を助ける範囲に限定する。

半透明な面は自然な透過と弱い反射を持ち、深い黒は光を強く返さない落ち着いた面として扱う。

質感は、軽い階調、透明度、明度差として表現する。紙の繊維、汚れ、強いノイズ、金属光沢、濡れた質感などを再現しない。

## Avoided Expressions

- 有彩色による装飾的なアクセント
- 装飾のためのグラデーション
- ネオン、グロー、強い反射
- 金属、プラスチック、ガラスを強調する光沢
- 強い影や過度なカード化
- 紙の繊維、汚れ、筆致、強いノイズ
- 文化や様式を想起させる装飾
- 情報を判別しにくくする透明表現
- 注意を奪う大きな動きや長いアニメーション
- 役割を説明できない装飾、ラベル、アイコン、コンテナ

## Navigation

### Active Item

選択状態は淡いSurfaceまたはLiquidレイヤーと、Charcoalのインジケーターで表現する。

```css
width: 3px;
background: var(--color-primary);
```

現在位置、操作対象、選択状態は十分に分かる濃度で示す。

---

### Outline Navigation

アウトラインは番号を付けず、控えめな目次として表示する。

見出し階層は左余白と細い境界線で表現する。

番号ブロックや大きな装飾見出しは使用しない。

例:

```text
Overview
  Goals
  References
    Notes
```

## Components

### Tabs

- 明確な境界線
- 過剰な影を使用しない
- アクティブ状態はCharcoalの細線、Primary Text、Selected Surfaceで表現
- 非アクティブ状態はSecondary TextとHairline Borderで表現

---

### Tables

- グリッドを強調
- 行間を広く確保
- 情報密度を保ちながら読みやすさを優先

---

### Buttons

Primary:

```css
background: var(--color-primary);
color: var(--color-bg);
```

Secondary:

```css
background: var(--color-overlay);
border: 1px solid var(--color-border);
```

破壊的操作、警告、成功、情報などを示す場合だけFunctional Colorsを使う。

## Decorative Elements

装飾要素は機能的なレイアウトを補助する目的でのみ使用する。

装飾はLiquid Charcoalの質感を誇張するためではなく、情報構造、境界、視線誘導を支えるために使う。

許可される要素:

- 細い境界線
- Canvasより一段低いSurface
- 透明または半透明のLiquidレイヤー
- Charcoalのアクティブインジケーター
- 読みやすさを損なわない軽い階調

装飾が情報を上回ってはならない。

強いノイズ、汚れ加工、グロー、ガラス反射、金属光沢、ネオン化、有彩色の強いアクセント、装飾のためのグラデーション、角丸や影による過度なカード化は使用しない。

和風化、書道、墨絵、筆致の表現、紙の繊維や凹凸の再現、液体を過剰に演出するアニメーションも使用しない。

## 基本レイアウト原則

- **左レール + プライマリサイドバー + タブ式メインエリア + 右パネル**を基本構造にする
- **プライマリサイドバー**は、ファイルモードではワークスペースの作成・登録、ファイルツリー、検索、ピン留めを扱う
- **メインエリア**はタブを表示する作業領域とし、Markdownファイル、グラフ、ファイル加工、フロントマター設定、設定をタブとして開く
- **右パネル**はMarkdownファイルタブの補助領域として扱い、アウトライン、リンク一覧、復元版のいずれか一つを表示する

---

## 全体レイアウト

```
┌──────────────────────────────────────────────────────────────┐
│  タイトルバー                                                 │
├────┬──────────────┬────────────────────────┬───────────┤
│左  │              │ ペインのタブ行          │           │
│レ  │ プライマリ   │────────────────────────│ 右パネル  │
│ー  │ サイドバー   │ ファイルタイトル行      │ 開閉式    │
│ル  │ （可変幅）   │────────────────────────│           │
│    │              │ メインエリア            │           │
│    │              │ ファイル / パネル /     │           │
│    │              │ チャート                │           │
├────┴──────────────┴────────────────────────┴───────────┤
│  ステータスバー                                               │
└──────────────────────────────────────────────────────────────┘
```

- メインウィンドウの初期サイズは `1240 x 820` とする
- メインウィンドウの最小サイズは `640 x 400` とする
- macOSでは隠しタイトルバーを使い、ウィンドウ操作ボタンは左上に残す
- Windowsではアプリ用アイコンを設定し、メニューバーは自動で隠す

## 左レール

左端の縦アイコン領域。プライマリサイドバーの開閉、メインエリアへ開くパネルタブやチャートタブの入口を持つ。

| 入口 | 動作 |
|------|------|
| ファイル | ファイルサイドバーを開閉する |
| ファイル加工 | メインエリアにファイル加工ツールのパネルタブを開く |
| フロントマター | メインエリアにフロントマター設定のパネルタブを開く |
| グラフビュー | メインエリアにグラフビューチャートタブを開く |
| 設定 | メインエリアに設定のパネルタブを開く |

- 各ボタンにはツールチップを表示する
- ファイル、グラフビュー、設定は常に表示する
- ファイル加工、フロントマターは、対応する機能トグルがオフの場合は入口を表示しない
- 初期状態では、ファイル、グラフビュー、設定を表示する
- 初期状態では、ファイル加工、フロントマターを表示しない
- 登録済みワークスペースがある場合は、レール下部からワークスペースを切り替えられる

## タイトルバー

画面上部のタイトルバーには、ウィンドウ操作領域とアプリ全体の上部領域を置く。

- macOSのウィンドウ操作領域と左レール・プライマリサイドバーに重ならないようにする
- ペインごとのタブ列は、タイトルバーではなくメインエリア上部に表示する
- ファイル本文上部のファイルタイトル行には、ソースモード切り替えとフロントマター固定プロパティ追加操作を置く
- PDF保存はMarkdownファイルのタブ右クリックメニューに置く

---

## プライマリサイドバー

左レールのファイル入口で開く可変幅のパネル。

ファイルモードで扱うこと:

- ワークスペースの作成、登録
- ワークスペース内のフォルダ・Markdownファイルのツリー表示
- ファイル / フォルダの作成、リネーム、移動、複製、削除
- ピン留めしたファイル / フォルダの表示
- 検索ボタンからのクイックスイッチャー起動

サイドバーは、macOSでは `⌘B`、Windows/Linuxでは `Ctrl+B`、またはレール上のファイルボタンで開閉できる。幅はユーザーがドラッグで変更できる。
登録済みワークスペースの切り替えは左レール下部で扱う。

---

## メインエリア

タブ式の作業領域。左右2ペイン分割に対応し、ペインごとに独立したタブ列を持つ。

### タブ種別

| 種別 | 内容 |
|------|------|
| ファイルタブ | Markdownファイルを編集する |
| パネルタブ | ファイル加工、フロントマター設定、設定を表示する |
| チャートタブ | `graph` でワークスペースのMarkdown、タグ、添付画像のリンクグラフを表示する |

右パネルの年表では、`chronicle` の単年または期間を上から下へ時系列順に並べる。現在編集中のファイルを強調し、`chronicle` 入力へ即時追従して一覧上の位置を更新する。項目は等間隔で配置し、項目間の年数差を補助表示する。
塗りの幅がラベル表示に足りない場合、ラベルは表示中の横軸範囲内で塗りの近くに表示する。

### ファイルタイトル行

- ファイルタイトル
- ソースモード切り替え
- フロントマター固定プロパティ追加

ファイルタイトル行はファイルタブ本文上部に表示する。ファイル名変更はファイルタイトルで扱い、ファイル移動はファイルツリーの操作で扱う。分割表示と右パネル表示は、それぞれの対応ボタンまたはショートカットで切り替える。

---

## ファイルタブ

Markdownファイルを表示・編集するタブ。

**固定表示:**
- ファイルタイトル
- エディタ本体
- エディタ右上のフロントマター固定プロパティ追加 `+` ボタン
- ステータスバーの文字数・単語数

**表示モード:**
- ライブプレビュー
- ソースモード
- タイプライターモード

フロントマターがある場合は、ライブプレビューではファイル先頭のYAMLブロックを初期状態で折りたたんだフォームとして控えめに表示する。折りたたみ時はプロパティ件数を表示し、隠れたYAML行の空白や行番号は残さない。クリックで展開して値編集や削除を行える。ソースモードではフォーム化せず、Markdown内のYAMLとして表示する。
エディタ右上の `+` ボタンはフロントマターの有無に関係なく同じ位置に表示し、固定プロパティ追加メニューを開く。
フロントマターがない場合は選択した固定プロパティ付きでファイル先頭にフロントマターを作成し、既存フロントマターがある場合はその末尾に追加する。
フロントマターの各プロパティ行には、このファイルからそのプロパティを削除する操作を表示する。

---

## パネルタブ

Markdownファイル以外の作業画面をメインエリア内のタブとして表示する。

| パネル | 内容 |
|--------|------|
| ファイル加工 | タイトル一覧、目次生成、タグ別索引、フォルダ内マージを、設定入力なしの実行アクションとして扱う |
| フロントマター設定 | 固定プロパティを常時確認できる一覧と `category` 候補管理を表示し、YAML書式例は補助情報として開閉表示する |
| 年表 | `chronicle` を単一時間軸へ縦に並べ、編集中ファイルの位置を表示する |
| 設定 | テーマ、フォント、行間、最大幅、言語、行番号、スペルチェック、機能トグルを管理する |

---

## 右パネル

ファイルタブの右側に開閉表示される補助領域。

- アウトラインパネル: 見出し一覧とジャンプ
- リンクパネル: バックリンク・アウトゴーイングリンク一覧
- 復元パネル: 現在開いているMarkdownファイルの保存前復元版
- 右パネル内では、アウトライン / リンク / 復元版のいずれか一つだけを表示する
- 右パネルのアウトライン機能トグルがオフの場合はアウトライン切り替え操作を表示せず、右パネルのリンク機能トグルがオフの場合はリンク切り替え操作を表示しない
- フロントマター機能トグルがオフの場合は、左レールのフロントマター設定入口を表示しない

フロントマターの固定プロパティ説明や `category` 候補管理は、メインエリアのフロントマター設定パネルタブで扱う。個別ファイル内のフロントマター値編集はファイルタブ内のフロントマター表示で扱う。ファイル加工、設定も右パネルではなく、メインエリアのパネルタブで扱う。

---

## 初回起動時

ワークスペースが未登録、または登録済みワークスペースはあるがアクティブワークスペースがない状態で起動したとき、プライマリサイドバーのファイルモードに初期設定ビューを表示する。

- 既存フォルダをワークスペースとして開く
- 新規ワークスペースを作成

新規ワークスペース作成時は、毎回ユーザーが保存場所を選ぶ。
既存フォルダを開く操作と新規ワークスペース作成操作は、どちらもダイアログをキャンセルした場合、現在のワークスペース登録状態を変えない。

---

# Navigation Flow

アプリの画面遷移・ビュー切り替えフローを定義するドキュメント。

**前提**: ナビゲーションは左レール、プライマリサイドバー、メインエリアのタブ操作で行う。Markdownファイル以外の作業画面も、メインエリアのタブとして開く。

この文書では、画面遷移やビュー切り替えの「流れ」を定義する。
ナビゲーション要素の機能仕様は [../features/navigation.md](../features/navigation.md)、画面内の配置はこの文書の「Screen Structure」を正とする。

---

## 起動フロー

```
アプリ起動
  │
  ├─ 前回ワークスペースIDが有効 → メイン画面（ワークスペース状態を読み込む）
  │
  └─ アクティブワークスペースなし → プライマリサイドバーのファイルモードに初回設定ビューを表示
        │
        ├─ 新規ワークスペース作成
        └─ 既存フォルダを開く
              │
              └─ → メイン画面
```

登録済みワークスペースがあっても、前回ワークスペースIDが登録一覧に存在しない場合は、アクティブワークスペースなしとして起動する。この場合、左レール下部には登録済みワークスペース一覧を表示できるが、プライマリサイドバー本体は初回設定ビューを表示する。

---

## 左レール

```
左レール
  ├─ ファイル        → ファイルサイドバーを開閉
  ├─ ファイル加工    → パネルタブを開く（機能トグルON時のみ）
  ├─ フロントマター  → パネルタブを開く（機能トグルON時のみ）
  ├─ グラフビュー    → グラフビューチャートタブを開く
  └─ 設定            → パネルタブを開く
```

既に同じパネルタブまたはチャートタブが開いている場合は、新規作成せずそのタブをアクティブにする。
ファイル、グラフビュー、設定は常に表示する。ファイル加工、フロントマターは、対応する機能トグルがオフの場合は左レールに表示しない。年表は左レールではなく右パネルの切り替えボタンから開く。
初期状態では、ファイル、グラフビュー、設定を表示し、ファイル加工、フロントマターは表示しない。

---

## プライマリサイドバー

プライマリサイドバーは、ファイルモードではワークスペースとMarkdownファイルの操作を扱う。

```
プライマリサイドバー
  ├─ ファイルモード
  │   ├─ ワークスペース作成 / 登録
  │   ├─ ファイルツリー
  │   ├─ ピン留め
  │   └─ 検索ボタン
```

macOSでは `⌘B`、Windows/Linuxでは `Ctrl+B`、または左レールのファイルボタンで開閉する。検索ボタンまたは検索コマンドを実行した場合は、クイックスイッチャーを開く。

---

## ファイルを開く

```
ファイルサイドバーでファイルをクリック
  │
  ├─ 新しいファイルタブで開く
  └─ 既に開いているファイルタブがある場合 → そのタブをアクティブにする
```

内部リンクをクリックした場合も、リンク先ファイルをファイルタブとして開く。リンク先が存在しない場合は新規ファイルを作成して開く。

---

## タブの操作

```
タブをクリック → そのタブをアクティブにする
タブを閉じる  → 表示履歴ベースで直前に表示していたタブをアクティブにする
              → タブが0枚になった場合は新規ファイル作成画面を表示
```

表示履歴は、タブをアクティブにした順に記録する。現在のタブを閉じたら、直近で表示していた未クローズのタブへ戻る。

---

## 分割表示

```
分割ボタンを押す → メインエリアが左右2ペインに分割
  │
  ├─ 左ペイン：元のタブをそのまま表示
  └─ 右ペイン：独立したタブ列を持つ

分割解除ボタンを押す → 1ペインに戻る（アクティブなペインのタブを表示）
```

分割表示中は、ペインごとに独立したタブ列を持つ。ファイルタブ、パネルタブ、チャートタブはいずれもペイン内のタブとして扱う。

---

## 右パネルの開閉

```
アウトラインパネルボタンを押す      → 右パネルにアウトラインを表示 / 閉じる
リンクパネルボタンを押す           → 右パネルにリンク一覧を表示 / 閉じる
復元パネルボタンを押す             → 右パネルに現在ファイルの保存前復元版を表示 / 閉じる
```

右パネルはファイルタブの補助領域であり、アウトライン、リンク一覧、復元版を同時表示しない。いずれかを開いている状態で別の表示を開いた場合は、右パネル内の表示内容を切り替える。
右パネルのアウトライン機能トグルがオフの場合はアウトライン切り替え操作を表示せず、右パネルのリンク機能トグルがオフの場合はリンク切り替え操作を表示しない。

---

## 現在ファイルの出力

```
ファイルタブの右クリックメニューでPDFとして保存を選ぶ
  │
  └─ 右クリックしたMarkdownファイルからPDFを生成
        │
        └─ 保存ダイアログで保存先を選んで保存
```

PDF保存は、Markdownファイルのタブ右クリックメニューから実行する。ユーザーが保存先を選ばない限りワークスペースや任意の保存先には残さない。

---

## ワークスペースの切り替え

```
左レール下部のワークスペース切替を操作
  │
  └─ 別のワークスペースを選択 → 開いているタブを閉じ、そのワークスペース状態を読み込む
```

ワークスペース切り替えでは、登録情報、ファイルツリー、ピン留めなどのワークスペース状態を読み込む。前ワークスペースで開いていたタブは持ち越さない。

---

## 新規ワークスペース作成

```
新規ワークスペース作成を選択
  │
  ├─ 保存ダイアログで作成先とフォルダ名を選択
  └─ 指定場所にワークスペースフォルダを作成して開く
```

保存場所は毎回ユーザーが選択する。
既存フォルダを開く操作や新規ワークスペース作成操作でダイアログをキャンセルした場合は、登録済みワークスペースやアクティブワークスペースを変更しない。
ただし、これらの操作ではダイアログ表示前に開きタブの保存確認を行うため、確認を通過した後にキャンセルした場合も、開きタブは閉じた状態になる。
