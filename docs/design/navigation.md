# design/navigation.md

アプリの画面遷移・ビュー切り替えフローを定義するドキュメント。

**前提**: ナビゲーションは左レール、プライマリサイドバー、Cowork用セカンダリサイドバー、メインエリアのタブ操作で行う。Markdownファイル以外の作業画面も、メインエリアのタブとして開く。

この文書では、画面遷移やビュー切り替えの「流れ」を定義する。
ナビゲーション要素の機能仕様は [../features/navigation.md](../features/navigation.md)、画面内の配置は [screens.md](screens.md) を正とする。

---

## 起動フロー

```
アプリ起動
  │
  ├─ 前回ワークスペースIDが有効 → メイン画面（ワークスペース状態を読み込む）
  │
  └─ アクティブワークスペースなし → プライマリサイドバーのファイルモードに初回設定ビューを表示
        │
        ├─ 新規ワークスペース作成
        └─ 既存フォルダを開く
              │
              └─ → メイン画面
```

登録済みワークスペースがあっても、前回ワークスペースIDが登録一覧に存在しない場合は、アクティブワークスペースなしとして起動する。この場合、左レール下部には登録済みワークスペース一覧を表示できるが、プライマリサイドバー本体は初回設定ビューを表示する。

---

## 左レール

```
左レール
  ├─ ファイル        → ファイルサイドバーを開閉
  ├─ Cowork          → AIチャット履歴を表示し、AIチャットを開く
  ├─ ファイル加工    → パネルタブを開く（機能トグルON時のみ）
  ├─ フロントマター  → パネルタブを開く（機能トグルON時のみ）
  ├─ 暦設定          → パネルタブを開く（機能トグルON時のみ）
  ├─ 年表            → 年表チャートタブを開く（機能トグルON時のみ）
  ├─ カレンダー      → 日付チャートタブを開く（機能トグルON時のみ）
  └─ 設定            → パネルタブを開く
```

既に同じパネルタブまたはチャートタブが開いている場合は、新規作成せずそのタブをアクティブにする。
ファイル、設定は常に表示する。Cowork、ファイル加工、フロントマター、暦設定、年表、カレンダーは、対応する機能トグルがオフの場合は左レールに表示しない。
初期状態では、ファイル、Cowork、カレンダー、設定を表示し、ファイル加工、フロントマター、暦設定、年表は表示しない。

## プライマリサイドバー

プライマリサイドバーは、ファイルモードではワークスペースとMarkdownファイルの操作を扱い、CoworkモードではAIチャット履歴を扱う。

```
プライマリサイドバー
  ├─ ファイルモード
  │   ├─ ワークスペース作成 / 登録
  │   ├─ ファイルツリー
  │   ├─ ピン留め
  │   └─ 検索
  └─ Coworkモード
      ├─ AIチャット履歴
      ├─ 新規チャット
      ├─ チャット削除確認
      └─ AI使用量表示
```

macOSでは `⌘B`、Windows/Linuxでは `Ctrl+B`、または左レールのファイルボタンで開閉する。検索コマンドを実行した場合は、ファイルサイドバーを開き、検索入力へフォーカスする。

---

## Cowork

```
左レールのCoworkボタンを押す
  │
  ├─ プライマリサイドバーをAIチャット履歴表示にする
  └─ セカンダリサイドバーをAIチャット表示で開く
        │
        ├─ チャットを選ぶ → 選択中チャットの会話へ切り替える
        ├─ 新規チャット → 空のチャットを作って選択する
        ├─ メッセージ送信 → AI応答とMarkdown反映結果を会話に表示する
        └─ 応答生成中の中断 → 生成を止める
```

プライマリサイドバーを閉じても、セカンダリサイドバーは開いたままAIチャットを続けられる。セカンダリサイドバーを閉じた場合も、AIチャット履歴やAI系データは削除しない。

---

## ファイルを開く

```
ファイルサイドバーでファイルをクリック
  │
  ├─ 新しいファイルタブで開く
  └─ 既に開いているファイルタブがある場合 → そのタブをアクティブにする
```

内部リンクをクリックした場合も、リンク先ファイルをファイルタブとして開く。リンク先が存在しない場合は新規ファイルを作成して開く。

---

## タブの操作

```
タブをクリック → そのタブをアクティブにする
タブを閉じる  → 表示履歴ベースで直前に表示していたタブをアクティブにする
              → タブが0枚になった場合は新規ファイル作成画面を表示
```

表示履歴は、タブをアクティブにした順に記録する。現在のタブを閉じたら、直近で表示していた未クローズのタブへ戻る。

---

## 分割表示

```
分割ボタンを押す → メインエリアが左右2ペインに分割
  │
  ├─ 左ペイン：元のタブをそのまま表示
  └─ 右ペイン：独立したタブ列を持つ

分割解除ボタンを押す → 1ペインに戻る（アクティブなペインのタブを表示）
```

分割表示中は、ペインごとに独立したタブ列を持つ。ファイルタブ、パネルタブ、チャートタブはいずれもペイン内のタブとして扱う。

---

## 右パネルの開閉

```
アウトラインパネルボタンを押す → 右パネルにアウトラインを表示 / 閉じる
リンクパネルボタンを押す      → 右パネルにリンク一覧を表示 / 閉じる
```

右パネルはファイルタブの補助領域であり、アウトラインとリンク一覧を同時表示しない。片方を開いている状態でもう片方を開いた場合は、右パネル内の表示内容を切り替える。
右パネルのアウトライン機能トグルがオフの場合はアウトライン切り替え操作を表示せず、右パネルのリンク機能トグルがオフの場合はリンク切り替え操作を表示しない。

---

## 現在ファイルの出力

```
ファイルタブの右クリックメニューで印刷を選ぶ
  │
  └─ 右クリックしたMarkdownファイルから一時PDFを生成
        │
        └─ Chromium PDFプレビューで開く

ファイルタブの右クリックメニューでPDFとして保存を選ぶ
  │
  └─ 右クリックしたMarkdownファイルからPDFを生成
        │
        └─ 保存ダイアログで保存先を選んで保存
```

印刷とPDF保存は、Markdownファイルのタブ右クリックメニューから実行する。印刷時に生成するPDFはプレビュー用の一時ファイルであり、ユーザーが保存先を選ばない限りワークスペースや任意の保存先には残さない。

---

## ワークスペースの切り替え

```
左レール下部のワークスペース切替を操作
  │
  └─ 別のワークスペースを選択 → 開いているタブを閉じ、そのワークスペース状態を読み込む
```

ワークスペース切り替えでは、登録情報、ファイルツリー、ピン留めなどのワークスペース状態を読み込む。前ワークスペースで開いていたタブは持ち越さない。

---

## 新規ワークスペース作成

```
新規ワークスペース作成を選択
  │
  ├─ 保存ダイアログで作成先とフォルダ名を選択
  └─ 指定場所にワークスペースフォルダを作成して開く
```

保存場所は毎回ユーザーが選択する。
既存フォルダを開く操作や新規ワークスペース作成操作でダイアログをキャンセルした場合は、登録済みワークスペースやアクティブワークスペースを変更しない。
ただし、これらの操作ではダイアログ表示前に開きタブの保存確認を行うため、確認を通過した後にキャンセルした場合も、開きタブは閉じた状態になる。
