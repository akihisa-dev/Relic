# design/navigation.md

アプリの画面遷移・ビュー切り替えフローを定義するドキュメント。

**前提**: ナビゲーションは左レール、ファイルサイドバー、メインエリアのタブ操作で行う。Markdownファイル以外の作業画面も、メインエリアのタブとして開く。

---

## 起動フロー

```
アプリ起動
  │
  ├─ ワークスペース登録済み → メイン画面（前回のタブ状態を復元）
  │
  └─ ワークスペース未登録 → ファイルサイドバーに初回設定ビューを表示
        │
        ├─ 新規ワークスペース作成
        └─ 既存フォルダを開く
              │
              └─ → メイン画面
```

---

## 左レール

```
左レール
  ├─ ファイル        → ファイルサイドバーを開閉
  ├─ ファイル加工    → パネルタブを開く
  ├─ フロントマター  → パネルタブを開く
  ├─ チャート        → 年表 / 日付チャートタブを開く
  └─ 設定            → パネルタブを開く
```

既に同じパネルタブまたはチャートタブが開いている場合は、新規作成せずそのタブをアクティブにする。
ファイル加工とフロントマターは、対応する機能トグルがオフの場合は左レールに表示しない。

---

## ファイルサイドバー

ファイルサイドバーは、ワークスペースとMarkdownファイルの操作を扱う。

```
ファイルサイドバー
  ├─ ワークスペース作成 / 登録
  ├─ ファイルツリー
  ├─ ピン留め
  └─ 検索
```

`⌘B` または左レールのファイルボタンで開閉する。検索コマンドを実行した場合は、ファイルサイドバーを開き、検索入力へフォーカスする。

---

## ファイルを開く

```
ファイルサイドバーでファイルをクリック
  │
  ├─ 新しいファイルタブで開く
  └─ 既に開いているファイルタブがある場合 → そのタブをアクティブにする
```

内部リンクをクリックした場合も、リンク先ファイルをファイルタブとして開く。リンク先が存在しない場合は新規ファイルを作成して開く。

---

## タブの操作

```
タブをクリック → そのタブをアクティブにする
タブを閉じる  → 表示履歴ベースで直前に表示していたタブをアクティブにする
              → タブが0枚になった場合は新規ファイル作成画面を表示
```

表示履歴は、タブをアクティブにした順に記録する。現在のタブを閉じたら、直近で表示していた未クローズのタブへ戻る。

---

## 分割表示

```
分割ボタンを押す → メインエリアが左右2ペインに分割
  │
  ├─ 左ペイン：元のタブをそのまま表示
  └─ 右ペイン：独立したタブ列を持つ

分割解除ボタンを押す → 1ペインに戻る（アクティブなペインのタブを表示）
```

分割表示中は、ペインごとに独立したタブ列を持つ。ファイルタブ、パネルタブ、チャートタブはいずれもペイン内のタブとして扱う。

---

## 右パネルの開閉

```
アウトラインパネルボタンを押す → 右パネルにアウトラインを表示 / 閉じる
リンクパネルボタンを押す      → 右パネルにリンク一覧を表示 / 閉じる
```

右パネルはファイルタブの補助領域であり、アウトラインとリンク一覧を同時表示しない。片方を開いている状態でもう片方を開いた場合は、右パネル内の表示内容を切り替える。
右パネル機能トグルがオフの場合は、上部操作バーの右パネル切り替え操作を表示しない。

---

## ワークスペースの切り替え

```
左レール下部のワークスペース切替を操作
  │
  └─ 別のワークスペースを選択 → そのワークスペースの前回の状態を復元する
```

---

## 新規ワークスペース作成

```
新規ワークスペース作成を選択
  │
  ├─ フォルダ名を入力
  ├─ 保存場所を選択
  └─ 指定場所にワークスペースフォルダを作成して開く
```

保存場所は毎回ユーザーが選択する。
