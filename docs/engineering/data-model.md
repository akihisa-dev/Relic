# engineering/data-model.md

アプリが扱う主なデータの構造を定義する。

---

## ワークスペース

ユーザーがアプリに登録した任意のローカルフォルダ1つ。保存場所はユーザーが自由に選択できる。

| フィールド | 内容 |
|-----------|------|
| id | アプリ設定内でワークスペースを識別するID |
| name | フォルダ名（＝ワークスペース名） |
| path | ローカルのフォルダパス |

### ワークスペース内フォルダ

Relicは、ワークスペース作成時または既存フォルダ登録時にRelic専用の必須フォルダを自動作成しない。ユーザーが作成した通常のフォルダとMarkdownファイルをそのまま扱う。

---

## ファイル

ワークスペース内の `.md` ファイル1つ。IPCで読み書きするときは、ファイル名、ワークスペース相対パス、Markdown本文全体を扱う。

| フィールド | 内容 |
|-----------|------|
| name | ファイル名（拡張子なし） |
| path | ワークスペースルートからの相対パス |
| content | フロントマターを含むMarkdown本文全体 |

`frontmatter` や本文部分は、`content` から必要に応じて解析する派生データとして扱う。
Relic専用の不可視メタデータや独自データ形式をMarkdown本文へ埋め込まない。

---

## Markdown画像記法

Markdown本文に書かれた `![alt](path)` 記法。Relicは画像管理アプリではないため、画像ファイル単体の整理・タグ付け・検索対象にはしない。ワークスペース内の相対パス画像だけ実画像として表示し、外部URLやワークスペース外参照はライブプレビューで `alt` またはパスを使ったプレースホルダーとして扱う。

| フィールド | 内容 |
|-----------|------|
| raw | Markdown本文中の画像記法 |
| alt | Markdown画像記法に書かれた代替テキスト |
| pathOrUrl | Markdown画像記法に書かれた相対パスまたはURL |

OS上の画像ファイルをMarkdown本文へドロップした場合は、画像ファイルをワークスペース内へ追加し、そのワークスペース相対パスを `pathOrUrl` に持つMarkdown画像記法を本文へ挿入する。追加された画像ファイル自体は、検索やタグ付けの正本データにはしない。

## PDF添付ファイル

ワークスペース内の `.pdf` ファイル。RelicはPDFをMarkdownファイルの正本データにはせず、サイドバーから開ける閲覧用添付ファイルとして扱う。PDF本文の検索、タグ付け、編集、注釈管理、Markdownへの変換は行わない。

| フィールド | 内容 |
|-----------|------|
| name | PDFファイル名 |
| path | ワークスペースルートからの相対パス |
| dataUrl | 表示時に生成する派生データ。PDFファイルには保存しない |

---

## ファイル埋め込み記法

Markdown本文に書かれた `![[...]]` 記法。RelicはMarkdown本文内の参照記法を正本として扱い、表示時に対象Markdownファイルを読み込む。埋め込み先の本文や読み込み状態は派生データであり、Markdownファイルとは別の永続データとして保存しない。

| フィールド | 内容 |
|-----------|------|
| raw | Markdown本文中の埋め込み記法 |
| target | 正規化後のワークスペース相対Markdownパス |
| state | 表示時の読み込み状態。読み込み中、読み込み済み、大きすぎる、エラーのいずれか |

埋め込み先は空文字、絶対パス、URL、ワークスペース外へ出るパス、`.md` 以外の拡張子を対象外とする。対象外の場合はファイル読み込みを行わず、元の記法をエラー表示として扱う。

---

## 図表コードブロック

Markdown本文中の `mermaid` / `d2` コードブロック。Relicはコードブロック内のソース文字列を正本として扱い、表示時にSVGへ変換する。変換結果や表示中のpan / zoom状態は永続化しない。

| フィールド | 内容 |
|-----------|------|
| language | `mermaid` または `d2` |
| source | コードブロック本文 |
| renderedSvg | 表示時に生成される派生データ。Markdownファイルには保存しない |

---

## ワークスペースファイルインデックス

Relicは、ワークスペース内のMarkdownファイルを速く扱うため、全Markdownファイルの更新型インデックスを持つ。
正本は常に実際の `.md` ファイルであり、インデックスはRelicの設定領域に保存する再作成可能な控えとして扱う。

初期実装では、各Markdownファイルについてワークスペース相対パス、表示名、サイズ、更新日時、読み取り状態、全文検索用の行単位テキストを保持する。
インデックスが壊れた場合は実ファイルから作り直す。

検索、バックリンク、タグ、別名、グラフ、年表表示用データは、同じワークスペースの要求中にファイルインデックスとMarkdown解析結果を共有する。
アプリ内でMarkdownを保存した場合は変更された相対パスだけを読み直し、外部リネームや変更パスを特定できない場合はワークスペース単位で再生成する。
共有中の索引や解析結果も再作成可能な派生データであり、Markdownファイルより新しい正本として扱わない。

---

## フロントマター

ファイル先頭の `---` で囲まれたYAMLブロック。アプリ設定が一律に付与する属性ではなく、ユーザーが任意のMarkdownファイルに後から追加・削除できるメタ情報として扱う。フィールドの詳細仕様は [features/frontmatter.md](../features/frontmatter.md) を参照。

| 種類 | フィールド例 | 内容 |
|------|------------|------|
| 固定 | aliases / category / tags / chronicle | Relicが意味を知っている予約フィールド。設定パネルで常時確認できる |
| 未認識 | （任意） | 既存ファイルや外部エディタで作成されたYAMLキー。壊さず保持し、テキストとして表示する |
| 旧登録済み | （ユーザーが命名） | 旧設定ファイルに保存済みの場合だけ互換的に読み込む入力能力定義。現行UIでは追加・編集しない |

現行UIでは、任意名のカスタムプロパティへ入力能力を割り当てる管理画面を持たない。固定プロパティは予約名として扱い、Relicがタグと年表で利用する。

固定プロパティも、デフォルトで各ファイルに作成しない。Markdownファイル内に存在する場合に、Relicがその意味を認識して利用する。

---

## フォルダ

ワークスペース内のディレクトリ。ファイルをグループ化する入れ物。

| フィールド | 内容 |
|-----------|------|
| name | フォルダ名 |
| path | ワークスペースルートからの相対パス |
| children | 中に含まれるファイル・フォルダの一覧 |

## 内部リンク

`[[ファイル名]]` / `![[ファイル名]]` 記法で定義されるファイル間の接続。ファイルに保存されるのはMarkdown本文中のリンク記法であり、リンク一覧とバックリンクは本文から生成する派生データとして扱う。
ファイルまたはフォルダのリネーム・移動時は、コードブロック内を除くMarkdown内の内部リンク記法を更新対象にする。
未リンク参照は、既存Markdownファイルの拡張子なしファイル名が通常テキストとして本文に出ている箇所を検索して生成する派生データとして扱う。候補をリンク化した場合だけ、該当するMarkdown本文を内部リンク記法へ書き換える。

| フィールド | 内容 |
|-----------|------|
| sourceFile | リンクが書かれているファイル |
| targetName | リンク先として指定されたファイル名 |
| targetFile | 実際に存在するリンク先ファイル（存在しない場合はnull） |
| resolved | リンク先ファイルが存在するか否か |

## ワークスペースグラフ

グラフビューとスフィアビューで共有するワークスペース全体の関係データ。実際のMarkdownファイルとフロントマターが正本であり、グラフは要求時に生成する派生データとして扱う。

| ノード種別 | 内容 |
|-----------|------|
| file | ワークスペース内のMarkdownファイル。ファイル名、相対パス、被参照数、参照数を持つ |
| tag | フロントマター `tags:` と、グラフ生成時に本文から検出した `#タグ`。本文中の `#タグ` は通常のタグ一覧・タグ検索には使わない |
| attachment | Markdown画像記法または画像のWiki埋め込みから参照された、ワークスペース内の対応画像ファイル |
| unresolved | 解決できなかったWikiリンクまたは相対Markdownリンクの参照先 |

リンクにはファイル間・ファイルと添付画像を結ぶ `link` と、ファイルとタグを結ぶ `tag` がある。同じ始点・終点・種別のリンクは件数をまとめて1件にする。コードブロック内の記法、外部URL、ワークスペース外の画像参照はグラフへ追加しない。

グラフビューの2次元座標とスフィアビューの3次元座標、各ビューの視点、ホバー・右クリックによる強調状態は表示中だけの状態であり、保存しない。フィルター、力学設定、色グループ、設定パネルの開閉状態、設定セクションの折りたたみだけをレンダラー側のブラウザー保存領域へ保存する。スフィアビューは共有データを複製して描画し、3次元ライブラリが付加する座標を共有データへ反映しない。

---

## タグ

ファイルに付与されたラベル。フロントマターの固定プロパティ `tags:` から収集する。本文中の `#タグ名` はタグとして扱わない。

| フィールド | 内容 |
|-----------|------|
| name | タグ名（`#` を除いた文字列） |
| files | そのタグが付いているファイルの一覧 |

## アプリ設定

ユーザーの設定値。Electronの `userData` 配下の `app-settings.json` に保存する。

| フィールド | 内容 |
|-----------|------|
| editorSettings | テーマ、言語、エディタフォント、フォントサイズ、行間、最大幅、行番号、スペルチェック、フロントマターの日付表示順 |
| featureToggles | ファイル加工ツール・フロントマター設定・グラフ・スフィア・年表・右パネルのアウトラインとリンクのオン/オフ |
| userDefinedFields | 旧設定ファイルに保存済みの場合だけ互換的に読み込むカスタムフロントマターフィールドの一覧。現行UIでは管理しない |
| frontmatterTemplates | アプリ設定に保存されるフロントマターテンプレート定義（現行UIでは管理画面なし） |
| lastWorkspaceId | 前回開いていたワークスペースのID |
| workspaces | アプリに登録済みのワークスペース一覧 |

`editorSettings.theme` は `system`、`light`、`dark` のいずれかを保存する。`system` の場合はルート要素に `data-theme` を付けず、`light` / `dark` の場合だけ固定値として `data-theme` を設定する。
`editorSettings.language` は `system`、`en`、`ja` のいずれかを保存する。`system` の場合は実行環境の言語コードが `ja` で始まると日本語、それ以外は英語として扱う。

`featureToggles` は、左レール、右パネル表示、補助機能の入口表示を制御する。ファイル、設定はトグル対象にしない。
既存設定に値がない場合、ファイル加工ツール、フロントマター、スフィア、年表はオフ、グラフ、右パネルのアウトラインとリンクはオンとして扱う。

`frontmatterTemplates` は `{ name, fieldNames }` の配列として保存する。現行UIではテンプレート管理画面も適用操作も表示しないが、設定ファイル上の値を読む場合は、名前が空でないもの、同名が重複しないもの、1件以上の有効なフィールド名を持つものだけを復元する。

`workspaces` は、IDが英数字・`_`・`-` だけで構成され、名前が空でなく、パスが正規化済み絶対パスであるものだけを登録済みワークスペースとして復元する。
重複IDや危険なID、相対パス、`..` を含む未正規化パスは読み込み時に除外する。
`lastWorkspaceId` は、復元後の `workspaces` に存在するIDだけを有効とする。

## ワークスペース設定

ワークスペースごとに異なる設定値。Electronの `userData` 配下の `workspaces/{workspaceId}.json` に、ワークスペースと紐付けて保存する。

| フィールド | 内容 |
|-----------|------|
| workspacePath | 対象ワークスペースのパス |
| pinnedPaths | ファイルサイドバーでピン留めしているワークスペース相対パス |
| charts | クロニクルビューの対象ファイル設定 |
| frontmatterCategoryChoices | `category` 固定プロパティの候補値 |

`frontmatterCategoryChoices` はワークスペースごとの候補値として保存する。
読み込み時に、非文字列、空文字、前後空白だけの値、重複値は除外する。
既存Markdownに候補外の `category` 値がある場合も、Markdown本文の値は変更しない。

ワークスペース別設定ファイル名に使う `workspaceId` は、英数字・`_`・`-` だけを有効とする。
危険なIDを指定した読み込みは初期設定として扱い、保存は失敗させる。
`pinnedPaths` と `charts.filePaths` は、読み込み時にワークスペース相対パスだけへ正規化し、重複を除外する。
旧 `ganttCharts` キーは `charts` として読み込むが、保存時は `charts` だけを書き込む。
チャート設定はクロニクルビューの固定1件として復元し、保存済みの表示名は復元対象にしない。

## タブと画面状態

タブ、ペイン、サイドバー、右パネルなどの画面状態はレンダラーのZustand storeで扱う。

| 種類 | 内容 |
|------|------|
| ファイルタブ | Markdownファイルの `name`、`path`、`content` を持つ編集タブ |
| パネルタブ | ファイル加工、フロントマター設定、設定を表示するタブ |
| PDFタブ | PDF添付ファイルの `name`、`path` を持つ閲覧タブ |
| チャートタブ | グラフビュー、スフィアビュー、またはクロニクルビューを表示するタブ |
| ペイン | 左右分割時のタブ列とアクティブタブ、表示履歴 |
| UI状態 | プライマリサイドバー、右パネル、タイプライターモードなどの開閉・選択状態 |

同じタブIDは左右ペインの両方に含まれる場合がある。
タブ本体は `tabs` に1件だけ保持し、片方のペインで閉じても他方のペインに残っている間は削除しない。
分割解除時は右ペインだけにあるタブを左ペインへ統合し、右ペイン状態は空に戻す。

## チャート表示用データ

チャートはMarkdownファイルとフロントマターから生成する派生データであり、Markdown本文とは別の永続データとしてワークスペース内に保存しない。

| 種類 | 内容 |
|------|------|
| クロニクルビューの年表 | `chronicle` プロパティから生成する単年または期間のCanvas描画項目 |

チャート表示用データは、読み込み時に壊れたMarkdownファイルや無効な年表値をスキップして生成する。

## エンティティ間の関係

```
Workspace
  ├── Folder（ネスト可）
  │     └── File
  └── File
        ├── Frontmatter
        │     └── Tag（tags: フィールド）
        └── InternalLink → File（リンク先）
             └── Backlink / OutgoingLink（派生データ）

MarkdownImageSyntax
  └── File（Markdown本文中の記法。独立した永続モデルは持たない）

PdfAttachment
  └── Workspace（ワークスペース内の添付ファイル。検索・タグ付け対象にはしない）

DiagramCodeBlock
  └── File（Markdown本文中のコードブロック。SVGと表示状態は派生データ）
```
