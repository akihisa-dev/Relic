# engineering/architecture.md

Relicのアーキテクチャ全体像。主要コンポーネントの構成と責務を定義する。

---

## 全体構成

Electron アプリは「メインプロセス」「preload境界」「レンダラープロセス」に分かれる。

```
┌─────────────────────────────────────────────┐
│              Renderer Process               │
│  左レール・サイドバー・AIチャット・タブ・右パネル │
│  （CodeMirror 6 / 画面描画）                 │
├─────────────────────────────────────────────┤
│              Preload / contextBridge         │
│  window.relic API と型付きIPC呼び出し          │
├─────────────────────────────────────────────┤
│                  IPC 通信                   │
│  （レンダラー ↔ メイン間のメッセージング）    │
├─────────────────────────────────────────────┤
│               Main Process                  │
│  ファイル読み書き・検索/リンク/タグ生成・設定保存 │
│  （Node.js）                                │
└─────────────────────────────────────────────┘
         ↕
  Local Workspace
  Folder
```

---

## メインプロセスの責務

ファイルシステム、設定保存、検索・バックリンク・タグ・別名・年表 / 日付チャート表示用データの生成はメインプロセスが担う。レンダラーはpreloadで公開された `window.relic` APIだけを通して操作する。

| コンポーネント | 責務 |
|--------------|------|
| File Operations | Markdownファイルとフォルダの読み書き・作成・削除・移動・リネーム・複製 |
| Workspace Service | ワークスペース登録、切り替え、一覧削除、名前変更、前回ワークスペース復元 |
| Search / Backlinks / Tags / Aliases | Markdownファイルを読み取り、検索結果、バックリンク、タグ一覧、別名索引、フロントマター値候補を生成 |
| Chronicle | フロントマターから年表・日付チャート表示用データを生成し、暦換算・日付範囲・チャート書き戻し用の純粋処理を扱う |
| Tool Actions | マージ、見出し分割、タイトル一覧、目次生成を実行し、結果を新規Markdownファイルとして保存 |
| Cowork Service | ワークスペース内MarkdownのAI参照インデックス、AIチャット、AIによるMarkdown作成・編集・削除、保存済みAIデータの整合性修復を扱う |
| Settings Service | アプリ設定・ワークスペース設定・フロントマターテンプレート設定・入力能力設定の読み書き |
| IPC Handlers | レンダラーへ公開する操作単位を検証し、active workspaceとメインプロセスの処理へ接続する |
| Window Security | ウィンドウ生成オプション、外部遷移制限、権限要求拒否、終了前確認を扱う |

Cowork Serviceは、`aiWorkspaceService.ts` を公開操作の接続役にし、チャットモデル、参照収集、Markdown変更案の検証・適用、AI応答メッセージ整形、状態変換、中断判定を責務別モジュールへ分ける。AI変更案のパス検証とMarkdown書き込みはメインプロセス内に閉じ、レンダラーはpreload API経由で操作単位だけを呼び出す。現行UIでは変更案の承認キューを表示せず、送信処理の中で検証を通った変更だけを反映し、結果をAI回答文にまとめる。

アウトゴーイングリンクとアウトラインは、開いているファイルタブの内容からレンダラー側で導出する。ワークスペース全体を読む必要があるバックリンク、検索、チャートはメインプロセスに要求する。

## Electron境界と安全条件

- メインウィンドウは `contextIsolation: true`、`nodeIntegration: false`、`sandbox: true`、`webSecurity: true`、`allowRunningInsecureContent: false` で作成する
- レンダラーはNode.js APIへ直接アクセスせず、preloadの `contextBridge` で公開された `window.relic` APIだけを使う
- `window.relic` は、ファイル操作、ワークスペース操作、検索、設定、チャート、出力、ツール、AI設定、Cowork、クリップボード、ワークスペース変更通知、ウィンドウ終了確認応答を公開する
- IPC handlerは、レンダラーからの入力を処理前に検証し、不正な入力はメイン処理へ渡さず失敗として返す
- ファイル・フォルダ・ツール・検索・チャート系IPCでは、ワークスペース相対パス、ワークスペースID、設定値、選択肢配列、出力HTML/SVGなどを用途ごとに検証する
- 検索IPCだけは旧呼び出し形式の互換入力を現行の検索入力へ正規化してから処理する。正規化できない検索入力はメイン処理へ渡さない
- 設定ファイルの読み込みでは既存の壊れた保存値を安全側に正規化または除外する。一方、設定保存IPCでは不正な入力を丸めて保存せず、検証失敗として返す
- アクティブワークスペースが必要なIPCは、アプリ設定から現在のワークスペースを確認し、未選択の場合は処理せずエラーを返す
- メインプロセスはアクティブワークスペースだけを再帰監視し、`rename` と `change` イベントをワークスペース変更通知としてレンダラーへ送る
- ワークスペース変更通知は500ms程度の静かな時間を待ってまとめ、連続変更が続く場合でも最初のイベントから約2秒以内に通知する
- 通常のメインウィンドウでは、外部ページ遷移、新規ウィンドウ生成、webviewアタッチ、権限要求を許可しない
- 外部URLを開く場合は、`https://github.com` またはそのサブドメインだけをOS標準ブラウザへ渡し、アプリ内には読み込まない
- 開発時のVite dev server読み込みだけは、`localhost`、`127.0.0.1`、`::1` の候補へ短時間リトライする。本番ではビルド済みHTMLを読み込む
- 出力用の一時ウィンドウも、Node.js統合なし、sandboxあり、権限要求拒否、webview禁止、新規ウィンドウ禁止を維持する
- PDF出力用ウィンドウではインラインスクリプトを許可しない。印刷用ウィンドウでは印刷操作バーのためのインラインスクリプトだけを許可する
- アプリ終了時はレンダラーへ終了確認を送り、未保存本文や外部変更衝突があれば保存確認を通してから閉じる。開発版の終了処理中だけは、この終了前確認を通さず終了できる

## レンダラープロセスの責務

画面の描画とユーザー操作の受け付けを担う。

| コンポーネント | 責務 |
|--------------|------|
| LeftRail | プライマリサイドバー、Cowork、各種パネルタブ、チャートタブを開く入口 |
| FilesSidebar / AIChatsSidebar | プライマリサイドバー。ファイルモードではファイルツリー表示、ファイル/フォルダ操作、ピン留め、ファイル検索、ワークスペース作成/登録を扱い、CoworkモードではAIチャット履歴、チャット作成・削除、AI使用量表示を扱う |
| AppSecondarySidebar / AIWorkspacePanel | CoworkのAIチャット本文、ユーザー発言編集、メッセージ入力、送信、中断を扱う |
| FileTab | Markdownファイルの編集単位。CodeMirrorエディタ、ライブプレビュー、ソースモード、タイプライターモード、図表プレビューを含む |
| PanelTab | ファイル加工、フロントマター設定、暦設定、設定を表示する補助タブ |
| ChartTab | 年表 / 日付チャートを表示するタブ |
| RightPanel | アウトライン、バックリンク、アウトゴーイングリンクを表示する補助パネル |
| Editor | Markdown編集本体。ライブプレビュー、ソースモード、タイプライターモード、フロントマター固定プロパティ追加メニュー、ライブプレビュー表操作、Mermaid / D2 図表表示を含む |
| Diagram Preview | `mermaid` / `d2` コードブロックをSVGへ変換し、DOMPurifyで安全化してpan / zoom可能な表示へ接続する |
| EditorContextMenu | エディタの右クリックメニューからMarkdown書式挿入操作を提供 |
| SplitView | タブ表示領域の左右2ペイン分割表示 |
| StatusBar | 文字数・単語数の表示 |

## データの流れ（基本）

1. ユーザーがファイルを開く → レンダラーがメインにIPCで要求
2. メインがファイルを読み取り → レンダラーに内容を返す
3. レンダラーがファイルタブを開き、CodeMirror 6 に内容をセット → 編集可能になる
4. ユーザーが編集 → 入力停止後1秒でメインに自動保存要求
5. メインがファイルに書き込み → 次の検索・リンク・タグ・年表読み取りでは更新後のMarkdownを正として生成する
6. アウトラインとアウトゴーイングリンクは、アクティブなファイルタブの内容からレンダラー側で再導出する
7. チャートのバー編集はメインへIPCで要求し、メインが対象Markdownのフロントマターを書き換えた後、年表 / 日付チャートを再生成する

## データの流れ（Cowork）

1. ユーザーがCoworkを開く → レンダラーがメインにCowork状態を要求
2. メインがワークスペース内Markdownを走査し、AI参照用インデックスを作成または再利用する
3. ユーザーがAIチャットへ送信する → レンダラーはメッセージ、現在ファイルの未保存本文、未保存Markdownパスをメインへ渡す
4. メインが参照Markdown本文を集め、選択中のAI接続方式に応じてCodex App ServerまたはOpenAI APIへ依頼する
5. メインがAI回答とMarkdown変更案を受け取り、Markdownファイル、ワークスペース内パス、既存ファイル状態、未保存タブ状態を検証する
6. 検証を通った変更だけをメインがMarkdownへ反映し、反映結果または拒否理由をAI回答本文へ追加する
7. Markdownへ反映した場合はワークスペース変更通知を送り、レンダラー側のファイルツリー、ピン留め、開いているMarkdownタブを更新する

## ストレージ

| 種別 | 場所 | 内容 |
|------|------|------|
| Markdownファイル | ユーザーが登録した任意のローカルフォルダ内 | ユーザーが作成した `.md` ファイル |
| アプリ設定 | Electronのユーザーデータフォルダ内の `app-settings.json` | エディタ設定・機能トグル・画面設定・入力能力・フロントマターテンプレート・登録ワークスペース・AI接続設定 |
| ワークスペース設定 | Electronのユーザーデータフォルダ内の `workspaces/{workspaceId}.json` | ワークスペースパス・ピン留め・チャート設定 |
| Coworkデータ | Electronのユーザーデータフォルダ内の `ai-workspaces/{workspaceId}.json` | AIチャット、内部操作記録、Markdown参照用ローカルインデックス、ローカル埋め込み |
| OpenAI APIキー | Electronのユーザーデータフォルダ内の `ai-openai-key.json` | Electron `safeStorage` で暗号化したAPIキー。通常のアプリ設定、ワークスペース設定、Markdownには保存しない |
| 検索・リンク・タグ・年表表示用データ | メモリ上または要求時生成 | Markdownファイルとフロントマターから生成される派生データ。ワークスペース内には保存しない |

ローカルフォルダは、iCloud Drive / OneDrive / Dropbox などのクラウド同期フォルダでもよい。Relicは各クラウドサービスのAPI連携を持たず、OS上の通常のフォルダとして読み書きする。

CoworkデータはMarkdown正本ではない。Cowork状態の読み込み時に現在のMarkdownファイル一覧と照合し、削除済みMarkdownに対応するインデックス、埋め込み、古い会話参照、適用できない内部操作記録を取り除く。

OpenAI APIキーは `safeStorage` が使える環境でだけ保存できる。保存時は暗号化した値を専用ファイルへ安全書き込みし、macOS / Linuxでは可能な範囲でファイル権限を `0600` にする。
